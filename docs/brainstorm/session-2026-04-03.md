### Topic 4: Anchor Syntax — Decision-level Drift Detection

**Granularity chọn**: Decision-level (fine-grained, per decision anchor).

**Design principle**: AI sinh anchor tự động, user không thấy / không cần viết.

**Anchor format** (HTML comment → invisible khi render):
```markdown
<!-- vp:decision id="sidebar-behavior" session="session-2026-04-10" ts="ISO8601" -->
Sidebar collapsible mobile, slide-in 200ms, breakpoint 768px, active state: blue-600
<!-- /vp:decision -->
```

**Sinh khi nào**: Mỗi bullet trong `**Decisions**:` block → một anchor. `/save` scan + backfill anchors còn thiếu.

**ARCHITECTURE.md consumed anchor**:
```markdown
<!-- vp:consumed decision="sidebar-behavior" from="session-2026-04-03" at="ISO8601" -->
**Behavior**: Collapsible on mobile (breakpoint 768px), slide-in 200ms animation.
<!-- /vp:consumed -->
```

**Drift check algorithm**:
```
for each vp:consumed in docs:
  find matching vp:decision in latest sessions
  normalize text → compare
  if changed → DRIFT with risk:
    LOW:  visual/copy (color, label, text)
    MED:  layout/flow (component swap, structure)
    HIGH: logic/data/auth (matches compliance_domains → Gap G re-check)
```

**Drift report format**: per-item action — (a) auto-update / (b) skip / (c) flag for review.

**Docs covered**:
- ✅ ARCHITECTURE.md
- ✅ ROADMAP.md (task descriptions)
- ✅ TASK.md (objective + context_required)
- ❌ SYSTEM-RULES.md (too noisy)
- ❌ PROJECT-META.md (stable)

**Decisions**:
- [x] Decision-level granularity với anchor syntax
- [x] Anchor AI-generated, user-transparent (HTML comment)
- [x] Risk classification: LOW/MED/HIGH tự động
- [x] HIGH risk drift → Gap G compliance re-check tự động

### Topic 19: Định nghĩa Agents Orchestration — background + cô lập context (user)

**Context**: User làm rõ mục tiêu nâng cấp — không chỉ “nhiều agent” mà là **agent nền** với **nhiệm vụ cụ thể** và **context riêng**, tránh trà trộn vào main agent.

**Problem statement (đồng bộ với vp-auto hiện tại)**:
- **vp-auto** chạy **một luồng autonomous liên tục** trong **một cửa sổ ngữ cảnh** của host.
- Các lần đọc / hướng dẫn / quyết định **sớm trong phiên** dễ bị **đẩy ra khỏi context** (eviction) khi chuỗi task dài.
- Hệ quả: **forget**, lệch **workflow** đã thống nhất, implement phần sau **không còn khớp** constraints / UI direction / entity rules đã đọc trước đó — dù file state (HANDOFF) vẫn có thể đúng.

**Định nghĩa chốt (user)**:
- **Agents Orchestration** = các **background agent** được giao **charter hẹp** (read-only explore, cập nhật state, validation, v.v.), mỗi agent giữ **context sạch** cho phần việc đó.
- **Không** trộn ngữ cảnh suy nghĩ của worker vào **main agent** điều phối — tránh “prompt soup” và drift vai trò.
- **Handoff** giữa main ↔ worker phải **đi qua artifact có cấu trúc** (TASK.md, HANDOFF.json, PHASE-STATE, log append-only) thay vì hy vọng model nhớ toàn bộ lịch sử chat.

**Implications cho thiết kế ViePilot**:
- File-based state không chỉ để audit — trở thành **single source of truth** khi context model đã quên; orchestration **bắt buộc** đọc lại artifact trước khi bước tiếp.
- **Token budget / HANDOFF** (đã có trong horizon Post-MVP) và **Fork state updates** (background bookkeeping) là **đường ống liền mạch** với định nghĩa này — orchestration là **generalization**: mọi delegate đều có **scope + output schema** rồi merge.
- Host có **subagent / fork / background** (Claude Code) map trực tiếp; Cursor có thể **simulate** bằng session tách + quy ước merge file (cần adapter).

**Decisions**:
- [x] Ưu tiên thiết kế orchestration theo trục **context isolation + structured handoff**, không theo trục “thêm prompt dài cho main agent”.
- [x] Công nhận **vp-auto monolithic long-run** là **nguồn rủi ro fidelity** chính đáng — orchestration là **mitigation** chứ không chỉ tối ưu song song.
- [x] Background workers **không** thay thế control point / recovery — chỉ giảm tải ngữ cảnh và tách việc có ranh giới rõ.

**Open Questions** *(cập nhật sau Topic 19)*:
- Charter tối thiểu cho một background task: chỉ **read-only** trước, hay cho phép **write** trong `write_scope` con?
- Main agent sau mỗi delegate: **bắt buộc re-read** manifest + TASK.md hay chỉ summary ngắn từ worker?
- Mức “background”: **synchronous fork** (chờ kết quả) vs **async** (main tiếp tục — cần merge conflict policy)?

### Topic 4: Anchor Syntax — Decision-level Drift Detection

**Granularity chọn**: Decision-level (fine-grained, per decision anchor).

**Design principle**: AI sinh anchor tự động, user không thấy / không cần viết.

**Anchor format** (HTML comment → invisible khi render):
```markdown
<!-- vp:decision id="sidebar-behavior" session="session-2026-04-10" ts="ISO8601" -->
Sidebar collapsible mobile, slide-in 200ms, breakpoint 768px, active state: blue-600
<!-- /vp:decision -->
```

**Sinh khi nào**: Mỗi bullet trong `**Decisions**:` block → một anchor. `/save` scan + backfill anchors còn thiếu.

**ARCHITECTURE.md consumed anchor**:
```markdown
<!-- vp:consumed decision="sidebar-behavior" from="session-2026-04-03" at="ISO8601" -->
**Behavior**: Collapsible on mobile (breakpoint 768px), slide-in 200ms animation.
<!-- /vp:consumed -->
```

**Drift check algorithm**:
```
for each vp:consumed in docs:
  find matching vp:decision in latest sessions
  normalize text → compare
  if changed → DRIFT with risk:
    LOW:  visual/copy (color, label, text)
    MED:  layout/flow (component swap, structure)
    HIGH: logic/data/auth (matches compliance_domains → Gap G re-check)
```

**Drift report format**: per-item action — (a) auto-update / (b) skip / (c) flag for review.

**Docs covered**:
- ✅ ARCHITECTURE.md
- ✅ ROADMAP.md (task descriptions)
- ✅ TASK.md (objective + context_required)
- ❌ SYSTEM-RULES.md (too noisy)
- ❌ PROJECT-META.md (stable)

**Decisions**:
- [x] Decision-level granularity với anchor syntax
- [x] Anchor AI-generated, user-transparent (HTML comment)
- [x] Risk classification: LOW/MED/HIGH tự động
- [x] HIGH risk drift → Gap G compliance re-check tự động

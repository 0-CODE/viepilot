### Topic 2: Query Engine — Bài học cho vp-auto

**Context**: `~/DEV_PROJECTS/for-learn-claude-cli-main/reports/REPORT-query-engine.md`

**Insight cốt lõi**:

1. **Async Generator Pattern** — `submitMessage()` yield events realtime; tools execute overlap với streaming
2. **Withheld Messages + Layered Recovery**:
   ```
   Error → L1: collapse_drain_retry
         → L2: reactive_compact_retry
         → L3: max_output_tokens_escalate
         → L4: multi-turn recovery (max 3)
         → CHỈ surface cho user khi tất cả fail
   ```
3. **7 typed state transitions**: `collapse_drain_retry` | `reactive_compact_retry` | `max_output_tokens_escalate` | `max_output_tokens_recovery` | `stop_hook_blocking` | `token_budget_continuation` | `completed`
4. **Token budget + diminishing returns**: delta < 500 tokens + continuationCount ≥ 3 → inject nudge
5. **Stop Hooks** — fire-and-forget sau mỗi turn: extractMemories, autoDream, shell scripts

**Áp dụng cho ViePilot v2**:
- vp-auto cần typed state transitions thay vì prose workflow
- 3-layer silent recovery trước control point
- Task complexity estimate để detect "đang bị stuck"

**Decisions**:
- [x] Thay thế prose-based workflow bằng typed state machine trong PHASE-STATE.md
- [x] Implement 3-layer silent recovery: lint → test → scope-reduce → control_point

### Topic 4: Multi-Agent Coordinator — Bài học cho parallel execution

**Context**: `~/DEV_PROJECTS/for-learn-claude-cli-main/reports/REPORT-multi-agent-coordinator.md`

**3 execution models của Claude Code**:
```
SYNC:     AgentTool → block → return result
ASYNC:    AgentTool → return agentId ngay → <task-notification> XML khi xong
TEAMMATE: spawnTeammate → mailbox-based messaging
```

**Fork Pattern + Prompt Cache Sharing**:
- Tất cả fork children có prefix **byte-identical** với parent
- Chỉ khác final directive → maximize cache hits, ~70% cost reduction
- Services dùng fork: extractMemories, autoDream, compact, SessionMemory

**Coordinator triết lý**:
> *"Parallelism is your superpower. Workers are async. Launch independent workers concurrently whenever possible."*

**Plan Approval Gate**: Teammate tạo plan → gửi `plan_approval_request` → Lead approve/reject trước khi execute.

**Áp dụng cho ViePilot v2**:
- Sau task PASS: fork background agent cho CHANGELOG/TRACKER/ROADMAP updates (không blocking)
- Task file khai báo `can_parallel_with: [task-X]` → spawn parallel workers
- Sub-agent scope contract (fork boilerplate) để prevent drift

**Fork Boilerplate cho vp-auto** (áp dụng Claude Code pattern):
```
SCOPE CONTRACT - Task {X}
├── Execute: {specific objective only}
├── Write scope: {specific files/dirs}
├── Do NOT: modify out-of-scope files, ask questions
├── Output: YAML report starting with "SCOPE: Task {X}"
└── End with: "RESULT: PASS|FAIL|BLOCKED — {reason}"
```

**Decisions**:
- [x] Background fork cho state updates sau task PASS
- [x] Scope contract bắt buộc cho mỗi task execution
- [ ] Parallel execution infrastructure (Post-MVP)

### Topic 5: Permission System — Bài học cho safety và recovery

**Context**: `~/DEV_PROJECTS/for-learn-claude-cli-main/reports/REPORT-permission-system.md`

**5 permission modes**:
```
default → acceptEdits → plan → bypassPermissions → auto (ML)
```

**8-stage BashTool pipeline**: mode → path constraints → sed constraints → sandbox → 23 security checks → split commands → rule matching → ML classifier

**Defense in depth**:
```
Permission System (layer 1): Tool có được execute không?
Sandbox System   (layer 2): Khi execute, OS cho phép làm gì?
```

**Circuit Breaker** (auto mode): 3 consecutive denials → fallback to manual.

**Áp dụng cho ViePilot v2**:
- vp-auto "plan mode": read-only exploration trước khi execute (tương đương plan mode của Claude Code)
- Task write_scope declaration → vp-auto verify AI không write ngoài scope
- Recovery circuit breaker: 3 lần L1/L2 silent recovery fail → escalate to control point ngay

**Decisions**:
- [x] Task metadata `write_scope` declaration bắt buộc
- [x] vp-auto verify write scope sau mỗi sub-task commit
- [x] Circuit breaker: max 2 silent recovery attempts per layer

### Topic 7: Vấn đề kiến trúc sâu — 3 cải tiến cốt lõi

**Vấn đề 1: Prose workflow vs Typed State Machine**

ViePilot hiện tại: `autonomous.md` là 468 dòng prose — AI phải re-read mỗi lần.

Claude Code: typed transitions, AI biết state hiện tại + transitions khả dụng mà không cần đọc lại.

**Đề xuất**:
```yaml
# .viepilot/phases/N/PHASE-STATE.md — thêm section:
execution_state:
  current: task_2_3
  status: executing
  available_transitions:
    on_pass: → update_state → task_2_4
    on_fail_l1: → auto_fix_lint (max 1) → re_verify
    on_fail_l2: → auto_fix_tests (max 2) → re_verify
    on_fail_l3: → control_point
```

**Vấn đề 2: TRACKER.md blob vs Index + Topic Files**

ViePilot: TRACKER.md chứa tất cả → phình theo thời gian.
Claude Code: MEMORY.md = index ≤200 dòng LUÔN có, topic files chỉ load khi relevant.

**Đề xuất**: TRACKER.md ≤30 dòng (current state pointer only), chi tiết trong `logs/`.

**Vấn đề 3: 1-tầng recovery vs 4-tầng silent recovery**

ViePilot: bất kỳ failure → control point ngay.
Claude Code: 4 tầng automatic recovery trước khi hỏi user.

**Đề xuất**:
```
L1 — Lint/Format (silent, 1 attempt) → re-verify
L2 — Targeted test fix (silent, 2 attempts) → re-verify
L3 — Scope reduction (silent, 1 attempt) → PARTIAL
--- control_point (user thấy lần đầu) ---
```

**Decisions**:
- [x] Refactor PHASE-STATE.md template để include execution_state với typed transitions
- [x] Refactor TRACKER.md template thành index ≤30 dòng
- [x] Implement 3-layer silent recovery trong vp-auto workflow

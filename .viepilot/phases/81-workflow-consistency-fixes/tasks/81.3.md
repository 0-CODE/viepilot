# Task 81.3 — workflows/brainstorm.md: Pre-Save Phase Validation Gate (ENH-052)

## Objective
Add a mandatory pre-save validation block to `workflows/brainstorm.md` Step 6 "Save Session"
that blocks writing the session file when features exist but no phase assignments have been made.
Prevents downstream failure in `crystallize.md` Step 1 due to missing phase data.

## Paths
- `workflows/brainstorm.md`

## File-Level Plan

### Insert — in Step 6 (`save_session`), before the file-write block

Add the following validation block immediately before the session file is written to disk:

```markdown
### Pre-Save Phase Assignment Validation

Before writing the session file, check phase assignment completeness:

```bash
# Check 1: does session_data contain a non-empty ## Phases section?
PHASES_PRESENT=$(grep -c "## Phases" "$SESSION_DRAFT" 2>/dev/null || echo "0")

# Check 2: does Phase 1 have at least one feature assigned?
PHASE1_CONTENT=$(sed -n '/## Phases/,/## /p' "$SESSION_DRAFT" | grep -c "Phase 1" || echo "0")

# Check 3: are there unassigned features?
UNASSIGNED=$(grep -c "^- \[ \]" "$SESSION_DRAFT" 2>/dev/null || echo "0")
```

**Gate condition:**
- If scope was locked (`SCOPE_LOCKED=true`) AND (`PHASES_PRESENT == 0` OR `PHASE1_CONTENT == 0`):
  → **Block save**. Show:
  ```
  ⚠️  Phase assignment incomplete — cannot save session.

  Found features but no phase assignments. Before saving:
  1. Assign all features to phases (## Phases section)
  2. Ensure Phase 1 has at least one feature

  Return to Step 3 to assign phases, then re-run save.
  ```
- If `SCOPE_LOCKED=false` (exploratory session — no features discussed):
  → **Allow save** without phase check. Add note to session file:
  ```
  > ⚠️ Exploratory session — no phase assignments. Run /vp-brainstorm again to lock scope before crystallize.
  ```
- If all checks pass → proceed with file write.
```

## Implementation Notes
- `SCOPE_LOCKED` variable should already be set earlier in Step 5 (profile binding / scope confirmation)
- If `SCOPE_LOCKED` is not explicitly tracked in the current workflow, add it: set `SCOPE_LOCKED=true` when user confirms "scope locked" in Step 2 confirmation prompt
- The exploratory bypass covers legitimate use cases like pure idea-capture sessions

## Acceptance Criteria
- [ ] Pre-save validation block present in Step 6 before file-write
- [ ] Gate triggers when `SCOPE_LOCKED=true` AND phases missing or Phase 1 empty
- [ ] Exploratory sessions (`SCOPE_LOCKED=false`) bypass gate with advisory note
- [ ] Block message clearly tells user which step to return to
- [ ] Gate does NOT trigger for brownfield stub sessions (they have no phases by design)

## [2.18.0] - 2026-04-18

### Fixed
- **vp-rollback** (BUG-014): Step 7 now parses enriched tag format
  `{project}-{branch}-{version}-vp-p{N}-t{M}` introduced by ENH-050. Rollback to
  any Phase 57+ checkpoint correctly restores HANDOFF.json phase/task state.

### Changed
- **vp-crystallize** (ENH-051): Added "Brownfield Execution Path" table clarifying
  which sub-steps (1A–1D) run, skip, or are conditional in brownfield mode.
- **vp-brainstorm** (ENH-052): Step 6 now validates phase assignment before saving.
  Sessions with features but no phase assignments are blocked; exploratory sessions bypass gate.
- **Version bump rules** (ENH-053): Consolidated into `SYSTEM-RULES.md` as single
  canonical table. `evolve.md` and `autonomous.md` now reference this table instead of
  defining rules inline.
- **vp-audit post-phase hook** (ENH-054): Auto-hook integration upgraded from conceptual
  note to concrete `<step name="post_phase_audit">` block in `autonomous.md`. Runs fast
  Tier 1+2 checks after every phase-complete event; silent when no issues found.
- **AskUserQuestion enforcement** (ENH-055): All 13 AUQ prompt blocks across 4 workflows
  now explicitly mark Claude Code adapter as REQUIRED. Plain-text menus remain as
  fallback for non-Claude-Code adapters only.

### Files
- `workflows/rollback.md` — Step 7 enriched tag parse (3 formats)
- `workflows/crystallize.md` — Brownfield Execution Path table
- `workflows/brainstorm.md` — Pre-save phase validation gate

- `workflows/evolve.md` — Version bump ref → SYSTEM-RULES.md
- `workflows/autonomous.md` — Version bump ref + `post_phase_audit` step
- `workflows/audit.md` — Auto-hook section → concrete integration spec
- `workflows/evolve.md`, `request.md`, `brainstorm.md`, `crystallize.md` — AUQ REQUIRED
- `skills/vp-evolve`, `vp-request`, `vp-brainstorm`, `vp-crystallize` SKILL.md — AUQ REQUIRED

# Task 81.1 — workflows/rollback.md: Enriched Tag Parse (BUG-014)

## Objective
Fix `workflows/rollback.md` Step 7 "Update State Files" to parse the enriched tag format
introduced by ENH-050: `{project}-{branch}-{version}-vp-p{N}-t{M}[-done|-complete]`.
Without this fix, rolling back to any tag created from Phase 57 onward leaves HANDOFF.json
with incorrect phase/task state.

## Paths
- `workflows/rollback.md`

## File-Level Plan

### Edit — Step 7 (`update_state`): add enriched format parse branch

**Location**: The parse logic block under `## 7. Update State Files`

**Current logic:**
```
Parse tag to determine phase/task (support both legacy and project-scoped):
- `vp-p{N}-t{M}` or `{project}-vp-p{N}-t{M}` → Phase N, Task M, status: in_progress
- `vp-p{N}-t{M}-done` or `{project}-vp-p{N}-t{M}-done` → Phase N, Task M+1, status: not_started
- `vp-p{N}-complete` or `{project}-vp-p{N}-complete` → Phase N+1, Task 1, status: not_started
```

**Replace with (adds enriched format as 3rd alternative for each pattern):**
```
Parse tag to determine phase/task — 3 supported formats:

Format A (legacy):          `vp-p{N}-t{M}[-done]`
Format B (project-scoped):  `{project}-vp-p{N}-t{M}[-done]` or `{project}-vp-p{N}-complete`
Format C (enriched, ENH-050): `{project}-{branch}-{version}-vp-p{N}-t{M}[-done|-complete]`

For all formats, extract N and M by matching the terminal `-vp-p{N}` and `-t{M}` segments:
```bash
# Extract phase number
PHASE_NUM=$(echo "$TAG" | grep -oE 'vp-p[0-9]+' | grep -oE '[0-9]+$')
# Extract task number (if present)
TASK_NUM=$(echo "$TAG" | grep -oE 't[0-9]+' | tail -1 | grep -oE '[0-9]+$')

# Determine intent
if echo "$TAG" | grep -q '\-complete$'; then
  # Phase complete → restore to start of next phase
  RESTORE_PHASE=$((PHASE_NUM + 1)); RESTORE_TASK=1; RESTORE_STATUS="not_started"
elif echo "$TAG" | grep -q '\-done$'; then
  # Task done → restore to start of next task
  RESTORE_PHASE=$PHASE_NUM; RESTORE_TASK=$((TASK_NUM + 1)); RESTORE_STATUS="not_started"
else
  # Task in-progress
  RESTORE_PHASE=$PHASE_NUM; RESTORE_TASK=$TASK_NUM; RESTORE_STATUS="in_progress"
fi
```

Update HANDOFF.json with RESTORE_PHASE, RESTORE_TASK, RESTORE_STATUS.
Update TRACKER.md progress accordingly.
```

## Implementation Notes
- The `grep -oE 'vp-p[0-9]+'` pattern works for all 3 formats because each format ends with `...-vp-p{N}-...`
- Using `tail -1` on the task extract handles cases where the project prefix contains a digit-prefixed segment
- Do NOT change the backup or git reset logic — only the state-parse block changes

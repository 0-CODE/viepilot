# Task 49.1 — lib/viepilot-config.cjs: Language config schema, read/write, defaults

**Phase**: 49 | **Status**: in_progress

## Paths
- `lib/viepilot-config.cjs` (new)

## Objective
Create a new library module that manages the ViePilot language configuration stored in `~/.viepilot/config.json`.
Config schema has two keys:
- `language.communication` — language for AI↔user communication (banners, prompts). Default: `"en"`
- `language.document` — language for generated project files (ROADMAP, TRACKER, etc.). Default: `"en"`

## File-Level Plan
### lib/viepilot-config.cjs
- `DEFAULTS` constant: `{ language: { communication: 'en', document: 'en' } }`
- `getConfigPath(overrideHomedir?)` — returns `~/.viepilot/config.json`
- `readConfig(overrideHomedir?)` — reads JSON, deep-merges with DEFAULTS, returns merged object
- `writeConfig(patch, overrideHomedir?)` — deep-patches existing config, writes back
- `resetConfig(overrideHomedir?)` — writes DEFAULTS to file
- `getLanguage(key, overrideHomedir?)` — shorthand for `readConfig()[key]`
- All functions: handle missing file gracefully (return DEFAULTS when absent)
- Export all 5 functions + DEFAULTS

## Acceptance
- [ ] 4 exported functions: readConfig, writeConfig, resetConfig, getConfigPath
- [ ] Defaults are en/en
- [ ] Missing config file returns defaults (no throw)
- [ ] writeConfig merges (does not overwrite entire config)

## Implementation Notes
- Use `os.homedir()` for default home; accept `overrideHomedir` for tests
- JSON format; mkdir recursive if `~/.viepilot/` missing
- No external dependencies — only Node built-ins

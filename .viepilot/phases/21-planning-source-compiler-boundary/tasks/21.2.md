- `.viepilot/schemas/execution-graph.schema.json`: define the compile-time dependency graph contract for phases and tasks without embedding mutable runtime status.
- `.viepilot/schemas/active-packet.schema.json`: define the narrowed execution packet delivered to `/vp-auto`, including objective, scoped file reads/writes, verification directives, and packet lifecycle.
- `.viepilot/schemas/runtime-artifact-contracts.md`: document lifecycle ownership, compile versus runtime mutability, and explicit non-overlap rules among the three runtime artifacts.

- `.viepilot/ARCHITECTURE.md`: describe the runtime artifact boundaries in system/data-flow terms and keep the compiler/runtime/projection split explicit.
- `.viepilot/SPEC.md`: register the schema paths and add a concise runtime-artifact contract table so later phases can reference one canonical contract summary.
- `.viepilot/schemas/database-schema.sql`: align the logical persistence columns with the runtime-state and active-packet contract shape.

## Implementation Notes

- `runtime-state.json` owns mutable executor truth only. It should not duplicate the phase/task dependency graph or embed the full active packet body.
- `execution-graph.json` owns planned topology and ordering only. It should remain compiler-emitted and not carry mutable execution counters.
- `active-packet.json` owns the currently actionable work bundle only. It can reference runtime and graph artifacts, but it must not become a second source of truth for global runtime state.
- Keep contract granularity at the task level for now; the brainstorm explicitly left subtask granularity as a future decision.
- Model packet issuance as `per-task` for the initial contract so Phase 23 can narrow `/vp-auto` reads without introducing per-turn churn yet.
- Stack cache summaries are unavailable under `~/.codex/viepilot/stacks/` in this environment, so this task proceeds with repo-local system rules and explicit assumptions only.

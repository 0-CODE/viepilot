## Implementation Notes

- Keep runtime artifact details intentionally high level in this task. `runtime-state.json`, `execution-graph.json`, and `active-packet.json` field-level contracts belong to Task 21.2.
- Make ownership explicit with a singular source-of-truth rule: planning-source fields are canonical only until compile emits runtime artifacts; projections remain derived views.
- Avoid introducing fields that depend on rereading brainstorm prose at execution time. Brainstorm session identifiers and extracted decisions can be referenced, but runtime execution must consume compiled artifacts later.
- Preserve CommonJS and local-first framing; this task is documentation/schema work only.

## Best Practices

- Prefer machine-readable schema before prose explanation.
- Keep canonical versus projection ownership explicit for every field group.
- Use stable kebab-case artifact names under `.viepilot/schemas/`.
- Keep schema comments and markdown inventory synchronized so later compiler work has one interpretation.

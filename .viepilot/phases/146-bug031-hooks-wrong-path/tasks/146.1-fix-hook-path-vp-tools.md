# Task 146.1 — bin/vp-tools.cjs: Fix hook path + migrate stale entries

## Objective
Fix BUG-031: `vp-tools hooks scaffold` and `vp-tools hooks install` both use a hardcoded
`path.join(home, '.viepilot', 'hooks', 'brainstorm-staleness.cjs')` that never gets created.
Correct path uses `adapter.viepilotDir(home)`. Also fix the idempotency check so re-running
`hooks install` replaces stale wrong-path entries rather than leaving them in place.

## Paths
- `bin/vp-tools.cjs`

## File-Level Plan

### Fix 1: hookCommand path (lines 1010 and 1027)

**Note**: `adapter` is already in scope at both call sites (read context around lines 999-1027
to confirm the variable name used for the resolved adapter object).

### Fix 2: idempotency check migration (around line 1039-1048)

**Current** — checks if the exact `hookCommand` (new correct path) is already installed.
This leaves old wrong-path entries in place forever.

**Replace** with a two-step check:
```js
// Step A: remove any stale entry with the OLD wrong path
const wrongPath = `node ${path.join(home, '.viepilot', 'hooks', 'brainstorm-staleness.cjs')}`;
settings.hooks.Stop = settings.hooks.Stop.filter((entry) =>
  !(Array.isArray(entry.hooks) &&
    entry.hooks.some((h) => h.type === 'command' && h.command === wrongPath))
);

// Step B: idempotent check for correct path
const alreadyInstalled = settings.hooks.Stop.some((entry) =>
  Array.isArray(entry.hooks) &&
  entry.hooks.some((h) => h.type === 'command' && h.command === hookCommand)
);
if (alreadyInstalled) {
  console.log(formatSuccess('ViePilot staleness hook already installed.'));
  console.log(`  Config: ${configPath}`);
  process.exit(0);
}
```

This way: first run after upgrade → removes stale entry → adds correct entry.
Subsequent runs → finds correct entry → skips (idempotent).

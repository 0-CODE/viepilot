# Task 77.1 — Gap A: Git Submodule Detection

## Objective
Extend `workflows/crystallize.md` Signal Category 1 (Build Manifests) to detect and scan git submodules. The scanner must parse `.gitmodules` at repo root, run Signal Cat 1–4 on each initialized submodule path, and record all submodules in `modules[]` with `type: submodule`.

workflows/crystallize.md

## File-Level Plan

### Location in `workflows/crystallize.md`

Insert a new **Git Submodule Detection** block immediately after the existing monorepo detection block (after line ~166: "If monorepo detected → scan each module separately…").

### Content to add

**New subsection: "Git Submodule Detection"**

```markdown
**Git Submodule Detection** (run after monorepo check):
- Check for `.gitmodules` file at repo root
- If present: parse all `[submodule "name"]` blocks → extract `name`, `path`, `url`
- For each submodule path:
  - If path exists on disk (initialized):
    - Run Signal Cat 1 (manifest scan) on `{path}/`
    - Run Signal Cat 2 (framework) on `{path}/`
    - Run Signal Cat 4 (DB signals) on `{path}/`
    - Record `initialized: true`
  - If path absent (not initialized):
    - Record `initialized: false`, `primary_language: MISSING`
    - Add open question: "Submodule '{name}' not initialized — run `git submodule update --init {path}` to scan it"
- Add each submodule to `modules[]` with `type: submodule`

SAFETY RULE: Never run `git submodule update`, `git submodule init`, or any git network command during scan. Read filesystem only.
```

### Scan Report schema update

In the Scan Report Schema section, update `modules[]` entry to include new fields:

```yaml
modules:
  - name: string                    # submodule name from .gitmodules OR workspace package name
    type: submodule | workspace | root   # NEW field
    path: string                    # relative path from repo root
    submodule_url: string | null    # NEW: remote URL from .gitmodules (null if not submodule)
    initialized: bool               # NEW: true if path exists on disk (submodules only)
    primary_language: string
    framework: string | null
    # ... existing fields
```

### `workflows/crystallize.md` — 3 additions

#### Addition 1: Per-Module MUST-DETECT rules (new subsection in Signal Category 1)

Insert after the Polyrepo Detection block (Task 77.2), before Signal Category 2:

```markdown
**Per-Module Gap Detection** (applied to every entry in `modules[]` after scanning):

Each module is assessed independently against the following MUST-DETECT fields:

| Field | Source signals | Tier if absent |
|-------|---------------|---------------|
| `primary_language` | manifest extension, file survey (Signal Cat 12), `tsconfig.json`, `pyproject.toml` | MISSING — must ask user |
| `framework` | Signal Cat 2 dep patterns on module path | ASSUMED if no dep match; MISSING if no manifest |
| `module_purpose` | manifest `description`, directory name convention, README first line | ASSUMED (inferred from dir name) |
| `entry_point` | `main` field in `package.json`; `src/index.*`; `cmd/main.go`; `Application.java` | ASSUMED if standard path exists; MISSING otherwise |

**Gap tier assignment per module:**
```
DETECTED  — all MUST-DETECT fields sourced from file evidence (no inference)
ASSUMED   — ≥1 MUST-DETECT field inferred (no direct file evidence, but plausible)
MISSING   — ≥1 MUST-DETECT field has no evidence and cannot be inferred
```

**`must_detect_status` structure** (record per MUST-DETECT field):
```yaml
must_detect_status:
  primary_language: { value: TypeScript, source: tsconfig.json,  tier: DETECTED }
  framework:        { value: NestJS,     source: package.json,   tier: DETECTED }
  module_purpose:   { value: "REST API", source: inferred,       tier: ASSUMED  }
  entry_point:      { value: src/main.ts, source: package.json,  tier: DETECTED }
```
Use `source: inferred` when value is derived by convention (not read from a file).
Use `source: absent` when no evidence exists (`tier: MISSING`).

**Root gap tier rollup:**
```
root gap_tier = worst tier across all modules
Priority: MISSING > ASSUMED > DETECTED
```
If any module is MISSING → root gap_tier = MISSING → full scan blocks until resolved.
If all modules are DETECTED or ASSUMED → root gap_tier matches the worst module.

**Pause-and-ask rule for MISSING modules:**
When any module has `gap_tier: MISSING`:
```
⛔ Module '{name}' (path: {path}) has MISSING fields:
  - {field}: no evidence found

Please provide:
  {field}? 
```
Ask once per MISSING field per module. Record answer as `{ value: user_input, source: user, tier: DETECTED }`.
Do NOT proceed to artifact generation until all MISSING module fields are filled.
```

# Task 92.1 ‚Äî Update `workflows/brainstorm.md` ‚Äî skill registry load + silent apply

## Objective
Add a skill integration step to the UI Direction Mode section: load skill registry, match capabilities to UI signals, silently apply best practices, record `## skills_used` in notes.md.

## Paths
- `workflows/brainstorm.md`

## Implementation Notes

### Where to insert

Inside `### UI Direction Mode (design-in-the-loop; ...)` ‚Äî after the initial workspace creation block and before the "Required hook (multi-page only)" section.

New subsection: `### Skill Registry Integration (FEAT-020)`

### Content to add

```markdown
### Skill Registry Integration (FEAT-020)

**Trigger**: UI Direction Mode is active (at least one HTML file being generated or updated).

**Step ‚Äî Load registry:**
- Call `loadRegistry()` from `lib/skill-registry.cjs` (read `~/.viepilot/skill-registry.json`)
- If registry absent or empty: skip silently ‚Äî no warning, no prompt

**Step ‚Äî Match skills to UI signals:**
Map brainstorm UI signal keywords to capability tags:

| UI Signal keywords | Capability match |
|-------------------|-----------------|
| `component`, `layout`, `screen`, `page`, `UI`, `UX` | `ui-generation`, `component-design` |
| `responsive`, `mobile`, `grid` | `responsive-layout` |
| `design`, `theme`, `color`, `typography` | `design-system`, `design-tokens` |
| `form`, `button`, `input`, `modal` | `component-design` |

For each registered skill: if `skill.capabilities` intersects matched capability tags ‚Üí skill is **active**.

**Step ‚Äî Silent apply:**
- For each active skill: prepend `skill.best_practices[]` to the context used for HTML generation
- **No banner, no prompt, no mention to user** ‚Äî best practices blend into generated output
- This is background-only, like Background UI Extraction (ENH-026)

**Step ‚Äî Record in notes.md:**
After the session's first UI artifact is generated (or updated), append to `notes.md`:

```yaml
## skills_used
- id: frontend-design
  version: null
  trigger: ui-generation signal
  applied_at: brainstorm-session
  best_practices_applied:
    - Mobile-first
    - Design tokens
```

If `## skills_used` already exists: merge (add new skills, update applied_at).
```

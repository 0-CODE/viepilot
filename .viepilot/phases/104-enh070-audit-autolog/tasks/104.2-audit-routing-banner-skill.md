```markdown
### Auto-Log Behavior (ENH-070)
By default, `vp-audit` automatically logs each detected gap as a request file:
- Tier 1 findings → `BUG-N` (state inconsistency)
- Tier 2 findings → `BUG-N` (doc drift) or `ENH-N` (missing docs)
- Tier 3 findings → `BUG-N` (stack violation) or `ENH-N` (best-practice gap)
- Tier 4 findings → `ENH-N` (framework gap)

Duplicate detection: if a matching open request already exists, the finding is appended to it instead of creating a new file.

After logging, the post-audit banner shows all request IDs and offers `/vp-evolve` as the recommended next action.

To disable: `vp-audit --no-autolog`
```

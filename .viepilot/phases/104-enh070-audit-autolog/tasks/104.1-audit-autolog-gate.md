## Objective
Add an **Auto-Log Gate** block to `workflows/audit.md` that fires after each tier's report section (1g, 2f, 3g, 4f). For every detected ⚠️ or ⛔ finding, the gate automatically creates a `.viepilot/requests/{TYPE}-{N}.md` file and updates TRACKER.md — no manual `/vp-request` call needed.

## Paths
```
workflows/audit.md
```

## File-Level Plan

### `workflows/audit.md` — insert Auto-Log Gate block

Add a new section **`## Auto-Log Gate (ENH-070)`** after the existing tier report sections (after `### 4f. Report Tier 4 results` and before the `--fix` section). This single gate block is called after ALL tiers complete (not per-tier) to avoid TRACKER churn.

**Gate logic:**

```markdown
## Auto-Log Gate (ENH-070)

After all tiers complete, for each detected issue in the findings list:

### Determine request type
| Tier | Issue category | Request type |
|------|----------------|-------------|
| 1 | State inconsistency / HANDOFF drift / git tag missing | BUG (medium) |
| 1 | Execute-first ordering risk | BUG (medium) |
| 2 | Doc drift (README/CHANGELOG version) | BUG (low) |
| 2 | Missing docs / placeholder URLs | ENH (low) |
| 3 | Stack violation / correctness anti-pattern | BUG (high) |
| 3 | Stack improvement / best-practice gap | ENH (medium) |
| 4 | Framework integrity gap | ENH (high) |

### Duplicate detection (before creating)
Before writing any request file:
1. List `.viepilot/requests/*.md`
2. For each existing file with `Status: new | triaged | in_progress`:
   - Compare title (case-insensitive substring ≥ 70% match)
   - Compare `## Related → Files` (any file path overlap)
3. If duplicate found:
   - Append to existing file under `## Discussion`:
     `Re-detected by vp-audit {date}: {finding_text}`
   - Do NOT create a new file
4. If no duplicate: create new request file

### Create request file
Path: `.viepilot/requests/{TYPE}-{N}.md`
Content:
```markdown
# {TYPE}: {title from audit finding}

## Meta
- **ID**: {TYPE}-{N}
- **Type**: Bug | Enhancement
- **Status**: new
- **Priority**: {severity from tier table above}
- **Created**: {today}
- **Reporter**: vp-audit (auto-logged)
- **Assignee**: AI
- **Source**: auto-logged by vp-audit Tier {N}

## Summary
{audit finding text verbatim}

## Details
- **Tier**: {N}
- **Audit date**: {today}
- **Affected files**: {files from finding, if any}

## Acceptance Criteria
- [ ] Fix the detected issue
- [ ] Re-run vp-audit: this finding no longer appears

## Related
- Files: {affected files}
- Dependencies: —

## Resolution
—
```

### Update TRACKER.md
After creating all request files, append each new ID to the decision log:
```
| {today} | {TYPE}-{N} auto-logged by vp-audit: {short title} | `vp-audit` | Backlog |
```

### Summary counter
Track:
- `auto_logged_new`: count of new request files created
- `auto_logged_deduped`: count of existing files updated (re-detected)
```

## Implementation Notes

### 1 — Coverage gate (insert in Interactive Gap-Filling section)

In the "Interactive Gap-Filling (Step 0-B-ii)" section, BEFORE item 1 ("Display Scan Report
summary table"), insert:

```markdown
**Coverage gate (ENH-081):** Before presenting the Scan Report, verify the trace:

1. Read `## Signal Coverage` table from `BROWNFIELD-TRACE.md`.
2. Count rows still showing `planned` (never ran).
3. If any `planned` rows found: display warning banner:
   ```
   ⚠️ Coverage incomplete — {N} signal categories not executed:
     - Signal Cat {N}: {Category Name}
   Proceeding with partial Scan Report — gap fields will be MISSING.
   ```
   (Non-blocking — continue with gap-filling using available data.)
4. Update trace: Status → `scan_complete` in `## Session Info`.
5. Update `## Step Completion` row for "0-B Scanner" → `done` + timestamp.
```

### 2 — Gap Filling Log (extend Interactive Gap-Filling section)

At the end of the Interactive Gap-Filling section instructions (after item 6), add:

```markdown
**Gap Filling Log (ENH-081):** For each field where the user provides a value, append to
`## Gap Filling Log` in `BROWNFIELD-TRACE.md`:

| Field | Status | Source | Value |

| {field_name} | DETECTED → user-confirmed | user input | {value} |
| {field_name} | MISSING → user-filled | user input | {value} |

Write after each user response — do not batch at end of gap-filling.
```

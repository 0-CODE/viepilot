# Task 122.2 — Per-category trace update protocol

## Objective

Add a "Trace Update Protocol" section to `workflows/crystallize.md` that specifies exactly how
each signal category (1–13) updates the trace file as it runs. This is the core of the coverage
tracking: marking categories as `scanning` when they begin, logging visited files, and marking
`done`/`skipped`/`assumed` when complete.

## Paths

```
workflows/crystallize.md
```

## Implementation Notes

### Insertion point

Insert a new subsection **after the Signal Category 13 section** and **before the Scan Report
Schema (finalized)** section. Find:

```
### Scan Report Schema (finalized)
```

Insert a new section just before this heading.

### Content to insert

```markdown
### Brownfield Scan Trace — Per-Category Update Protocol (ENH-081)

For each signal category (1–13), follow this update protocol:

#### On category start:

Update the category's row in `## Signal Coverage` table:
- Status: `scanning`
- Started: current ISO timestamp

#### On category complete:

Update the category's row:
- Status: `done` | `skipped` | `assumed`
  - `done`: evidence found with high confidence (DETECTED)
  - `assumed`: evidence found with low confidence or indirect signal (ASSUMED)
  - `skipped`: no relevant files found for this category (no css/tests/infra etc.)
- Files Checked: N (integer count of files actually read)
- Signals Found: comma-separated key results, or "none" if nothing detected
- Duration: elapsed seconds since category started

Append to `## Files Read Log`:

```markdown
### Signal {N} — {Category Name}

- [x] {relative/path/to/file}  → {signal or "none found"}
- [x] {another/file}           → {result}
- [ ] {expected/missing.yaml}  → not present
```

Legend:
- `[x]` = file was read
- `[ ]` = file was expected (from workflow probe list) but not found on disk

**Empty category:** If no files were visited (category not applicable to project), append:

```markdown
### Signal {N} — {Category Name}

(No files probed — category not applicable)
```

#### Status transitions

```
planned → scanning  (on category start)
scanning → done     (evidence found, DETECTED tier)
scanning → assumed  (evidence found, ASSUMED tier — low confidence)
scanning → skipped  (no matching files found at all)
```

#### Example — Signal Category 5 (API Contract Files)

Row update:
```
| 5 | API Contract Files | done | 3 | openapi.yaml, swagger.json (REST) | 14:23:05 | 2s |
```

Files Read Log append:
```markdown
### Signal 5 — API Contract Files

- [x] openapi.yaml              → REST (OpenAPI 3.0)
- [x] swagger.json              → REST (Swagger 2.0)
- [ ] graphql/schema.graphql    → not present
- [x] src/routes/index.ts       → REST route patterns
```

#### Signal Cat 13 special fields

Signal Category 13 (UI/Design) additionally records in its row:
- Signals Found: `tailwind ({N} colors), {M} routes ({framework}), {K} components ({ui_library})`

```

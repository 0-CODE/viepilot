# Task 122.1 — Trace initialization + resume detection (Step 0-B start)

## Objective

Add a "Brownfield Scan Trace — Initialization" sub-step at the very beginning of Step 0-B in
`workflows/crystallize.md`, executed BEFORE any signal category runs. This creates
`.viepilot/BROWNFIELD-TRACE.md` with all 13 categories pre-populated as "planned" and handles
resume detection for interrupted or previously-completed scans.

## Paths

```
workflows/crystallize.md
```

## Implementation Notes

### Insertion point

In Step 0-B, BEFORE the "1. Run the full 12-category codebase scanner" instruction. Find:

```
1. Run the full 12-category codebase scanner (Signal Categories 1–12 below).
```

Insert immediately before this line.

### Content to insert

```markdown
### Brownfield Scan Trace — Initialization (ENH-081)

**Before running any signal category**, check for an existing trace:

1. Check if `.viepilot/BROWNFIELD-TRACE.md` exists.

2. **If exists AND `Status: scan_complete`:** offer resume via AUQ —
   - question: "Prior scan found (completed {date}). Re-scan or resume from gap-filling?"
   - header: "Brownfield Trace"
   - options:
     a. "Resume — skip re-scan, go to gap-filling (Recommended)"
     b. "Re-scan from scratch — overwrite trace"

3. **If exists AND `Status: scanning` (interrupted):** read which category was in-progress.
   Offer via AUQ —
   - question: "Prior scan was interrupted at Signal Cat {N}. How to proceed?"
   - header: "Interrupted Scan"
   - options:
     a. "Resume from Signal Cat {N} (Recommended)"
     b. "Re-scan from scratch"

4. **If not exists (or user chose "Re-scan"):** create `.viepilot/BROWNFIELD-TRACE.md`:

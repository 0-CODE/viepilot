# Task 107.3 — Add <version_check> block to all vp-* SKILL.md files

## Objective
Insert a `<version_check>` block immediately after the `<greeting>` block in every
`skills/vp-*/SKILL.md` file. The block instructs the skill to run `vp-tools check-update
--silent` after displaying the greeting banner, and show a notice banner if exit code = 1.

## File-Level Plan

In each SKILL.md, locate the closing `</greeting>` tag and insert the following block
**immediately after** it (before any other section):

```markdown
<version_check>
## Version Update Check (ENH-072)

After displaying the greeting banner, run:
```bash
node "$HOME/.claude/viepilot/bin/vp-tools.cjs" check-update --silent
```

**If exit code = 1** (update available — new version printed to stdout):
Display notice banner before any other output:
```
┌──────────────────────────────────────────────────────────────────┐
│ ✨ ViePilot {latest_version} available  (installed: {current})   │
│    npm i -g viepilot && vp-tools install --target {adapter_id}   │
└──────────────────────────────────────────────────────────────────┘
```
Replace `{latest_version}` with stdout from the command, `{current}` with the installed
version, `{adapter_id}` with the active adapter (claude-code / cursor / antigravity / codex / copilot).

**If exit code = 0 or command unavailable**: silent, continue.

**Suppression rules:**
- `--no-update-check` flag on skill invocation → skip this step entirely
- `config.json` → `update.check: false` → skip this step entirely
- Show at most once per session (`update_check_done` session guard)
</version_check>
```

**Note on `{envToolDir}`**: In installed skills the path resolves to the adapter's tool dir.
For SKILL.md source files (pre-install), use `$HOME/.claude/viepilot` as the canonical example;
the installer rewrites this to `{envToolDir}` on install.

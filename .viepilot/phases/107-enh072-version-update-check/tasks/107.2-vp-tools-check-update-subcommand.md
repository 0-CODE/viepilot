# Task 107.2 — vp-tools check-update subcommand

## Objective
Add a `check-update` subcommand to `bin/vp-tools.cjs`:

```
vp-tools check-update [--silent] [--json] [--force]

Options:
  --silent    Exit 0 if up-to-date (no output). Exit 1 + print latest version if update available.
  --json      Print JSON: { installed, latest, has_update }
  --force     Bypass 24h cache and check now

Exit codes:
  0  — up to date (or check failed silently)
  1  — update available (latest version printed to stdout)
```

## Paths
bin/vp-tools.cjs

## File-Level Plan

### `bin/vp-tools.cjs`
1. In the command dispatch block (`switch (cmd)` or `if (cmd === ...)` pattern), add case `'check-update'`
2. Parse flags from `process.argv`: `--silent`, `--json`, `--force`
3. Call `checkLatestVersion({ force })` from `lib/viepilot-update.cjs`
4. Output logic:
   - `--json`: `console.log(JSON.stringify({ installed, latest, has_update: !upToDate }))` → exit 0
   - `--silent` + upToDate: no output → exit 0
   - `--silent` + !upToDate: `console.log(latest)` → exit 1
   - default (no flags) + upToDate: print "ViePilot {installed} is up to date." → exit 0
   - default (no flags) + !upToDate: print notice banner → exit 1
5. Wrap in async IIFE or use `.then()/.catch()` — all errors → exit 0 (silent)
6. Add to `--help` / `list-commands` output: `check-update   Check for latest ViePilot version`

## Best Practices
- Keep all error paths exiting 0 (never crash skill invocations that call this)
- `process.exit()` only after async resolution — do not call synchronously before promise resolves

## Verification
```bash
node bin/vp-tools.cjs check-update --json
node bin/vp-tools.cjs check-update --silent; echo "exit: $?"
node bin/vp-tools.cjs check-update --force --json
```

## Acceptance Criteria
- [ ] `check-update` case handled in bin/vp-tools.cjs dispatch
- [ ] `--silent` exits 1 + prints latest when update available
- [ ] `--silent` exits 0, no output when up to date
- [ ] `--json` always exits 0, prints JSON result
- [ ] `--force` bypasses cache (passed to `checkLatestVersion`)
- [ ] Listed in help/list-commands output
- [ ] Any internal error → exit 0 (silent)

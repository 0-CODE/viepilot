# Task 107.1 — checkLatestVersion() + 24h cache

## Objective
Extend `lib/viepilot-update.cjs` with an exported `checkLatestVersion()` function that:
1. Reads installed version from `package.json` in the viepilot install root
2. Checks `~/.viepilot/update-cache.json` — if `checked_at` is within 24h, return cached result without network call
3. Fetches latest version from `https://registry.npmjs.org/viepilot/latest` with 3s timeout
4. On any failure (network, timeout, parse error, unreadable cache) → return `{ upToDate: true }` silently
5. Writes cache: `{ checked_at, installed, latest, has_update }`
6. Returns `{ upToDate: boolean, installed: string, latest: string }`

## Paths
lib/viepilot-update.cjs

## File-Level Plan

### `lib/viepilot-update.cjs`
- Add `checkLatestVersion(opts = {})` async function below existing exports
- `opts.force` — bypass 24h cache check
- `opts.cacheFile` — override cache path (default: `~/.viepilot/update-cache.json`; used for tests)
- Read installed version via `require()` of `package.json` at viepilot root (use `viepilot-info.cjs` pattern for path resolution)
- Cache TTL: `Date.now() - new Date(cache.checked_at).getTime() < 24 * 60 * 60 * 1000`
- HTTP fetch: use Node.js built-in `https.get()` (no external deps) — collect response body, parse JSON, extract `.version`
- semver compare: `latest !== installed` is sufficient (no semver lib needed — both are well-formed from npm registry)
- Export: `module.exports = { ...(existing exports), checkLatestVersion }`

## Best Practices
- No new runtime dependencies (use Node built-ins only: `https`, `fs`, `path`, `os`)
- All errors caught in try/catch → return `{ upToDate: true }` (never throw)
- Cache file directory auto-created if missing (`fs.mkdirSync(..., { recursive: true })`)
- Timeout implemented via `setTimeout` + `req.destroy()` pattern

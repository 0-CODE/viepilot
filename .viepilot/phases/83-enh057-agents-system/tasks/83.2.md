# Task 83.2 — Wire tracker-agent + changelog-agent into `autonomous.md`

## Objective
Update `viepilot/workflows/autonomous.md` to delegate two previously inline operations
to dedicated agents: TRACKER status updates (tracker-agent) and version bump + CHANGELOG
writes (changelog-agent). This resolves ENH-053 (version bump conflict) naturally.

## Paths
- `viepilot/workflows/autonomous.md`

## Changes Required

### 1. Tracker update delegation
Find all inline "Update TRACKER.md" instructions in autonomous.md and replace with an
`<invoke-agent>` pattern:

```markdown
**Invoke: tracker-agent**
- Operation: `update-phase-status`
- Phase: {phase_number}
- Status: in_progress | complete
- Task: {task_id}
```

Target locations in autonomous.md:
- Step where phase starts (set Current Phase → in_progress)
- Step where task completes (update Current Task)
- Step where phase completes (set to complete, clear Current Phase)

### 2. Changelog + version bump delegation
Find the post-phase version bump block (the one ENH-053 flagged as conflicting with
evolve.md's version bump logic). Replace with:

```markdown
**Invoke: changelog-agent**
- Version: {next_version}
- Entry: {phase summary bullets}
- Files to bump: `package.json`, `CHANGELOG.md`, `viepilot/templates/project/VIEPILOT-META.md`
```

Add a clear note: "Version bump authority lives in changelog-agent. evolve.md and
autonomous.md both invoke it — no duplication."

### 3. Add `<agents>` reference block
Add a new section near the top of autonomous.md (after the context/intro section):

```markdown
## Agent Delegation

This workflow delegates heavy/repetitive operations to sub-agents:

| Agent | When invoked | Operation |
|-------|-------------|-----------|
| tracker-agent | Task start, task complete, phase complete | Status updates |
| changelog-agent | Post-phase (last task) | CHANGELOG + version bump |
| test-generator-agent | Last task of each phase (contract tests) | Test file generation |
| doc-sync-agent | Bulk SKILL.md updates (adapter rows, banners) | Multi-file edits |

See `viepilot/agents/` for full agent definitions.
```

# Task 83.1 — Create 6 agent `.md` files in `viepilot/agents/`

## Objective
Create the `viepilot/agents/` directory and write 6 agent definition files. Each file
describes a specialized sub-agent that skills can invoke to offload repetitive, heavy,
or parallelizable work.

## Paths
- `viepilot/agents/tracker-agent.md`
- `viepilot/agents/research-agent.md`
- `viepilot/agents/file-scanner-agent.md`
- `viepilot/agents/changelog-agent.md`
- `viepilot/agents/test-generator-agent.md`
- `viepilot/agents/doc-sync-agent.md`

## Agent File Format
Each agent `.md` must contain these sections:

```markdown
# {Agent Name}

## Purpose
One paragraph: what this agent does and why it exists as a dedicated agent.

## Inputs
Bullet list of what the caller must provide.

## Outputs
Bullet list of what the agent produces/returns.

## Invocation Pattern

### Claude Code (terminal)
```
Agent({
  subagent_type: "general-purpose" | "Explore",
  prompt: "...",
  description: "..."
})
```

### Cursor / Codex / Antigravity
Sub-prompt fallback: describe in plain text what the agent should do as a scoped instruction.

## Adapter Behavior
| Adapter | Behavior |
|---------|----------|
| Claude Code | Spawns real subagent via Agent tool |
| Cursor | Scoped sub-prompt in same session |
| Codex | Scoped sub-prompt in same session |
| Antigravity | Scoped sub-prompt in same session |

## Notes
Any constraints, edge cases, or important implementation details.
```

## Per-Agent Specifications

### tracker-agent
- Reads/writes `.viepilot/TRACKER.md`
- Operations: update Current Phase, add Decision Log row, flip request status in Pending table
- Input: operation type + data (phase number, request ID, status, etc.)
- Output: confirmation of changes made

### research-agent
- Performs WebSearch + WebFetch + summarizes for feasibility/tech research
- Input: research topic, specific questions, optional domain filters
- Output: structured summary with findings + sources list
- Claude Code: `subagent_type: "general-purpose"` with web search tools
- Other adapters: inline web search if available, else text instruction to user

### file-scanner-agent
- Runs Glob + Grep patterns across repo to find affected files, detect stale refs
- Input: patterns list, keywords, repo root path
- Output: matched file list + relevant line excerpts
- Claude Code: `subagent_type: "Explore"` (specialized for codebase scanning)

### changelog-agent
- Reads `CHANGELOG.md`, prepends new `[X.Y.Z]` entry, bumps `package.json` version
- Also updates `viepilot/templates/project/VIEPILOT-META.md` version field
- Input: version string, entry lines (feature/fix bullets)
- Output: confirmation of CHANGELOG + package.json + VIEPILOT-META updates

### test-generator-agent
- Generates contract test file for a given phase/feature, runs suite, reports
- Input: phase number, feature ID, list of acceptance criteria to test
- Output: test file path + test count + pass/fail status
- Claude Code: `subagent_type: "general-purpose"` with write + bash access

### doc-sync-agent
- Bulk-updates multiple `.md` files with the same change (e.g., add table row to all SKILL.md)
- Input: target file glob pattern, change description (old string → new string or insert position)
- Output: list of files updated + diff summary
- Claude Code: `subagent_type: "general-purpose"` with edit access

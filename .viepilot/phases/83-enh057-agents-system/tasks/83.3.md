# Task 83.3 — Wire research-agent into `request.md` feasibility gate

## Objective
Add a research-agent gate to `viepilot/workflows/request.md` so that when a request
involves technology feasibility (new adapter, new integration, new platform), the
research-agent is automatically invoked to gather findings before the request file
is written. This makes the FEAT-019 research pattern (done manually) the standard workflow.

## Paths
- `viepilot/workflows/request.md`

## Changes Required

### 1. Add feasibility detection step (after Step 1 — Detect Request Type)
Insert a new Step 1B:

```markdown
### Step 1B: Feasibility Gate (auto-triggered for Feature + Enhancement requests)

If the request involves:
- A new adapter, platform, or IDE integration
- A new external tool, API, or protocol
- Technical unknown ("is this possible?", "how does X work?")

→ **Invoke: research-agent**
- Topic: {extracted from user description}
- Questions: ["What is the API/SDK?", "What are integration points?", "What is the feasibility?"]
- Output: append findings to request `.md` under `## Research Findings` section

For Bug reports and Enhancement requests on known code: skip this step.
```

### 2. Research findings section in request template
Update the request file template (Step 3) to include an optional `## Research Findings`
section between `## Summary` and `## Details`:

```markdown
## Research Findings
{Populated by research-agent if feasibility gate triggered; omit if not applicable}
```

### 3. Update Step 1 type detection note
Add a note to Feature Request type:
> "If this feature requires external platform research, research-agent will run automatically before gathering details."

## Acceptance Criteria
- [ ] `request.md` has Step 1B with feasibility gate logic
- [ ] Gate triggers automatically for Feature + new-platform Enhancement requests
- [ ] Gate skipped for Bug reports and internal Enhancement requests
- [ ] Request file template includes `## Research Findings` optional section
- [ ] Invocation pattern for research-agent documented (with Claude Code Agent tool + text fallback)

# Task 148.4 — agents/qa-templates/{codex,cursor,antigravity,copilot}/ — non-Claude-Code templates

## Objective
Create single-file QA agent templates for Codex, Cursor, Antigravity, and Copilot adapters.
These adapters don't support multi-agent fan-out, so all scan domains are packed into one
sequential scanning prompt.

## Paths
- `agents/qa-templates/codex/qa-agent.md.tmpl`
- `agents/qa-templates/cursor/qa-checklist.mdc.tmpl`
- `agents/qa-templates/antigravity/qa-skill.md.tmpl`
- `agents/qa-templates/copilot/qa-agent.md.tmpl`

## File-Level Plan

### codex/qa-agent.md.tmpl

Appended as a section to `AGENTS.md` in the target project. Codex reads `AGENTS.md` as
its system instructions. Uses `apply_patch` conventions.

```markdown
## QA Agent Instructions — {{PROJECT_NAME}}

Stack: {{STACK}}
Backend directories: {{BACKEND_DIRS}}

### Mission
Perform a sequential backend QA scan covering: {{DOMAINS_BLOCK}}

### Scan procedure

For each domain below, use `container.exec` to run grep/find commands. Report all findings
as a structured list before proceeding to the next domain.

#### 1. Completeness
- grep -rn "TODO\|FIXME\|PLACEHOLDER\|not implemented" {{BACKEND_DIRS}}
- Look for empty function bodies, stub returns

#### 2. Security
- grep -rn "eval(\|Math\.random()\|md5(\|sha1(" {{BACKEND_DIRS}}
- Look for hardcoded secrets: grep -rn "password\s*=\s*['\"]" {{BACKEND_DIRS}}
- Look for missing auth middleware on CRUD routes

#### 3. Performance
- Look for DB calls inside loops (N+1 pattern)
- Look for missing pagination (.find() / .findAll() with no LIMIT)
- Look for synchronous I/O in async handlers

#### 4. Context
- Look for files with no corresponding .test. file
- Look for exported symbols with zero references

**Stack-specific rules:**
{{STACK_RULES}}

### Output
After scanning all domains, output a flat list:
```
[SEVERITY] [DOMAIN] file:line — description
```

### Creating vp-request entries
For each issue, use apply_patch to create `.viepilot/requests/BUG-{N}.md` with:
- ID, Severity, Source: qa-agent (automated), Files, Summary
```

### cursor/qa-checklist.mdc.tmpl

MDC rule file applied automatically when working in Cursor.

```
---
description: QA checklist for {{PROJECT_NAME}} — auto-applied during backend edits
globs: ["{{BACKEND_DIRS_GLOB}}"]
alwaysApply: false
---

# QA Checklist — {{PROJECT_NAME}} ({{STACK}})

When editing backend files, verify the following before completing any task:

## Completeness
- [ ] No TODO/FIXME left in function bodies
- [ ] No stub returns (`return null`, `return []`) without implementation
- [ ] No empty handler bodies

## Security
- [ ] No hardcoded secrets or API keys
- [ ] Auth middleware present on sensitive routes
- [ ] User input validated before DB operations
- [ ] No `eval()` or `Math.random()` for security purposes

## Performance
- [ ] No DB calls inside loops
- [ ] List endpoints have pagination
- [ ] No synchronous I/O in async handlers

## Context
- [ ] Exported function has at least one reference
- [ ] New env var added to .env.example

**Stack-specific rules:**
{{STACK_RULES}}

To run a full scan: invoke `qa-orchestrator` agent (if available) or use `/vp-qa --run`.
```

name: qa-orchestrator

description: Sequential backend QA scan for {{PROJECT_NAME}}
---

## Objective
Perform a sequential backend QA scan covering completeness, security, performance,
and context domains. Report findings and create vp-request entries.

## Stack
{{STACK}}

## Backend directories
{{BACKEND_DIRS}}

## Process

### Step 1: Completeness scan
Use shell to grep for stubs and incomplete code.

### Step 2: Security scan
Use shell to grep for known vulnerability patterns.

### Step 3: Performance scan
Use shell to identify N+1 patterns and missing pagination.

### Step 4: Context audit
Use shell to find dead code and missing test files.

**Stack-specific rules:**
{{STACK_RULES}}

For each issue found, create `.viepilot/requests/BUG-{N}.md` using file_write.

For each domain, use `runCommands` to grep/find patterns in `{{BACKEND_DIRS}}`.

**Stack-specific rules:**
{{STACK_RULES}}

For each issue found, create a file at `.viepilot/requests/BUG-{N}.md` using `editFiles`:
```markdown

- Source: qa-orchestrator (automated)
- Files: {FILES}
## Summary
{DESCRIPTION}
```

After all domains scanned, summarize: total issues by severity, files created.
```

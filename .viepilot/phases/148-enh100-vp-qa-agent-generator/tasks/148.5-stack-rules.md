## Objective

`{{STACK_RULES}}`. Each file contains extra patterns specific to that stack's ecosystem

## File-Level Plan

### rules/node.md

```markdown
### Node.js / JavaScript Extras

**Completeness:**
- Express/Fastify/NestJS route handlers with `res.send()` or `res.json()` missing (no response sent)
- `async` functions that never `await` anything (forgot async)
- `next()` called without error handling in Express middleware chains
- Missing `module.exports` on files that should export

**Security:**
- `eval()` / `new Function()` with any variable input
- Prototype pollution: `obj[userKey] = userVal` without key sanitization
- `child_process.exec()` with unsanitized user input (shell injection)
- JWT verified with `algorithms: ['none']` or no algorithm check
- `res.setHeader('Access-Control-Allow-Origin', req.headers.origin)` without allow-list
- Missing `helmet()` middleware
- `serialize-javascript` or `JSON.stringify` of user data directly into HTML

**Performance:**
- `fs.readFileSync` / `fs.writeFileSync` inside route handlers
- `require()` inside a function (synchronous, blocks event loop on first call)
- `Promise.all` not used when multiple independent async calls exist
- Missing `compression()` middleware for large responses
- `setInterval` without `clearInterval` (memory leak)
- Large array `.filter().map().reduce()` chains on every request (should be precomputed or cached)

**Context:**
- `console.log` left in production code (use logger)
- `process.env.X` without `|| 'default'` and without validation
- `__dirname` + string concatenation instead of `path.join()`
```

### rules/python.md

```markdown
### Python Extras

**Completeness:**
- `pass` as the only statement in a view/handler function
- `raise NotImplementedError` without override in concrete class
- Django views returning `HttpResponse('')` with no content
- FastAPI endpoints with `return {}` stub

**Security:**
- Server-side template injection: `render_template_string(user_input)`
- `pickle.loads(user_data)` â€” arbitrary code execution
- `subprocess.call(user_input, shell=True)` â€” shell injection
- Django: `raw()` queries with f-string interpolation
- SQLAlchemy: `text(f"SELECT ... {user_val}")` â€” SQL injection
- `DEBUG = True` in settings (must be env-var controlled)
- `SECRET_KEY` hardcoded in settings.py
- Missing `@login_required` / `@permission_required` on sensitive views

**Performance:**
- Django ORM: `for obj in QuerySet` without `select_related`/`prefetch_related`
- Calling `.count()` then iterating (two queries instead of one)
- Missing `.only()` or `.defer()` on large model queries
- Sync view calling `requests.get()` (blocking I/O â€” use async view + httpx)
- Missing `cache_page` decorator on expensive read-only views

**Context:**
- `import *` from modules
- Unused imports (check with basic grep)
- `print()` left in production code (use logging)
- Missing `__all__` on public API modules
```

### rules/java.md

```markdown
### Java / Spring Boot Extras

**Completeness:**
- `@Override` methods with empty body or just `super.method()`
- `@NotImplemented` annotation on production service methods
- `TODO: implement` in `@Service` or `@Repository` methods
- Spring `@RestController` methods returning `null` or empty `ResponseEntity`

**Security:**
- OGNL injection: `${...}` in Thymeleaf templates from user input
- XXE: `DocumentBuilderFactory.newInstance()` without disabling external entities
- SQL injection: `entityManager.createNativeQuery("... " + userParam)`
- Insecure deserialization: Java serialization with untrusted input (`ObjectInputStream`)
- Missing `@PreAuthorize` / `@Secured` on sensitive service methods
- `BCryptPasswordEncoder` strength < 10
- Hardcoded credentials in `application.properties` (not using `${ENV_VAR}`)

**Performance:**
- Hibernate N+1: `@OneToMany` without `fetch = FetchType.LAZY` + `@EntityGraph`
- Missing `@Async` on long-running service methods called from controller
- `@Transactional` on read-only methods without `readOnly = true`
- No `@Cacheable` on expensive, idempotent service methods
- Connection pool not configured (`spring.datasource.hikari.*` missing)

**Context:**
- `System.out.println()` in production code (use SLF4J logger)
- Unchecked exceptions swallowed: `catch (Exception e) {}`
- Missing `@Slf4j` / logger declaration in classes that log
```

### rules/go.md

```markdown
### Go Extras

**Completeness:**
- Interface methods implemented with `panic("not implemented")`
- Handler functions returning without writing to `http.ResponseWriter`
- `TODO:` comments in handler or service functions

**Security:**
- `fmt.Sprintf` for SQL queries (SQL injection)
- `html/template` used with `template.HTML(userInput)` (bypasses escaping)
- `exec.Command` with user-controlled arguments without sanitization
- Missing context cancellation propagation (`ctx context.Context` not passed)
- Goroutine started without context â†’ runs forever if parent dies
- JWT: `jwt.ParseWithClaims` without verifying `alg` claim

**Performance:**
- Goroutine leak: goroutine started in request handler without `ctx.Done()` check
- Unbuffered channel used for high-frequency operations
- `sync.Mutex` on hot path â€” consider `sync.RWMutex` for read-heavy ops
- Missing connection reuse: `http.Client` created per request instead of package-level var
- `database/sql`: `rows.Close()` not deferred (connection leak)
- Missing `GOMAXPROCS` configuration for CPU-bound workloads

**Context:**
- `_` assigned for errors from important operations
- Missing `defer cancel()` after `context.WithTimeout`
- `log.Fatal` / `os.Exit` called outside of `main()` (prevents cleanup)
```

### rules/ruby.md

```markdown
### Ruby / Rails Extras

**Completeness:**
- `raise NotImplementedError` in service objects
- Rails controller actions with empty body or `render nothing: true`
- Missing `private` method implementations called by public interface

**Security:**
- `eval(user_input)` â€” arbitrary code execution
- `send(user_method_name)` without allow-list
- `params.permit!` (permits all â€” use strong parameters)
- SQL injection: `Model.where("name = '#{params[:name]}'")`
- Missing `before_action :authenticate_user!` on sensitive controllers
- `Marshal.load(user_data)` â€” deserialization vulnerability
- Hardcoded `secret_key_base` in `secrets.yml`

**Performance:**
- N+1: `@posts.each { |p| p.author.name }` without `includes(:author)`
- Missing `.select()` on queries that only need a subset of columns
- Callbacks (`after_save`, `after_create`) doing heavy work synchronously

**Context:**
- `puts` / `p` left in production code (use Rails logger)
- `rescue Exception` (catches too broadly â€” use `StandardError`)
```

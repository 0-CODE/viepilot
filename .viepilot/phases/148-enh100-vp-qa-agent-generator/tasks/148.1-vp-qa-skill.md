**Objective:**
```
Generate a project-level QA agent team tailored to this codebase.

Phase 1 — Research: Scan codebase to understand stack, structure, and patterns.
Phase 2 — Generate: Create QA agents in the correct format for the active adapter.
Phase 3 — Interact: Ask user to run scan now or defer.

Invocation:
  /vp-qa                → auto-detect adapter + stack, generate QA team
  /vp-qa --run          → generate + immediately invoke qa-orchestrator
  /vp-qa --focus sec    → bias research toward security domains
  /vp-qa --focus perf   → bias research toward performance domains
  /vp-qa --target <id>  → override adapter detection (claude-code/cursor/codex/antigravity/copilot)
```

**Process (detailed):**

```
### Phase 1: Research (REQUIRED before any generation)

1. Read .viepilot/PROJECT-META.md, STACKS.md, PROJECT-CONTEXT.md (if present)
2. Detect stack:
   - package.json → Node.js (detect: express/fastify/nestjs/koa)
   - pom.xml → Java (detect: spring-boot/quarkus)
   - go.mod → Go
   - requirements.txt / pyproject.toml → Python (detect: django/fastapi/flask)
   - Gemfile → Ruby (detect: rails/sinatra)
3. Read directory tree: find src/, app/, lib/, services/, api/, routes/
4. Sample 5-10 representative source files from detected backend dirs
5. Estimate project size: file count, service count, DB layer presence
6. Check existing .viepilot/requests/ to avoid duplicate issue reporting
7. Build research summary: {stack, version, patterns, dirs, size, known_issues[]}

### Phase 2: Generate agents

Based on research summary:
- Decide which scan domains are relevant
- Decide how many specialist agents (2 for small/simple, 4-5 for large/complex)
- Generate agent files using lib/qa-generator.cjs

Claude Code output: .claude/agents/qa-orchestrator.md + subagent files
Cursor output: .cursor/rules/qa-checklist.mdc
Codex output: AGENTS.md section appended
Antigravity output: .agents/skills/qa-orchestrator/SKILL.md
Copilot output: .github/agents/qa-orchestrator.agent.md

### Phase 3: AskUserQuestion (Claude Code only)

After generation:
- Show summary: "Generated N agents in .claude/agents/"
- AskUserQuestion:
    "QA team ready. What next?"
    1. "Run scan now" → invoke qa-orchestrator agent
    2. "Done for now" → exit

Text fallback: print agent paths + "Invoke qa-orchestrator to start scan."
```

**Generated agent behavior (qa-orchestrator, when invoked in target project):**
```
When user invokes qa-orchestrator in their project:
1. Verify it's still relevant (check if project changed since generation)
2. Fan out to specialist subagents (Claude Code) or run sequentially (other adapters)
3. Collect issue lists
4. Group by severity
5. For critical/high: AskUserQuestion → accept (create vp-request) or decline
6. For medium/low: batch AskUserQuestion → create all or skip all
7. Final AskUserQuestion: "N issues logged → /vp-evolve to plan fixes?"
```

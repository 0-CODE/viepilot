# Task 27.1 — CLI `vp-tools info` (+ `--json`)

## Objective
Cung cấp một lệnh ổn định để in (hoặc xuất JSON) phiên bản ViePilot đang dùng, bản mới nhất trên npm (khi có mạng), và inventory skills/workflows kèm version policy.

## Paths
- `lib/viepilot-info.cjs` (new) — pure helpers: resolve viepilot package root, read `package.json`, parse skill frontmatter `version`, list workflow files, optional `npm view viepilot version`
- `bin/vp-tools.cjs` — register commands `info` + help text + main list
- `tests/unit/viepilot-info.test.js` (stub in 27.5 or minimal here — prefer 27.5 for full tests; 27.1 may add 1–2 smoke tests)

## File-Level Plan
- **`lib/viepilot-info.cjs`**
  - `resolveViepilotPackageRoot()` — từ `__dirname` của vp-tools đi lên tìm `package.json` có `name === 'viepilot'`; fallback cwd.
  - `readInstalledVersion(root)` — `version` field.
  - `fetchLatestNpmVersion()` — `child_process.execSync('npm view viepilot version --json')` với timeout ngắn; catch → `{ ok: false, error }`.
  - `listSkillsWithVersions(root)` — `skills/*/SKILL.md`, parse `^version:\s*` từ frontmatter; missing → `"0.0.0"` hoặc `"unspecified"`.
  - `listWorkflows(root)` — `workflows/*.md` sorted; mỗi item `{ id, relativePath, semverInFile: null, note: 'no semver in workflow markdown' }`.
  - `buildInfoReport(root, options)` — object thống nhất cho CLI `--json`.
- **`bin/vp-tools.cjs`**
  - `info` subcommand: flags `--json`; pretty-print table khi TTY.

## Best practices
- Không `shell: true` với input user; chỉ `npm view` cố định.
- English comments in lib; user-facing CLI messages có thể EN (match existing vp-tools).

## Verification
- `node bin/vp-tools.cjs info` — exits 0 trong repo viepilot; hiển thị version khớp `package.json`.
- `node bin/vp-tools.cjs info --json` — valid JSON, keys: `installedVersion`, `latestNpm`, `skills`, `workflows`, `packageRoot`.
- `npm test` — pass (nếu có test mới trong 27.1 hoặc chờ 27.5).

# Task 90.1 — Create `lib/skill-registry.cjs` (scanner + registry builder)

## Objective
Implement the core skill scanner: traverse all adapter `skillsDir` paths, parse SKILL.md files (extended format + legacy fallback), and build/write `~/.viepilot/skill-registry.json`.

## Paths
- `lib/skill-registry.cjs`

## Implementation Notes

### Exported API

```js
// Scan all adapter skillsDirs and write ~/.viepilot/skill-registry.json
// Returns: { skills[], scan_paths[], last_scan }
function scanSkills(home = os.homedir()) { ... }

// Read and return the current registry (or null if not yet written)
function loadRegistry(home = os.homedir()) { ... }
```

### Scanner logic

1. Call `listAdapters()` from `lib/adapters/index.cjs`
2. For each adapter: resolve `adapter.skillsDir(home)` → check if dir exists
3. For each subdirectory inside skillsDir: look for `SKILL.md`
4. Parse SKILL.md:
   - **Extended format**: extract `## Capabilities`, `## Tags`, `## Best Practices` sections
   - **Legacy fallback**: if those sections are absent, extract first paragraph of description
5. Build skill entry:

```json
{
  "id": "frontend-design",
  "name": "Frontend Design",
  "source": null,
  "version": null,
  "description": "...",
  "capabilities": ["ui-generation", "component-design"],
  "tags": ["ui", "design", "frontend"],
  "best_practices": ["Mobile-first", "Design tokens"],
  "adapters": ["claude-code"],
  "installed_paths": {
    "claude-code": "/Users/user/.claude/skills/frontend-design/"
  }
}
```

6. Deduplicate by `id` — if same skill found in multiple adapter dirs, merge `adapters[]` and `installed_paths{}`
7. Write registry to `~/.viepilot/skill-registry.json`:

```json
{
  "version": "1.0",
  "last_scan": "<ISO datetime>",
  "scan_paths": ["~/.claude/skills/", "~/.cursor/skills/"],
  "skills": [ ... ]
}
```

### Section parsing rules

```
## Capabilities
- ui-generation         → capabilities: ["ui-generation"]
- component-design      → capabilities: [..., "component-design"]

## Tags
ui, design, frontend    → tags: ["ui", "design", "frontend"]
(comma or newline separated)

## Best Practices
- Mobile-first          → best_practices: ["Mobile-first"]
- Design tokens         → best_practices: [..., "Design tokens"]
```

If a section is missing: use `[]` for that field (never throw, never skip the skill).

### Edge cases
- `skillsDir` doesn't exist → skip silently (adapter not installed)
- Subdirectory has no SKILL.md → skip
- SKILL.md is empty → skill with empty fields (still indexed)
- `~/.viepilot/` doesn't exist → create it before writing registry

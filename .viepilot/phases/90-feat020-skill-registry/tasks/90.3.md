## Content Outline

```markdown
# Skill Registry

## Overview
ViePilot can discover and index skills installed across all adapter directories.
Run `vp-tools scan-skills` to build the registry.

## Extended SKILL.md Format

Third-party skills (and built-in vp-* skills) can add optional metadata sections:

    ## Capabilities
    - ui-generation
    - component-design
    - responsive-layout

    ## Tags
    ui, design, frontend, css, html

    ## Best Practices
    - Mobile-first: design breakpoints from 320px up
    - Use design tokens for consistency

### Section rules
- `## Capabilities` — one capability per line (prefixed with `-`)
- `## Tags` — comma-separated or one per line
- `## Best Practices` — one practice per line (prefixed with `-`)
- All sections are optional — missing sections default to `[]`
- Existing SKILL.md files without these sections continue to work

## Scanner

`vp-tools scan-skills` traverses:
- `~/.claude/skills/`  (Claude Code)
- `~/.cursor/skills/`  (Cursor)
- `~/.codex/skills/`   (Codex)
- `~/.antigravity/skills/` (Antigravity)
- `~/.copilot/skills/`  (GitHub Copilot)

Adapter dirs that don't exist are silently skipped.

## Commands

| Command | Description |
|---------|-------------|
| `vp-tools scan-skills` | Scan all adapter dirs + write registry |
| `vp-tools list-skills` | Display indexed skills from registry |
| `vp-tools install-skill <source>` | Install + auto-scan *(Phase 91)* |
```

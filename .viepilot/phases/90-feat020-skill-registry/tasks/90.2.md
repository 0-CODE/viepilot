# Task 90.2 — Add `scan-skills` + `list-skills` to `bin/viepilot.cjs`

## Objective
Expose `vp-tools scan-skills` and `vp-tools list-skills` as CLI subcommands.

## Paths
- `bin/viepilot.cjs`

## Implementation Notes

### Commands to add

#### `vp-tools scan-skills`
```
$ vp-tools scan-skills
Scanning skill directories...
  ✓ ~/.claude/skills/  — 3 skills found
  ✓ ~/.cursor/skills/  — 2 skills found
  - ~/.codex/skills/   — not installed (skipped)
  - ~/.antigravity/skills/ — not installed (skipped)
  - ~/.copilot/skills/ — not installed (skipped)

Registry written: ~/.viepilot/skill-registry.json
Total: 4 unique skills (1 in multiple adapters)
```

#### `vp-tools list-skills`
```
$ vp-tools list-skills

Installed skills (from ~/.viepilot/skill-registry.json):
Last scan: 2026-04-20T10:30:00Z

  ID                   Adapters              Capabilities
  ───────────────────  ────────────────────  ──────────────────────
  frontend-design      claude-code, cursor   ui-generation, component-design
  vp-auto              claude-code           (legacy — no capabilities)
  ...

Run `vp-tools scan-skills` to refresh.
```

If registry doesn't exist yet: `No registry found. Run vp-tools scan-skills first.`

### CLI wiring (follow existing pattern in bin/viepilot.cjs)

Find the `switch (command)` block and add:

```js
case 'scan-skills': {
  const { scanSkills } = require('../lib/skill-registry.cjs');
  const result = scanSkills();
  // print per-adapter summary + total
  break;
}
case 'list-skills': {
  const { loadRegistry } = require('../lib/skill-registry.cjs');
  const registry = loadRegistry();
  if (!registry) { console.log('No registry found. Run vp-tools scan-skills first.'); break; }
  // print table
  break;
}
```

Also add to `printHelp()`:
```
  scan-skills              Scan all adapter skill dirs and build skill registry
  list-skills              List all indexed skills from registry
```

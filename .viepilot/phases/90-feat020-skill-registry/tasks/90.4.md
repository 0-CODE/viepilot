### Test file: `tests/unit/vp-skill-registry.test.js`

Contract tests to cover (≥15):

```
Scanner — basic
  1. scanSkills() returns object with { version, last_scan, scan_paths, skills }
  2. scanSkills() calls listAdapters() to resolve skill dirs
  3. scanSkills() skips adapter dirs that don't exist (no crash)
  4. scanSkills() returns skills array (empty is valid)

Extended format parsing
  5. SKILL.md with ## Capabilities → skill.capabilities is array of strings
  6. SKILL.md with ## Tags (comma-separated) → skill.tags is array
  7. SKILL.md with ## Tags (one-per-line) → skill.tags is array
  8. SKILL.md with ## Best Practices → skill.best_practices is array
  9. All three sections present → all fields populated

Legacy format parsing
  10. SKILL.md without any extended sections → skill indexed with capabilities=[], tags=[], best_practices=[]
  11. Empty SKILL.md → skill indexed (no throw)

Deduplication
  12. Same skill id in two adapter dirs → single entry with both in adapters[]
  13. Same skill id in two adapter dirs → installed_paths has entries for both adapters

Registry persistence
  14. scanSkills() writes JSON to ~/.viepilot/skill-registry.json
  15. loadRegistry() reads and returns the written registry
  16. loadRegistry() returns null if file doesn't exist

CLI integration
  17. bin/viepilot.cjs --help output includes "scan-skills"
  18. bin/viepilot.cjs --help output includes "list-skills"
```

Use sinon/jest mocks for filesystem where needed to avoid side effects.

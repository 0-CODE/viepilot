## File-Level Plan
- `workflows/autonomous.md` control_point state: On entering control point → set HANDOFF.json `control_point.active = true`, `control_point.reason = "{reason}"`, `control_point.ts = now()`. Display message to user. Wait for user input. On user resolve → clear control_point fields → continue. On user skip → set task skip.

## Protocol Detail
```
Enter control_point:
  1. Update HANDOFF.json:
     control_point.active = true
     control_point.reason = "{why: budget exhausted / scope drift / other}"
     control_point.ts = ISO8601 timestamp
  2. Write HANDOFF.json
  3. Display to user:
     "━━━ CONTROL POINT ━━━
     Reason: {reason}
     Task: {task_id}
     Options: [c]ontinue / [s]kip / [r]etry"

Exit control_point (user resolves):
  4. Clear: control_point.active = false, reason = null, ts = null
  5. Write HANDOFF.json
  6. Continue per user choice
```

## Acceptance Criteria
- [ ] HANDOFF.json.control_point.active set to true on enter
- [ ] reason and ts fields populated on enter
- [ ] All 3 fields cleared on exit
- [ ] HANDOFF.json written before displaying to user (so vp-status can detect)
- [ ] User options displayed: continue / skip / retry
- [ ] Control point display format uses consistent separator `━━━`

## Do / Don't

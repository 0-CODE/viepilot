## Budget Logic
```
Parse budget from TASK.md (default M if missing):
S:  {l1_max: 1, l2_max: 1, l3_max: 0}
M:  {l1_max: 1, l2_max: 2, l3_max: 0}
L:  {l1_max: 2, l2_max: 2, l3_max: 1}
XL: {l1_max: 2, l2_max: 3, l3_max: 1}

Apply recovery_overrides:
if recovery_overrides.L1.block == true: l1_max = 0
if recovery_overrides.L2.block == true: l2_max = 0
if recovery_overrides.L3.block == true: l3_max = 0

Track attempts in HANDOFF.json:
  recovery.l1_attempts → compare vs l1_max
  recovery.l2_attempts → compare vs l2_max
  recovery.l3_attempts → compare vs l3_max
```

## Acceptance Criteria
- [ ] Budget table defined inline in autonomous.md (S/M/L/XL → attempt caps)
- [ ] Default: M budget applied if recovery_budget field missing (v1 compat)
- [ ] recovery_overrides.L*.block = true → sets max to 0 for that layer
- [ ] HANDOFF.json recovery.* incremented on each attempt
- [ ] Budget exhausted correctly detected (l1_used >= l1_max, etc.)
- [ ] L3 max = 0 for S/M budgets (confirmed correct)

## Do / Don't
### Do
- Default to M budget when TASK.md recovery_budget is missing — sensible default
- Document budget table directly in autonomous.md (không external config)

### Don't
- Không allow budget override by AI autonomously — only user can override via TASK.md
- Không increment attempts in HANDOFF.json before attempting recovery — increment after

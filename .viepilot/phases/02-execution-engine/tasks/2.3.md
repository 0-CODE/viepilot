## File-Level Plan
- `workflows/autonomous.md`: Thêm explicit "write HANDOFF.json" instruction sau mỗi sub-task pass trong execution step. Thêm "append HANDOFF.log event" instruction sau mỗi: task_start, l1/l2/l3_recovery attempt, control_point, task_pass, task_skip. Events phải non-blocking (append, không wait for confirm).

## HANDOFF.json Write Trigger Points
```
After each sub-task PASS:
  → update position.sub_task
  → update recovery.* counts
  → update meta.last_written = now()
  → write HANDOFF.json

On control_point:
  → set control_point.active = true
  → set control_point.reason = "..."
  → set control_point.ts = now()
  → write HANDOFF.json

On task PASS:
  → advance position.task to next
  → reset recovery.* counts
  → write HANDOFF.json
```

## HANDOFF.log Event Format
```jsonl
{"ts":"ISO8601","event":"task_start","task":"2.3","phase":"02","complexity":"M"}
{"ts":"ISO8601","event":"l1_recovery","task":"2.3","attempt":1,"trigger":"lint_error"}
{"ts":"ISO8601","event":"l2_recovery","task":"2.3","attempt":1,"trigger":"test_fail"}
{"ts":"ISO8601","event":"control_point","task":"2.3","reason":"budget exhausted"}
{"ts":"ISO8601","event":"task_pass","task":"2.3","duration_s":142}
{"ts":"ISO8601","event":"task_skip","task":"2.3","reason":"user decision"}
```

## Acceptance Criteria
- [ ] HANDOFF.json written after each sub-task (verify: 3 sub-tasks → 3 writes, not 1)
- [ ] HANDOFF.log appended after each event (not rewritten)
- [ ] task_start event emitted when task begins
- [ ] l1/l2/l3_recovery events emitted on each recovery attempt
- [ ] task_pass event emitted on task completion
- [ ] Events are non-blocking (append instruction, no user confirmation needed)
- [ ] HANDOFF.json.meta.last_written updated with ISO 8601 timestamp

## Do / Don't
### Do
- Emit events as instruction in workflow: "Append JSON line to HANDOFF.log: {...}"
- Write HANDOFF.json as instruction: "Update HANDOFF.json with current position"

### Don't
- Không rewrite HANDOFF.log — always append
- Không block execution on log write — log is fire-and-forget
- Không commit HANDOFF.log (Task 3.1 handles gitignore)

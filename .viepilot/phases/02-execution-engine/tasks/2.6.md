## Objective

Implement 3-tier validation pipeline trong `workflows/autonomous.md` trước khi mark task PASS:
1. Contract check — verify task output matches scope contract format
2. Write scope lock verify — verify no writes outside declared write_scope (Task 2.8 handles detection; this tier gates final PASS)
3. Git gate — verify clean commit với conventional message

Order is strict: tier 1 must pass before tier 2, tier 2 before tier 3.

## Validation Tier Details
```
Tier 1 — Contract Check:
  Verify: AI output contains "RESULT: PASS|FAIL|BLOCKED — {reason}"
  Fail action: retry execution (not recovery layer) → max 1 retry

Tier 2 — Write Scope Lock:
  Verify: all modified files are within declared write_scope paths
  Source: git diff --name-only HEAD~1 vs TASK.md write_scope
  Fail action: escalate to control_point immediately (not recovery)
  Note: This is where scope drift (Task 2.8) is actioned

Tier 3 — Git Gate:
  Verify: commit message follows conventional commits format
  Verify: no unstaged changes remain
  Fail action: prompt AI to fix commit message and recommit
```

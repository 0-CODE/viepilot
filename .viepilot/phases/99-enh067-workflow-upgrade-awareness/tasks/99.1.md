# Task 99.1 — Part C: Version Stamps

## Objective
Add version metadata fields to brainstorm session headers and crystallize artifact headers so
that the gap detection mechanisms in Tasks 99.2 and 99.3 have a reliable version baseline.

## Paths
- `workflows/brainstorm.md`
- `workflows/crystallize.md`
- `skills/vp-brainstorm/SKILL.md`
- `skills/vp-crystallize/SKILL.md`

## Implementation Notes

### workflows/brainstorm.md
In the **session file format** section (the `## Session header` block or the final save format),
add `workflow_version` to the YAML front-matter written when a session is created or saved:

```yaml
workflow_version: "{{VIEPILOT_VERSION}}"      # set from package.json version at save time
upgrade_supplement_version: ""                 # filled by Task 99.2 after supplement completes
```

- `{{VIEPILOT_VERSION}}` is resolved at runtime via `vp-tools info` → `version` field (or
  reading `package.json`).
- `upgrade_supplement_version` left empty on first save; set to current version after
  a gap-detection supplement is completed (idempotency guard).

### workflows/crystallize.md
In the **Step 11: Commit & Confirm** (or the artifact header generation step), add a comment
line to `PROJECT-CONTEXT.md` when written:

```markdown
<!-- crystallize_version: {{VIEPILOT_VERSION}} -->
```

Written as the **first line** of the generated `PROJECT-CONTEXT.md` (before `# {{PROJECT_NAME}}`).
Also record in `HANDOFF.json`:
```json
"crystallize_version": "{{VIEPILOT_VERSION}}"
```

### skills/vp-brainstorm/SKILL.md
Add to objective (after `**Language configuration (ENH-032):**` block):

```
**Workflow version stamps (ENH-067):**
- Session files record `workflow_version` (current ViePilot semver) at create/save time.
- `upgrade_supplement_version` stamped after gap-detection supplement completes (idempotency).
```

### skills/vp-crystallize/SKILL.md
Add to objective (after `**User Data Management Export (ENH-066):**` block):

```
**Crystallize version stamps (ENH-067):**
- Generated `PROJECT-CONTEXT.md` includes `<!-- crystallize_version: {semver} -->` header comment.
- `HANDOFF.json` records `crystallize_version` field.
- Used by `--upgrade` re-scan mode (Task 99.3) to compute delta.
```

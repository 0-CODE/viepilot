# Task 99.3 — Part B: vp-crystallize --upgrade Re-scan Mode

## Objective
Add `--upgrade` (re-scan) mode to `vp-crystallize`. Detects delta between when the project
was last crystallized and the current workflow version; lists missing sections in existing
`.viepilot/` artifacts; offers Patch (non-destructive append) or Full re-generate.

## Paths
- `workflows/crystallize.md`
- `skills/vp-crystallize/SKILL.md`

## Implementation Notes

### Location in workflows/crystallize.md
Add a new **`## Step 0-B: Upgrade Re-scan Mode (ENH-067)`** sub-step immediately after
`## Step 0-A: Load Language Configuration`.

**Trigger:** `--upgrade` flag OR auto-detect:
- Read `.viepilot/PROJECT-CONTEXT.md` first line for `<!-- crystallize_version: {semver} -->`
- OR read `.viepilot/HANDOFF.json` → `crystallize_version`
- If `crystallize_version` is absent: treat as `0.0.0` (never been stamped)
- If `crystallize_version` < current version: offer upgrade menu

**Delta computation (same version thresholds as brainstorm gap detection):**
```
v < 2.32.0: ## Admin & Governance missing from PROJECT-CONTEXT.md
v < 2.33.0: ## Content Management missing from PROJECT-CONTEXT.md
v < 2.34.0: ## User Data Management missing from PROJECT-CONTEXT.md
```
Cross-check: even if version threshold says section should be missing, confirm it's actually
absent from the current `PROJECT-CONTEXT.md` before listing as a gap.

**Upgrade menu (AUQ — Claude Code terminal):**
```
question: "🔄 Crystallize upgrade available (v{old} → v{new}).\nDetected gaps: {list}\nHow would you like to proceed?"
options:
  - label: "Patch — append missing sections (Recommended)"
    description: "Non-destructive: reads architect notes.md YAML sections → appends missing sections only"
  - label: "Full re-generate"
    description: "Backup .viepilot/ to .viepilot/backup-pre-regen-{timestamp}/ then regenerate all artifacts"
  - label: "Skip"
    description: "Proceed with current artifacts; skip upgrade"
```

Text fallback:
```
🔄 Crystallize upgrade available (v{old} → v{new})
Gaps: {list}

  1. Patch — append missing sections only (Recommended)
  2. Full re-generate (backup first)
  3. Skip
```

**Patch mode behavior:**
1. For each missing section, run only the corresponding Step 1D export item:
   - `## Admin & Governance` → Step 1D item 7
   - `## Content Management` → Step 1D item 8
   - `## User Data Management` → Step 1D item 9
2. Check if architect `notes.md` has the corresponding YAML section (`## admin`, `## content`,
   `## user_data`). If missing: soft warn — "architect notes missing `## {section}` — will
   append placeholder table instead"; non-blocking.
3. Append sections to `.viepilot/PROJECT-CONTEXT.md` (never overwrite existing content).
4. Update `<!-- crystallize_version: {new} -->` comment to current version.
5. Commit: `chore: crystallize patch v{old} → v{new} (ENH-067)`.

**Full re-generate mode behavior:**
1. Create backup: `cp -r .viepilot/ .viepilot/backup-pre-regen-{timestamp}/`
2. Run full crystallize flow (Step 0 onward) using existing brainstorm session(s) + architect
   artifacts as source (skip Step 0 metadata Q&A — pre-fill from existing `PROJECT-META.md`).
3. Overwrite all `.viepilot/` artifacts.
4. Commit: `chore: crystallize full re-generate v{old} → v{new} (ENH-067)`.

**Brainstorm supplement integration:**
- Before patch/re-generate: check if latest brainstorm session has `## Upgrade supplement`
  sections. If yes, fold supplement Q&A results into the extraction (treat as first-class
  session content). If no supplement exists but gaps include topics > brainstorm session
  coverage: soft warn "Run /vp-brainstorm --continue first to fill Topic gaps before
  crystallizing" (non-blocking).

# Task 99.2 — Part A: vp-brainstorm Gap Detection

## Objective
Add upgrade gap detection to `vp-brainstorm`. When a user continues an existing session on a
newer ViePilot install, detect which Topics Template topics are missing from the session and
offer inline supplementary Q&A to fill the gaps.

## Paths
- `workflows/brainstorm.md`
- `skills/vp-brainstorm/SKILL.md`

## Implementation Notes

### Location in workflows/brainstorm.md
Add a new sub-step **`### Step 1B: Upgrade Gap Detection (ENH-067)`** immediately after
the existing `### Step 1: Detect Existing Sessions` block (before intent prompt).

**Trigger condition:** only fires when:
- User chose `--continue` or selects "continue most recent" at intent prompt
- The loaded session file has a `workflow_version` field that is older than the current
  ViePilot version (compare semver)
- OR the session file has no `workflow_version` field (legacy session — treat as v0.0.0)
- AND `upgrade_supplement_version` is absent or older than current version

**Gap computation:**
1. Read `workflow_version` from session header (or default `0.0.0` if absent).
2. Compare against current version Topics Template:
   - v < 2.32.0: missing Topic 6 (Admin & Governance — ENH-063)
   - v < 2.33.0: missing Topic 7 (Content Management — ENH-065)
   - v < 2.34.0: missing Topic 8 (User Data Management — ENH-066)
   - Future topics follow the same pattern as new ENH items ship
3. Cross-check: even if version says topic should be present, confirm it's actually absent
   from the session file (look for `## Topic 6`, `## Admin & Governance`, `admin:` YAML, etc.)
   — avoids false positives for users who manually added content.

**Upgrade banner (AUQ — Claude Code terminal):**
```
question: "🔄 Upgrade gap detected — session was created with ViePilot v{old}, current is v{new}.\nMissing topics: {list}. Discuss these gaps now?"
options:
  - label: "Yes — discuss now (Recommended)"
    description: "Run Q&A for missing topics inline; append ## Upgrade supplement to session"
  - label: "Remind me at /save"
    description: "Continue session normally; re-surface gap at /save"
  - label: "Skip"
    description: "Ignore gaps for this session"
```

Text fallback (non-AUQ adapters):
```
🔄 Upgrade gap detected (v{old} → v{new})
Missing topics: {list}

Options:
  1. Discuss gaps now (Recommended)
  2. Remind at /save
  3. Skip
```

**If "Yes — discuss now":**
- Run only the Q&A sub-questions for each missing topic (re-use topic blocks from Topics Template)
- Append results under:
  ```markdown
  ## Upgrade supplement (v{old} → v{new})
  *Added {date} after ViePilot upgrade*

  ### Topic 6: Admin & Governance [if missing]
  {Q&A results}

  ### Topic 7: Content Management [if missing]
  {Q&A results}

  ### Topic 8: User Data Management [if missing]
  {Q&A results}
  ```
- After supplement is written: set `upgrade_supplement_version: "{current}"` in session header
- Suggest `/vp-crystallize --upgrade` to update project artifacts

**If "Remind at /save":**
- Store pending gap list in memory; re-surface with same AUQ at `/save` before saving

**If "Skip":**
- Proceed normally; do NOT set `upgrade_supplement_version` (so gap surfaces again next session)

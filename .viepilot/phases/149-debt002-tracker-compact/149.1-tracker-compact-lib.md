# Task 149.1 — lib/tracker-compact.cjs

## Objective
Create `lib/tracker-compact.cjs` implementing the `compact(trackerPath, options)` function that
reads a bloated TRACKER.md, extracts historical entries into `TRACKER-HISTORY.md`, and rewrites
TRACKER.md with only the current state (Current State block + last N Decision Log rows).
No data is ever deleted — all history goes to TRACKER-HISTORY.md.

## Paths
- lib/tracker-compact.cjs

## File-Level Plan

### lib/tracker-compact.cjs (new)
```js
// Public API:
// compact(trackerPath, options) → { linesRemoved, rowsArchived, historyFile, trackerLines }
// rewriteCurrentState(content, newStateBlock) → string  (also exported, used by tracker-agent)
```

**`compact(trackerPath, options)`**
- `options.keep` — number of phases to keep in Current State (default 5; also controls Decision Log rows = keep × 4, min 20)
- `options.dryRun` — if true, return stats without writing files
- Algorithm:
  1. Read `trackerPath` (TRACKER.md)
  2. Parse `## Current State` block: find all "**Current Phase**:" stanzas, keep only the LAST one (current) + any "**Last Completed Phase**:", "**Next Phase**:", "**Current Task**:", "**Last Activity**:" lines that follow
  3. Parse `## Decision Log` table: extract all data rows, keep last `max(keep * 4, 20)` rows, store older rows for archive
  4. Parse `## Version Info` → keep as-is (always small)
  5. Collect all archived content (old Current State stanzas + old Decision Log rows)
  6. Write archived content to `TRACKER-HISTORY.md` (sibling of TRACKER.md, append mode with datestamp header)
  7. Rewrite TRACKER.md: replace sections with compacted versions
  8. Return `{ linesRemoved, rowsArchived, historyFile, trackerLines }`

**`rewriteCurrentState(content, newBlock)`**
- Finds `## Current State` section in content string
- Replaces everything between `## Current State` and the next `##` heading with `newBlock`
- Returns updated content string
- This is the rewrite-not-append primitive used by tracker-agent guidance

**Section detection:**
- Use `## Current State` as section header marker (exact match, start of line)
- Use `/^## /m` to find next section boundary
- Preserve everything outside the sections being compacted

**Error handling:**
- If TRACKER.md not found: throw with clear message
- If section markers not found: return content unchanged (graceful no-op)
- If TRACKER-HISTORY.md exists: append (never overwrite)

## Acceptance Criteria
- [ ] `compact()` produces a TRACKER-HISTORY.md containing the archived stanzas
- [ ] `compact()` produces a TRACKER.md with Current State ≤ 20 lines
- [ ] `compact()` with `dryRun: true` returns stats but writes no files
- [ ] `rewriteCurrentState()` replaces (not appends) the Current State block
- [ ] No data loss — line count(original) = line count(compacted TRACKER.md) + line count(new TRACKER-HISTORY.md section)
- [ ] Gracefully handles missing section markers (returns content unchanged)

## Best Practices
- CommonJS (`module.exports`), no external dependencies (only `fs`, `path`)
- Pure functions where possible — easier to unit test
- Keep < 120 lines

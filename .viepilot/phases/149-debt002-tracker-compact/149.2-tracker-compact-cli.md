# Task 149.2 — bin/vp-tools.cjs: tracker compact subcommand

## Objective
Add `tracker compact [--keep N]` subcommand to `bin/vp-tools.cjs` that calls
`lib/tracker-compact.cjs` and reports the result. Enables users and `vp-auto` to rescue
already-bloated TRACKER.md files from the command line.

## Paths
- bin/vp-tools.cjs

## Dependencies
- Task 149.1 (lib/tracker-compact.cjs) must be complete first

## File-Level Plan

### bin/vp-tools.cjs (modify)

Add a new case in the subcommand dispatcher. Pattern: follow the same style as existing
subcommands (e.g. `context-files`).

**Subcommand: `tracker <sub> [--keep N] [--dry-run] [--path <file>]`**
- `sub = compact` is the only sub-subcommand for now
- `--keep N` — number of recent phases to keep (default 5)
- `--dry-run` — report stats only, no file writes
- `--path <file>` — path to TRACKER.md (default: `.viepilot/TRACKER.md` in cwd)

**Output (stdout):**
```
✓ Compacted .viepilot/TRACKER.md
  Lines removed:  243
  Rows archived:  17
  History file:   .viepilot/TRACKER-HISTORY.md
  Result:         58 lines remaining
```

**Dry-run output:**
```
[dry-run] Would compact .viepilot/TRACKER.md
  Lines to remove:  243
  Rows to archive:  17
  No files written.
```

**Error exit codes:**
- `0` — success (or dry-run)
- `1` — TRACKER.md not found or compact failed (print error to stderr)

**Help text:** add `tracker compact [--keep N]` entry to the help output block.

## Acceptance Criteria
- [ ] `node bin/vp-tools.cjs tracker compact --dry-run` exits 0 and prints stats
- [ ] `node bin/vp-tools.cjs tracker compact` writes TRACKER-HISTORY.md and shrinks TRACKER.md
- [ ] `node bin/vp-tools.cjs tracker compact --keep 10` respects the keep value
- [ ] `node bin/vp-tools.cjs tracker compact --path nonexistent.md` exits 1 with error message
- [ ] Help output includes `tracker compact` entry

# Task 53.5 — Jest contract tests: FEAT-013 adapter system

**Phase**: 53 | **Status**: planned

## Paths
- `tests/unit/viepilot-adapters.test.js`

## Objective
Contract tests for the new adapter system. 4 test groups covering:
1. Adapter interface shape (all required fields)
2. Install plan uses adapter paths (not hardcoded .cursor)
3. Adapter registry — getAdapter() + listAdapters()
4. dev-install.sh adapter variable

## File-Level Plan

### viepilot-adapters.test.js — ~18 tests

**Group 1: Adapter interface shape (6 tests)**
- claude-code adapter has id, name, skillsDir, viepilotDir, executionContextBase
- claude-code skillsDir(home) returns path containing `.claude`
- claude-code viepilotDir(home) returns path containing `.claude/viepilot`
- cursor adapter has id, name, skillsDir, viepilotDir, executionContextBase
- cursor skillsDir(home) returns path containing `.cursor`
- cursor has hooks.configFile === null (no settings.json)

**Group 2: Adapter registry (4 tests)**
- getAdapter('claude-code') returns claude-code adapter
- getAdapter('cursor-agent') and getAdapter('cursor-ide') both return cursor adapter
- getAdapter('unknown') throws error mentioning "Unknown adapter"
- listAdapters() returns deduplicated list (cursor-agent + cursor-ide = 1 entry)

**Group 3: Install plan adapter-driven (6 tests)**
- buildInstallPlan with installTargets:['claude-code'] → paths.skillsDir contains .claude
- buildInstallPlan with installTargets:['cursor-agent'] → paths.skillsDir contains .cursor
- buildInstallPlan with [] (empty targets) → defaults to claude-code paths
- buildInstallPlan with ['claude-code','cursor-agent'] → steps include both .claude and .cursor dirs
- steps do NOT contain .cursor/skills when only claude-code target used
- steps DO contain path-rewrite step (from .cursor to .claude) when claude-code target

**Group 4: dev-install.sh adapter variable (2 tests)**
- dev-install.sh contains VIEPILOT_ADAPTER env var reference
- dev-install.sh default is claude-code (not cursor)

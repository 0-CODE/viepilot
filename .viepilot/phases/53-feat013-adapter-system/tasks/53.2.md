# Task 53.2 — lib/viepilot-install.cjs: adapter-driven buildInstallPlan()

**Phase**: 53 | **Status**: planned

## Paths
- `lib/viepilot-install.cjs`

## Objective
Refactor `buildInstallPlan()` to derive all platform paths from the adapter registry
instead of hardcoded `~/.cursor/` constants. Remove the special-case
`installClaudeSkills` if-branch — claude-code is now a first-class adapter, not an addon.

Preserve the existing `installTargets: string[]` API in `opts` for backward compatibility
(tests and CLI pass target IDs as strings). Internally, resolve each ID to its adapter.

## File-Level Plan

### Changes to viepilot-install.cjs

**Add import at top:**
```js
const { getAdapter } = require('./adapters/index.cjs');
```

**Refactor `buildInstallPlan()`:**

1. Remove hardcoded constants:
   - `const cursorSkillsDir = path.join(home, '.cursor', 'skills')` → removed
   - `const viepilotDir = path.join(home, '.cursor', 'viepilot')` → removed
   - `const installClaudeSkills = installTargets.includes('claude-code')` → removed
   - `const claudeSkillsDir = ...` → removed
   - `const claudeViepilotDir = ...` → removed

2. Resolve targets to adapters:
   ```js
   // Default to ['claude-code'] if no targets specified (FEAT-013: Claude Code primary)
   const targetIds = installTargets.length > 0 ? installTargets : ['claude-code'];
   // Deduplicate by adapter id (cursor-agent + cursor-ide → one cursor install)
   const seenAdapterIds = new Set();
   const selectedAdapters = [];
   for (const id of targetIds) {
     const adapter = getAdapter(id);
     if (!seenAdapterIds.has(adapter.id)) {
       seenAdapterIds.add(adapter.id);
       selectedAdapters.push(adapter);
     }
   }
   ```

3. Replace the two separate install loops (cursor block + claude-code if-block) with
   a single loop over `selectedAdapters`:
   ```js
   for (const adapter of selectedAdapters) {
     const skillsDir = adapter.skillsDir(home);
     const vpDir = adapter.viepilotDir(home);
     // mkdir steps
     steps.push({ kind: 'mkdir', path: skillsDir });
     for (const sub of adapter.installSubdirs) {
       steps.push({ kind: 'mkdir', path: path.join(vpDir, sub) });
     }
     // copy skills
     for (const name of listSkillDirNames(root)) {
       const src = path.join(root, 'skills', name);
       const dest = path.join(skillsDir, name);
       steps.push(env.symlinkSkills
         ? { kind: 'symlink_dir', target: dest, sourceAbsolute: path.resolve(src) }
         : { kind: 'copy_dir', from: src, to: dest });
     }
     // copy viepilot files (workflows, templates, bin, lib, ui-components)
     // ... (same content-copy logic as current claude-code block)
     // path rewrite (if adapter has pathRewrite)
     if (adapter.pathRewrite) {
       steps.push({
         kind: 'rewrite_paths_in_dir',
         dir: skillsDir,
         glob: '**/*.md',
         from: adapter.pathRewrite.from,
         to: adapter.pathRewrite.to,
       });
     }
   }
   ```

4. Update `paths` object in return value to use first selected adapter:
   ```js
   const primaryAdapter = selectedAdapters[0];
   const paths = {
     skillsDir:    primaryAdapter.skillsDir(home),
     viepilotDir:  primaryAdapter.viepilotDir(home),
     // keep legacy keys for test backward compat:
     cursorSkillsDir: selectedAdapters.find(a => a.id === 'cursor')?.skillsDir(home) || null,
     claudeSkillsDir: selectedAdapters.find(a => a.id === 'claude-code')?.skillsDir(home) || null,
     claudeViepilotDir: selectedAdapters.find(a => a.id === 'claude-code')?.viepilotDir(home) || null,
     viepilotUserDataDir,
     viepilotProfilesDir,
     viepilotProfileMapPath,
   };
   ```

5. Update `normalizeInstallEnv()` default profile:
   ```js
   profile: envSource.VIEPILOT_INSTALL_PROFILE || 'claude-code',  // was 'cursor-ide'
   ```

**Preserve unchanged:**

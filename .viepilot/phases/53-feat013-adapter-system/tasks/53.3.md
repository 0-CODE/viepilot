# Task 53.3 — bin/viepilot.cjs + dev-install.sh: adapter-aware CLI + VIEPILOT_ADAPTER

**Phase**: 53 | **Status**: planned

## Paths
- `bin/viepilot.cjs`
- `dev-install.sh`

## Objective
Update the two entry points to load platform configuration from the adapter registry
instead of using hardcoded lists. Default adapter changes to `claude-code`.

## File-Level Plan

### bin/viepilot.cjs

**Replace hardcoded TARGETS:**
```js
// Before:
const TARGETS = [
  { id: 'claude-code', label: 'Claude Code' },
  { id: 'cursor-agent', label: 'Cursor Agent' },
  { id: 'cursor-ide', label: 'Cursor IDE' },
];

// After:
const { listAdapters, adapters: adapterMap } = require(path.join(__dirname, '..', 'lib', 'adapters', 'index.cjs'));

// Build display list from registry (unique adapters + aliases)
// Keep cursor-agent and cursor-ide as separate UI targets (users know these names)
const TARGETS = [
  { id: 'claude-code',  label: adapterMap['claude-code'].name  + ' (default)' },
  { id: 'cursor-agent', label: 'Cursor Agent' },
  { id: 'cursor-ide',   label: 'Cursor IDE' },
];
```

**`normalizeTargets()` validation**: stays the same (uses TARGETS array), no change needed.

**Default target in non-interactive mode**: when `--yes` without `--target` → use `['claude-code']`
```js
// In the install flow, when targets is null after prompting:
const resolvedTargets = options.targets
  ? normalizeTargets(options.targets)
  : ['claude-code'];  // was implicitly cursor-ide
```

**`--list-targets` output**: should list from registry:
```js
function printTargets() {
  TARGETS.forEach((t) => console.log(`${t.id}\t${t.label}`));
}
```

### dev-install.sh

**Add `VIEPILOT_ADAPTER` env var with switch case:**

Replace hardcoded:
```bash
CURSOR_SKILLS_DIR="$HOME/.cursor/skills"
VIEPILOT_DIR="$HOME/.cursor/viepilot"
INSTALL_PROFILE="${VIEPILOT_INSTALL_PROFILE:-cursor-agent}"
```

With:
```bash
ADAPTER="${VIEPILOT_ADAPTER:-claude-code}"

case "$ADAPTER" in
  claude-code)
    SKILLS_DIR="$HOME/.claude/skills"
    VIEPILOT_DIR="$HOME/.claude/viepilot"
    PROFILE_LABEL="Claude Code"
    ;;
  cursor|cursor-agent|cursor-ide)
    SKILLS_DIR="$HOME/.cursor/skills"
    VIEPILOT_DIR="$HOME/.cursor/viepilot"
    PROFILE_LABEL="Cursor"
    ;;
  *)
    echo "Unknown adapter: $ADAPTER. Use: claude-code, cursor-agent, cursor-ide"
    exit 1
    ;;
esac
```

Update all references from `$CURSOR_SKILLS_DIR` → `$SKILLS_DIR` throughout the script.
Update display echo: `"  Adapter: $ADAPTER ($PROFILE_LABEL)"` instead of `"  Profile: $INSTALL_PROFILE"`.

**Backward compatibility**: `VIEPILOT_INSTALL_PROFILE` still accepted as alias for `VIEPILOT_ADAPTER`:
```bash
# Backward compat: VIEPILOT_INSTALL_PROFILE → VIEPILOT_ADAPTER
ADAPTER="${VIEPILOT_ADAPTER:-${VIEPILOT_INSTALL_PROFILE:-claude-code}}"
```

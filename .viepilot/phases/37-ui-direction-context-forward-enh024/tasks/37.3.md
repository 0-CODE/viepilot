After loading `tasks/{task}.md`, check if this is a UI task:
- Task description/objective mentions: UI, frontend, page, component, layout, design, screen, style, HTML, CSS.

If UI task detected:
1. Check `context_required` in task file for any `.viepilot/ui-direction/` entry.

   ```bash
   ls -la .viepilot/ui-direction/ | sort | tail -1  # get latest session-id
   ```
   Read: `.viepilot/ui-direction/{latest-session}/notes.md` and `style.css`.
3. **If present** → load as specified.

Purpose: prevent implementation diverging from design intent when task file was created before ui-direction artifacts existed, or when injection was missed during crystallize.

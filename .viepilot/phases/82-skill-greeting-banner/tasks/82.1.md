outputs the version banner **before** any other work when a skill is invoked.

## File-Level Plan
Each SKILL.md gets a `<greeting>` block inserted immediately **after** the YAML
frontmatter closing `---` (before any other content). The block contains:
1. The banner template
2. The instruction: "Output this banner immediately when invoked, before questions or work"

### `<greeting>` block template
```xml
<greeting>
## Invocation Banner

On every invocation, output this banner as the **first** thing — before questions, work, or any other output:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 VIEPILOT ► {SKILL_NAME_UPPER}  v{SKILL_VERSION} (fw {FW_VERSION})
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Where:
- `{SKILL_NAME_UPPER}` = skill name in uppercase (e.g. `VP-AUTO`)
- `{SKILL_VERSION}` = `version:` from this file's frontmatter (e.g. `0.2.2`)
- `{FW_VERSION}` = framework version — read from `~/.claude/viepilot/` installed copy or use `2.19.0` as default
</greeting>
```

Insert position: immediately after the closing `---` of YAML frontmatter.

- [ ] Instruction is clear: output BEFORE questions or work
- [ ] `grep -l '<greeting>' skills/*/SKILL.md | wc -l` == 17

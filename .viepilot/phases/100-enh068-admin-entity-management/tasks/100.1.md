### A — New Topic 7 in Topics Template (brainstorm.md)

**Insert after Topic 6 (Admin & Governance) block, before current Topic 7 (Content Management).**

```
7. **Admin Entity Management** (ENH-068)
   - Which DB entities / domain objects need admin CRUD interfaces?
     (e.g., products, orders, categories, tenants, campaigns, bookings…)
   - For each entity: which operations are needed? (Create / Read / Update / Delete)
   - **List view** requirements: columns, default sort, pagination style (offset vs. cursor),
     search & filter fields, range filters (date, price), multi-select filters
   - **Bulk actions**: bulk delete, bulk status change, bulk export — which entities need these?
   - **Create/Edit form**: required vs. optional fields, validation rules, nested relation fields
     (e.g., product → category), rich text / file upload fields, inline vs. separate page,
     auto-save / draft mode needed?
   - **Delete semantics**: hard delete vs. soft delete (`is_deleted` / `deleted_at`);
     cascade rules for related records; restore / undelete capability needed?
   - **Import / Export**: CSV/XLSX import for bulk seeding (e.g., product catalog);
     export filtered data from list views; import validation error reporting
   - **Audit trail per entity**: which entities need change history (actor + timestamp + diff)?
     Separate audit table vs. event-sourced; surface in admin UI or log-only?
   - **Admin ownership scoping**: are entities scoped per tenant/org (multi-tenant)?
     Can org_admin manage only their org's entities while super_admin sees all?
   - **Read-only vs. editable**: which entities are system-generated (read-only in admin)
     vs. user/admin-generated (editable)?
```

# ViePilot Hooks

ViePilot supports Claude Code's hook system to automate actions after each AI response.

## Brainstorm Staleness Hook

**Event**: `Stop` (fires after each Claude response)  
**Script**: `~/.viepilot/hooks/brainstorm-staleness.cjs`  
**Config**: `~/.claude/settings.json`

### What it does
After each exchange in a `vp-brainstorm` session, the hook:
1. Finds the active brainstorm session notes
2. Scans for keywords matching architect HTML pages
3. Marks relevant architect items `data-arch-stale="true"` (amber badge in browser)
4. Non-blocking — if it fails, your session continues normally

### Install
```bash
node bin/vp-tools.cjs hooks install
# or: node bin/vp-tools.cjs hooks install --adapter claude-code
```

### Verify
Check `~/.claude/settings.json` contains a `Stop` entry, OR run:
```bash
node bin/vp-tools.cjs hooks scaffold
```

### Cursor users
Cursor does not support `settings.json` hook events. Use `/sync-arch` manually
within the brainstorm session to trigger architect delta sync (ENH-034).

## Adapters and hooks
Hook support depends on the adapter:
| Adapter | Hook config | Supported events |
|---------|-------------|-----------------|
| `claude-code` | `~/.claude/settings.json` | Stop, PreToolUse, PostToolUse, … |
| `cursor`      | `.cursorrules` (MDC)      | None (programmatic) |

## Adding custom hooks
Use `vp-tools hooks scaffold --adapter <id>` to see the config format, then
add your hook command to the `Stop` (or other event) array.
```

## vp-evolve/SKILL.md — Add to `<context>` section

```
**Task path convention (BUG-009):**
When generating task files, ALL paths in `## Paths` blocks MUST be
repo-relative (relative to the repository root where package.json lives).

CORRECT:   `workflows/foo.md`, `skills/vp-bar/SKILL.md`, `lib/foo.cjs`
INCORRECT: `~/.claude/viepilot/workflows/foo.md`, `/absolute/path`

Absolute paths inside code block content (bash examples, etc.) are fine —
only the `## Paths` header block must use repo-relative paths.
```

## vp-auto/SKILL.md — Add to `<context>` section

```
**Task path validation (BUG-009):**
Before executing each task, validates that all paths in the `## Paths`
block are repo-relative. If any path starts with `~/` or `/`:
- Execution stops for that task
- Error message identifies the offending path and task file
- User must fix the task file before continuing

This guard prevents accidentally editing the live ViePilot install
(~/.claude/) instead of the source repository.
```

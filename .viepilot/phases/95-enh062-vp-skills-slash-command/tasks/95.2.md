# Task 95.2 — Add `get-registry` to `bin/vp-tools.cjs`

## Objective
Thêm subcommand `get-registry` vào vp-tools.cjs: output registry JSON ra stdout để agent (và workflows) có thể đọc và parse bằng shell.

## Paths
- `bin/vp-tools.cjs`

## Implementation Notes

### Command signature
```bash
vp-tools get-registry [--id <skill-id>]
```

Output (stdout):
- **Không có `--id`**: toàn bộ registry JSON (`~/.viepilot/skill-registry.json`)
- **Có `--id <id>`**: chỉ object của skill đó `{ id, capabilities, best_practices, adapters, ... }` hoặc `{}` nếu không tìm thấy

Exit codes:
- 0 — success (kể cả registry absent → output `null`)
- 1 — parse error

### Implementation pattern (follow existing scan-skills pattern)
```js
if (command === 'get-registry') {
  const { loadRegistry } = require('../lib/skill-registry.cjs');
  const registry = loadRegistry();
  if (!registry) { process.stdout.write('null\n'); process.exit(0); }

  const idFilter = rest[0] === '--id' ? rest[1] : null;
  if (idFilter) {
    const skill = registry.skills.find(s => s.id === idFilter) || null;
    process.stdout.write(JSON.stringify(skill) + '\n');
  } else {
    process.stdout.write(JSON.stringify(registry) + '\n');
  }
  process.exit(0);
}
```

### printHelp() addition
```
  get-registry [--id <id>]         Output global skill registry as JSON
```

### Usage trong workflows (sau khi fix BUG-016)
```bash
node ~/.claude/viepilot/bin/vp-tools.cjs get-registry 2>/dev/null
# → full JSON registry (null nếu vắng)

node ~/.claude/viepilot/bin/vp-tools.cjs get-registry --id frontend-design 2>/dev/null
# → { id: "frontend-design", capabilities: [...], best_practices: [...], ... }
```

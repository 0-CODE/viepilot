# Task 95.3 — Fix BUG-016: Replace `Call loadRegistry()` with shell commands in workflows

## Objective
Thay `Call loadRegistry() from lib/skill-registry.cjs` (JS function — không executable) bằng shell command thực thi được trong `workflows/autonomous.md` và `workflows/brainstorm.md`.

## Paths
- `workflows/autonomous.md`
- `workflows/brainstorm.md`

## Implementation Notes

### workflows/autonomous.md — Skill Context Load fix

**Thay đoạn:**
```
Build `SKILL_CONTEXT_MAP` in session memory:
- Parse `## Skills` table rows
- For each row: extract `id`, `source`, `required` (yes/no), `phases` (comma list)
- Call `loadRegistry()` to get `best_practices[]` for each skill
- Structure: `{ required: [{id, phases: [1,2], best_practices: [...]}], optional: [...] }`
```

**Thành:**
```
Build `SKILL_CONTEXT_MAP` in session memory:
- Parse `## Skills` table rows
- For each row: extract `id`, `source`, `required` (yes/no), `phases` (comma list)
- Load best_practices[] for each skill from global registry:
  ```bash
  node ~/.claude/viepilot/bin/vp-tools.cjs get-registry --id {skill-id} 2>/dev/null \
    || node ~/.cursor/viepilot/bin/vp-tools.cjs get-registry --id {skill-id} 2>/dev/null
  ```
- Parse JSON output → extract `best_practices[]`
- Structure: `{ required: [{id, phases: [1,2], best_practices: [...]}], optional: [...] }`

If command absent or returns null: `best_practices = []` — silent no-op.
```

### workflows/brainstorm.md — Skill Registry Integration fix

**Thay đoạn:**
```
**Step — Load registry:**
- Call `loadRegistry()` from `lib/skill-registry.cjs` (read `~/.viepilot/skill-registry.json`)
- If registry absent or empty: skip silently — no warning, no prompt
```

**Thành:**
```
**Step — Load registry:**
```bash
node ~/.claude/viepilot/bin/vp-tools.cjs get-registry 2>/dev/null \
  || node ~/.cursor/viepilot/bin/vp-tools.cjs get-registry 2>/dev/null \
  || cat ~/.viepilot/skill-registry.json 2>/dev/null
```
Parse JSON output → extract `skills[]`.
If output is `null` or command fails: skip silently — no warning, no prompt.
```

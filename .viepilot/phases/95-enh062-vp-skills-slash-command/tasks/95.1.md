## Objective
Tạo skill mới `vp-skills` với đầy đủ commands để agent có thể gọi `/vp-skills` từ bất kỳ project nào. Skill tự động được install vào adapter dirs bởi `vp-tools install` (vì nằm trong `skills/` directory).

### Commands (all 6)
```
/vp-skills scan          — Refresh ~/.viepilot/skill-registry.json
/vp-skills list          — Display indexed skills table
/vp-skills install <src> — Install from npm/github/local
/vp-skills uninstall <id>— Remove skill from all adapter dirs
/vp-skills update <id>   — Re-install from original source
/vp-skills info <id>     — Show capabilities, best_practices, adapters
```

### Execution (installed path — cross-project)
Dùng installed vp-tools binary, **không phụ thuộc cwd**:
```bash
# scan
node ~/.claude/viepilot/bin/vp-tools.cjs scan-skills

# list
node ~/.claude/viepilot/bin/vp-tools.cjs list-skills

# install
node ~/.claude/viepilot/bin/vp-tools.cjs install-skill <source>

# uninstall
node ~/.claude/viepilot/bin/vp-tools.cjs uninstall-skill <id>

# update
node ~/.claude/viepilot/bin/vp-tools.cjs update-skill <id>

# info — get-registry (task 95.2), filter by id
node ~/.claude/viepilot/bin/vp-tools.cjs get-registry --id <id>
```

Fallback: nếu `~/.claude/viepilot/bin/vp-tools.cjs` absent → thử `~/.cursor/viepilot/bin/vp-tools.cjs`.

### Scope policy
Skill này quản lý global registry — áp dụng cho tất cả projects. Không giới hạn vp-* namespace (người dùng có thể cài skills từ bất kỳ nguồn nào).

### Cross-references
- Install: `vp-tools install --target <adapter>`
- Workflow integration: `docs/user/features/skill-registry.md`

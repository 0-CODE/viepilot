# Task 43.3 — brainstorm.md: incremental update rules + notes.md schema

**Phase**: 43 | **Status**: todo

## What
Add incremental update rules and notes.md YAML schema to Architect Design Mode section in `workflows/brainstorm.md`.

## Incremental update rule
When user changes a decision:
- Identify which HTML file is affected: decision → `decisions.html`; tech choice → `tech-stack.html`; component → `architecture.html`; feature scope → `feature-map.html`
- Update ONLY that file + update `notes.md`
- Add `data-updated="true"` to affected `<tr>` or `<section>`
- Reset `data-updated` at start of each new session

## notes.md YAML schema
```yaml
---

project: {project name}
created: {ISO date}

---

  chosen: {chosen}

  devops: {value}
  cache: {value} # optional
  monitoring: {value} # optional
```

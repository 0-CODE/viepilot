## Implementation Notes

Create `docs/user/features/adapters.md` with:

```markdown
# Supported Adapters

ViePilot supports multiple AI coding platforms via its adapter system.

## Supported Platforms

| Adapter ID | Platform | Skills dir | ViePilot dir | Hooks |
|------------|----------|------------|--------------|-------|
| `claude-code` | Claude Code (default) | `~/.claude/skills/` | `~/.claude/viepilot/` | ✅ Stop, PreToolUse, … |
| `cursor-agent` / `cursor-ide` | Cursor | `~/.cursor/skills/` | `~/.cursor/viepilot/` | — |
| `antigravity` | Google Antigravity | `~/.antigravity/skills/` | `~/.antigravity/viepilot/` | — |

## Install for a specific platform

```bash
# Claude Code (default)
viepilot install

# Cursor
VIEPILOT_ADAPTER=cursor-agent viepilot install
# or via dev-install.sh:
VIEPILOT_ADAPTER=cursor-agent ./dev-install.sh

# Antigravity
VIEPILOT_ADAPTER=antigravity viepilot install
# or:
VIEPILOT_ADAPTER=antigravity ./dev-install.sh
```

## Adding a new adapter

Create `lib/adapters/{name}.cjs` with:
- `id`, `name`, `skillsDir(home)`, `viepilotDir(home)`
- `executionContextBase` — used to resolve `{envToolDir}` in SKILL.md files at install time
- `hooks.configFile` — path to settings file (or `null` if no hooks support)
- `installSubdirs` — dirs to copy into `viepilotDir`
- `isAvailable(home)` — returns true if platform is detected

Register in `lib/adapters/index.cjs`. No `pathRewrite` needed (handled automatically by `{envToolDir}` substitution).
```

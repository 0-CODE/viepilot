# Task 52.1 — workflows/brainstorm.md: architect_delta_sync step

**Phase**: 52 | **Status**: planned

## Paths
- `workflows/brainstorm.md`

## Objective
Add a new `<step name="architect_delta_sync">` step at the end of the brainstorm workflow
(after the `close_session` step). This step bridges the gap between UI Direction Mode
and the Architect HTML workspace: when a UI brainstorm session surfaces architect-related
gaps or changes, those deltas are parsed and written back to the relevant architect HTML
template files in `.viepilot/architect/{session-id}/`.

## File-Level Plan

### brainstorm.md — add architect_delta_sync step

**Trigger conditions** (either):
1. End of a UI brainstorm session (`.viepilot/ui-direction/{session-id}/` exists) when
   the session contains ≥1 architect keyword hit during UI discussion.
2. User types `/sync-arch` at any point in a session.

**Step logic:**
```
1. Check architect session exists:
   - Find .viepilot/architect/{session-id}/ matching the current session date/context
   - If no architect session exists: skip (note: "no architect workspace found — run
     /vp-brainstorm --architect first to create one")

2. Gap detection: scan current brainstorm session text for architect keywords by page:
   - architecture / erd / apis / deployment / data-flow / use-cases / sequence-diagram
   - Use the existing trigger keyword lists already defined in brainstorm.md for each page

3. For each affected page with detected gaps:
   a. Read the current HTML file (.viepilot/architect/{id}/{page}.html)
   b. Identify the affected <tr> rows or <div class="card"> items by matching
      the context of the brainstorm gap to the nearest item title
   c. Update the item content (cell text / heading / description) with the new information
   d. Add data-updated="true" + class="updated" to the changed element
   e. Add data-arch-sync-note="{brief reason}" attribute to record why it changed

4. Record delta in notes.md:
   Append under ## architect_sync:
   ```yaml
   ## architect_sync
   - synced_at: {datetime}
     source_session: {ui-session-id}
     changes:
       - page: {slug}
         item_id: {data-arch-id}
         change: "{brief description of what was updated}"
   ```

5. Output sync report to user:
   ```
   🔄 Architect Sync complete
   Updated {N} items across {M} pages:
   - {page}: {item-id} — {what changed}
   ...
   Items marked with data-updated="true" in HTML. Review in architect workspace.
   ```
```

**When NO changes needed:**
- If no gaps found: skip silently (no output unless `/sync-arch` was explicit command).
- If `/sync-arch` was used and no gaps found: report "✓ No architect gaps detected in this session."

# Task 57.1 — Untrack .viepilot/ from git index

## Goal
Run `git rm -r --cached .viepilot/` to remove all `.viepilot/` files from the git index (while keeping them on disk). Commit the result so that `.viepilot/` is no longer tracked by version control.

## Paths
```
(no file edits — this task is a git operation only)
```

## Implementation Notes

Run the following commands in sequence:

```bash
# 1. Remove all .viepilot/ files from the git index (keep on disk)
git rm -r --cached .viepilot/

# 2. Verify no .viepilot/ files remain tracked
git ls-files .viepilot/   # must return empty

# 3. Commit the removal
git commit -m "chore: untrack .viepilot/ (gitignored internal state)"
```

After this commit:
- `.viepilot/` files remain on disk (only the git index entry is removed)
- `git status` will show them as `??` (untracked) — which is correct and expected
- Subsequent `git status --porcelain` will NOT include them in modified/staged output

## Acceptance Criteria
- [ ] `git ls-files .viepilot/` returns empty after commit
- [ ] `.viepilot/` directory still exists on disk
- [ ] `git status --porcelain` no longer shows `.viepilot/` files as modified/staged
- [ ] Commit message is `chore: untrack .viepilot/ (gitignored internal state)`

### workflows/brainstorm.md — addition in Architect Design Mode step
Add after the existing architect workspace description:

```markdown
**Architect Item Actions (ENH-033):**
- Each item in the Architect HTML workspace has a unique ID (`[ARCH:{page-slug}:{item-id}]`).
- Approve prompt: `[ARCH:decisions:D1] APPROVE` — confirm item as-is on that page only.
- Edit prompt: `[ARCH:decisions:D1] EDIT` — open targeted edit conversation for that item only.
- **Isolation rule:** Each action is STRICTLY scoped to the named page and item.
  Approving `[ARCH:architecture:C2]` does NOT approve any item in erd, apis, decisions, or any other page.
  Never infer or cascade approval across pages. Cross-page updates require separate explicit prompts.
- When receiving an EDIT prompt: acknowledge the item by its full ID, state current content, ask what to change.
- When receiving an APPROVE prompt: confirm the item, do not touch any other item or page.
```

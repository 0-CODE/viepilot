# Task 50.2 — templates/architect/architect-actions.js: shared JS

**Phase**: 50 | **Status**: planned

## Paths
- `templates/architect/architect-actions.js` (new)

## Objective
Shared vanilla JS file included in all 11 content templates. Injects Approve/Edit buttons into every element with `data-arch-id`, and handles clipboard copy with "Copied!" feedback.

## File-Level Plan
### architect-actions.js
Constants:
- `APPROVE_TEMPLATE` — `"[ARCH:{slug}:{id}] APPROVE — \"{title}\" on {slug} page. No changes needed."`
- `EDIT_TEMPLATE` — `"[ARCH:{slug}:{id}] EDIT — \"{title}\" on {slug} page. Current: \"{excerpt}\". What should I change?"`

Functions:
- `copyArchPrompt(type, slug, id, title, excerpt)` — builds prompt string from template; calls `navigator.clipboard.writeText()` with fallback `execCommand('copy')`; toggles `.copied` class on button for 1500ms
- `injectActionButtons()` — runs on DOMContentLoaded; queries all `[data-arch-id]`; reads `data-arch-slug` (page slug) from `<body>` or passed constant; reads title from `data-arch-title` attribute or first `<td>`/`<h3>` text; reads excerpt from `data-arch-excerpt` or first text content ≤80 chars; creates `.arch-item-actions` div with badge + 2 buttons; appends to element

## Acceptance Criteria
- [ ] `APPROVE_TEMPLATE` and `EDIT_TEMPLATE` constants defined
- [ ] `copyArchPrompt()` uses navigator.clipboard with execCommand fallback
- [ ] Buttons show "✅" / "✏️" icons with label
- [ ] "Copied!" feedback class applied for 1500ms then reverted
- [ ] `injectActionButtons()` fires on DOMContentLoaded
- [ ] Prompt contains page slug — enforces isolation

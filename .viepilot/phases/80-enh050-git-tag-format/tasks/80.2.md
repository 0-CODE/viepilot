# Task 80.2 — workflows/audit.md + workflows/rollback.md: Regex Update

## Objective
Update tag detection patterns in `audit.md` and `rollback.md` to match the new enriched format `{prefix}-{branch}-{version}-vp-p{n}-*` while still matching the legacy format `{prefix}-vp-p{n}-*` for backward compatibility.

## Paths
- `workflows/audit.md`
- `workflows/rollback.md`

## File-Level Plan

### audit.md — Change 1: COMPLETE_TAGS regex (Step 1d)

**Current:**
```bash
COMPLETE_TAGS=$(git tag 2>/dev/null | grep -E '(^vp-p[0-9]+-complete$)|(^[a-z0-9-]+-vp-p[0-9]+-complete$)' | sort)
```

**New** (add third alternative for enriched format `prefix-branch-version-vp-pN-complete`):
```bash
COMPLETE_TAGS=$(git tag 2>/dev/null | grep -E \
  '(^vp-p[0-9]+-complete$)|(^[a-z0-9-]+-vp-p[0-9]+-complete$)|(^[a-z0-9._-]+-[a-z0-9._-]+-[0-9]+\.[0-9]+\.[0-9]+-vp-p[0-9]+-complete$)' \
  | sort)
```

### audit.md — Change 2: PREV_TAG grep (Step 2 docs-drift check)

**Current:**
```bash
PREV_TAG=$(git tag --sort=-version:refname 2>/dev/null | grep -E "(vp-p.*-complete|[a-z0-9-]+-vp-p.*-complete)" | grep -A1 "^$PHASE_TAG$" | tail -1)
```

The existing pattern `[a-z0-9-]+-vp-p.*-complete` already matches enriched format because `[a-z0-9-]+` can match `prefix-branch-version`. However the `-` in the character class should also include `.` for version dots:

**New:**
```bash
PREV_TAG=$(git tag --sort=-version:refname 2>/dev/null \
  | grep -E "(vp-p.*-complete|[a-z0-9._-]+-vp-p.*-complete)" \
  | grep -A1 "^$PHASE_TAG$" | tail -1)
```

### rollback.md — Change 1: tag grep pattern

**Current:**
```bash
git tag --sort=-creatordate | rg "(^vp-p|^-*vp-backup|^[a-z0-9-]+-vp-p|^[a-z0-9-]+-vp-backup)" | head -20
```

**New** (add `.` to character classes to match version strings):
```bash
git tag --sort=-creatordate | rg "(^vp-p|^-*vp-backup|^[a-z0-9._-]+-vp-p|^[a-z0-9._-]+-vp-backup)" | head -20
```

## Implementation Notes
- All changes are additive — legacy tags (`viepilot-vp-p60-complete`) still match.
- The enriched format anchor: `prefix-branch-version-vp-pN` where version is `\d+\.\d+\.\d+`.
- The COMPLETE_TAGS regex adds an explicit third alternative for precision; the PREV_TAG and rollback patterns extend character classes to include `.` (version dots).
- Do NOT change the audit check logic — only the tag matching patterns.

# Task 80.1 — workflows/autonomous.md: Tag Format Update

## Objective
Update `workflows/autonomous.md` to resolve `BRANCH_SAFE` + `VERSION` at initialize time and use the enriched tag format in all 3 tag creation points.

## Paths
- `workflows/autonomous.md`

## File-Level Plan

### Change 1 — Initialize step: add tag variables block
In the **Initialize** section (Step 1 / startup block), after loading TRACKER and ROADMAP, add a tag variable resolution block:

```markdown
#### Tag Prefix Resolution
```bash
# Resolve enriched tag prefix — include branch + version
PROJECT_PREFIX=$(node bin/vp-tools.cjs info --prefix 2>/dev/null \
  || grep -i "^prefix:" .viepilot/PROJECT-META.md 2>/dev/null | awk '{print $2}' \
  || basename "$(pwd)")
BRANCH_SAFE=$(git rev-parse --abbrev-ref HEAD 2>/dev/null \
  | sed 's/[^a-zA-Z0-9._-]/-/g' || echo "main")
VERSION=$(node -e "try{console.log(require('./package.json').version)}catch(e){console.log('0.0.0')}" 2>/dev/null \
  || grep '"version"' package.json 2>/dev/null | head -1 | sed 's/.*"\([0-9.]*\)".*/\1/' || echo "0.0.0")
TAG_PREFIX="${PROJECT_PREFIX}-${BRANCH_SAFE}-${VERSION}"
```
Tags use this prefix: `${TAG_PREFIX}-vp-p{phase}-t{task}`
\```
```

### Change 2 — Task start checkpoint: update tag pattern
**Find:** `Create git tag: \`{projectPrefix}-vp-p{phase}-t{task}\``
**Replace with:** `Create git tag: \`{TAG_PREFIX}-vp-p{phase}-t{task}\``
where `{TAG_PREFIX}` = `${PROJECT_PREFIX}-${BRANCH_SAFE}-${VERSION}` resolved above.

Also add the bash form:
```bash
git tag "${TAG_PREFIX}-vp-p${PHASE}-t${TASK}"
```

### Change 3 — Task done tag: update pattern
**Find:** `Create git tag: \`{projectPrefix}-vp-p{phase}-t{task}-done\``
**Replace with:** `Create git tag: \`{TAG_PREFIX}-vp-p{phase}-t{task}-done\``

### Change 4 — Phase complete tag: update pattern
**Find:** `Create git tag: \`{projectPrefix}-vp-p{phase}-complete\``
**Replace with:** `Create git tag: \`{TAG_PREFIX}-vp-p{phase}-complete\``

### Change 5 — Doc-first gate comment: update reference
The guard comment:
> "Do not create `{projectPrefix}-vp-p{phase}-t{task}`"

Update to:
> "Do not create `{TAG_PREFIX}-vp-p{phase}-t{task}`"

## Implementation Notes
- All 5 changes are to markdown instructions (not code) — they teach the AI what commands to run.
- The bash block under Tag Prefix Resolution is an example/template for the AI to follow — it will execute this pattern at runtime.
- Dots in version string are valid in git tag names (e.g. `viepilot-main-2.17.0-vp-p80-t1`).
- If `package.json` is absent (bare workflow usage), falls back to `0.0.0` — non-blocking.

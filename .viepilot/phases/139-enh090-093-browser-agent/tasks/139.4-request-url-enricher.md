### workflows/request.md
Add pre-step BEFORE Step 1 (Request Type Detection):

```markdown
### Pre-Step: URL Detection
If `{{VP_ARGS}}` matches a known issue tracker URL pattern (GitHub Issue, Linear, Jira,
Trello, Notion), run URL enrichment before prompting for request type:

1. Detect URL type via `lib/request/url-enricher.cjs detectIssueUrl()`
2. On match: dispatch `browser-intake-agent` with `op: read_url, extract_mode: ticket`
3. Show pre-fill preview:
   ```
   Context extracted from: {url}
     Title:       {title}
     Type:        {type} (detected from labels/tags)
     Description: {description preview}
     Priority:    {priority}
   Confirm? [y / edit]
   ```
4. On confirm: skip redundant questions in Step 2, proceed to acceptance criteria
5. On no match / extraction failure: proceed normally from Step 1
6. browser-intake-agent not available: skip enrichment silently, proceed from Step 1
```

## Best Practices
- url-enricher is a pure dispatch layer — no network calls, no browser automation itself
- `enrichFromUrl` must never throw — wrap in try/catch, return null on any failure
- regex patterns in URL_ROUTES should not be overly greedy; test with unit assertions
- Fallback path (no browser-intake-agent) must be silent — don't interrupt vp-request UX

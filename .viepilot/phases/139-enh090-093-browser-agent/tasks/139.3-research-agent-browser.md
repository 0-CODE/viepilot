## File-Level Plan

### agents/claude-code/research-agent.md
Current model: `claude-sonnet-4-6` (keep).
Add to system prompt / ops section:

```
## Browser Research Operations (requires vercel-labs/agent-browser)

When input contains a URL:
- `browse_url`: Open URL with agent-browser, snapshot -i, extract structured content
  (features list, tech stack, pricing, UX patterns). Return as markdown summary.
- `compare_products`: Accept list of URLs, browse each, produce comparison table
  (columns: feature, product_A, product_B, ...).
- `extract_pricing`: Navigate to /pricing page (or follow pricing link), extract tiers,
  prices, feature lists as structured JSON.

Fallback (no agent-browser): Use WebFetch. Warn if content appears JS-rendered (empty body).
```

Keep existing WebSearch + WebFetch behavior for non-URL inputs unchanged.

### workflows/brainstorm.md
Find Step 3 (Explore / Research phase). Add URL research sub-block:

```markdown
### Step 3.X: Reference URL Research (when user provides URL)
If user provides a reference URL (competitor, inspiration, documentation):

```

```xml
## Step 0-B: Brownfield Mode Detection

Trigger brownfield mode when ANY of the following is true:
- `--brownfield` flag passed explicitly
- `docs/brainstorm/` directory does not exist or is empty (no `session-*.md` files)
  AND `.viepilot/` directory does not yet exist

If NEITHER condition is met (greenfield): skip this step entirely, proceed to Step 1.

If brownfield triggered AND `.viepilot/` already exists:
  → Warn user: "`.viepilot/` already exists. Re-running brownfield mode will overwrite artifacts."
  → Ask: "Continue? (y/n)" — abort if n.

When brownfield mode is active:
  → Run Step 0-B scanner (below).
  → After scanner + gap-filling complete → skip Step 1 (`analyze_brainstorm`); proceed to Step 0-C (brainstorm stub generation) then Step 2 onward.
```

## What to implement

### skills/vp-crystallize/SKILL.md

1. In the `<objective>` or flags section, add `--brownfield` flag:

```markdown
### Flags
- (no flag)       : Greenfield mode — requires `docs/brainstorm/session-*.md`
- `--brownfield`  : Brownfield mode — bootstraps context from existing codebase.
                    Auto-triggered when no brainstorm session + no `.viepilot/` found.
```

2. Add a **Brownfield Mode** subsection under the objective summarizing the scanner:

```markdown
## Brownfield Mode (`--brownfield`)

Scans the existing codebase across 12 signal categories:
1. Build manifests (package.json, pom.xml, pyproject.toml, Cargo.toml, go.mod, …)
2. Framework detection (backend, frontend, ORM, auth, message broker, test)
3. Architecture layers (directory structure — 18 patterns)
4. Database schema signals (migration files, ORM, docker-compose services)
5. API contracts (OpenAPI, gRPC, GraphQL)
6. Infrastructure & deployment (Dockerfile, docker-compose, k8s, Terraform, PaaS)
7. Environment config shape (.env.example — never .env)
8. Test coverage signals (framework configs, coverage report dirs)
9. Code quality tools (ESLint, SonarQube, pre-commit, Checkstyle, …)
10. Documentation files (README, CHANGELOG, ADRs, docs/)
11. Git history (commit convention, version pattern, contributors)
12. File extension survey (language distribution)

Produces a **Scan Report** with DETECTED / ASSUMED / MISSING gap classification.
MUST-DETECT gaps block artifact generation until user fills them interactively.
Generates `docs/brainstorm/session-brownfield-import.md` stub for audit compatibility.
```

### skills/vp-audit/SKILL.md

Find the section that checks for brainstorm session files (Tier 1 or equivalent consistency check). Add a rule:

```markdown
### Brownfield Import Compatibility

If `docs/brainstorm/` exists and contains only `session-brownfield-import.md` (no `session-*.md` greenfield files):
- This is a **valid brownfield import** — do NOT flag as missing brainstorm session.
- Verify that `session-brownfield-import.md` contains a Scan Report YAML block.
- If Scan Report block is absent → flag as LOW severity: "Brownfield stub missing Scan Report content."
- If `.viepilot/TRACKER.md` contains `## Brownfield Import` section → confirm brownfield metadata is present; no further brainstorm check needed.
```

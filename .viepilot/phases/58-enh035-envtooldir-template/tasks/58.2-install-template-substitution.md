# Task 58.2 — Update viepilot-install.cjs: unconditional {envToolDir} substitution

## Goal
Replace the conditional `if (adapter.pathRewrite)` block in `lib/viepilot-install.cjs` with an unconditional `rewrite_paths_in_dir` step that substitutes `{envToolDir}` → `adapter.executionContextBase` for every adapter.

## Paths
```
lib/viepilot-install.cjs
```

## Implementation Notes

Around line 202, replace:
```js
// path rewrite in skills (if adapter requires it — e.g. claude-code rewrites .cursor → .claude)
if (adapter.pathRewrite) {
  steps.push({
    kind: 'rewrite_paths_in_dir',
    dir: skillsDir,
    glob: '**/*.md',
    from: adapter.pathRewrite.from,
    to: adapter.pathRewrite.to,
  });
}
```

With:
```js
// resolve {envToolDir} placeholder → adapter's actual install base (ENH-035)
steps.push({
  kind: 'rewrite_paths_in_dir',
  dir: skillsDir,
  glob: '**/*.md',
  from: '{envToolDir}',
  to: adapter.executionContextBase,
});
```

This is unconditional — every adapter gets the substitution. For Cursor it replaces `{envToolDir}` with `.cursor/viepilot`; for Claude Code with `.claude/viepilot`; for future adapters with their `executionContextBase`.

## Acceptance Criteria
- [ ] `lib/viepilot-install.cjs` no longer references `adapter.pathRewrite`
- [ ] The `rewrite_paths_in_dir` step uses `from: '{envToolDir}'` and `to: adapter.executionContextBase`
- [ ] Step is always emitted (no `if` gate)
- [ ] No other logic in the file is changed

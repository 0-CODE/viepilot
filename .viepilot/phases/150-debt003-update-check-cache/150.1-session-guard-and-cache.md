# Task 150.1 — OS session guard + 6h TTL

## Objective
Add an OS-level session guard to `check-update --silent` so that calls 2–N in the same OS session
exit immediately with **zero stdout** (zero tokens). Reduce the disk cache TTL from 24h → 6h.

## Paths
- bin/vp-tools.cjs
- lib/viepilot-update.cjs

## Context
`lib/viepilot-update.cjs` already has a `~/.viepilot/update-cache.json` with 24h TTL (line 164).
`bin/vp-tools.cjs` `check-update` handler is at line 1173.

The guard must be in `bin/vp-tools.cjs` (the CLI entry point) — before even calling
`checkLatestVersion()` — so Node.js module loading overhead is also bypassed on the fast path.

## Implementation Plan

### Change 1 — bin/vp-tools.cjs: OS session guard (primary fix)

In the `'check-update': (args) => { ... }` handler, **before** the `require('../lib/viepilot-update.cjs')` call, insert:

```js
const os = require('os');
const today = new Date().toISOString().split('T')[0]; // "2026-05-25"
const guardFile = require('path').join(os.tmpdir(), `vp-update-check-${today}.done`);

if (!force && fs.existsSync(guardFile)) {
  try {
    const g = JSON.parse(fs.readFileSync(guardFile, 'utf8'));
    if (silent) {
      if (g.updateAvailable) { process.stdout.write(g.latest + '\n'); process.exit(1); }
      process.exit(0);
    }
    // non-silent: also fast-path for humans (no banner spam)
    process.exit(0);
  } catch (_) {
    // corrupt guard file — fall through to real check
  }
}
```

After the real check completes (inside `.then(({ upToDate, installed, latest }) => {`),
write the guard file **before** the existing exit logic:

```js
// Write OS session guard (silent mode only — non-silent humans may want banner every time)
if (silent) {
  try {
    fs.writeFileSync(guardFile, JSON.stringify({
      updateAvailable: !upToDate,
      latest,
      installed,
      checkedAt: new Date().toISOString()
    }));
  } catch (_) { /* non-fatal */ }
}
```

**Note:** `fs` is already required at the top of `bin/vp-tools.cjs`. Check for existing `require('fs')` — do not add duplicate.
`path` is also already required. Only add `os` if not already required (check first).

### Change 2 — lib/viepilot-update.cjs: reduce TTL 24h → 6h

Find line:
```js
const TTL_MS = 24 * 60 * 60 * 1000; // 24 hours
```

Replace with:
```js
const TTL_MS = 6 * 60 * 60 * 1000; // 6 hours
```

## Acceptance Criteria
- [ ] `node bin/vp-tools.cjs check-update --silent` writes `/tmp/vp-update-check-{today}.done` after first call
- [ ] Second `node bin/vp-tools.cjs check-update --silent` in same OS day exits 0 with no stdout (fast-path)
- [ ] `--force` flag bypasses guard file (existing behavior preserved)
- [ ] `lib/viepilot-update.cjs` TTL_MS = 6h (not 24h)
- [ ] Guard file contains `{ updateAvailable, latest, installed, checkedAt }`

## Best Practices
- Do not duplicate `require('os')` / `require('fs')` / `require('path')` if already at top of file
- Guard file writes are try/catch non-fatal — never crash check-update on fs errors
- Keep changes < 25 lines added to `bin/vp-tools.cjs`

## Verification
```bash
rm -f /tmp/vp-update-check-$(date +%F).done
node bin/vp-tools.cjs check-update --silent; echo "first exit: $?"
node bin/vp-tools.cjs check-update --silent; echo "second exit: $?"
cat /tmp/vp-update-check-$(date +%F).done
```
Expected: second call exits in < 20ms with same exit code as first, no npm network call.

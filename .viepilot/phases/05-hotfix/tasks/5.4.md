## Objective

Fix 3 audit-identified gaps trong event naming và format của HANDOFF.log trong `workflows/autonomous.md`:

1. **control_point_enter / control_point_exit**: Thay `control_point` event bằng 2 events riêng biệt để phân biệt lifecycle
2. **task_skip JSON format**: Đảm bảo task_skip event có explicit JSON emit instruction (không chỉ reference)
3. **Recovery layer event format**: L1/L2/L3 recovery log calls dùng shorthand — cần đồng bộ với JSON format của `task_start` event

## Paths
- `workflows/autonomous.md`

## File-Level Plan

**Gap 1 — Control point events:**
Tìm đoạn control point enter protocol và thay:
```
append event `control_point` to HANDOFF.log
```
bằng:
```
append event `control_point_enter` to HANDOFF.log: {"ts":"<ISO8601>","event":"control_point_enter","task":"<T>","phase":"<P>","reason":"<reason>"}
```

Tìm đoạn control point exit (HANDOFF.json cleared) và thêm:
```
append event `control_point_exit` to HANDOFF.log: {"ts":"<ISO8601>","event":"control_point_exit","task":"<T>","phase":"<P>","resolution":"<retry|skip|rollback|stop>"}
```

**Gap 2 — task_skip event:**
Tìm Handle Result → SKIP path và thêm explicit emit:
```
append {"ts":"<ISO8601>","event":"task_skip","task":"<T>","phase":"<P>","reason":"user_skip"} to HANDOFF.log
```

**Gap 3 — Recovery layer event format:**
L1/L2/L3 recovery log calls hiện dùng pseudo-code `append_handoff_log({...})`. Chuẩn hóa thành JSON format tương tự `task_start`:
```
{"ts":"<ISO8601>","event":"l1_recovery","task":"<T>","phase":"<P>","attempt":<N>,"trigger":"<lint_error|test_fail>"}
```

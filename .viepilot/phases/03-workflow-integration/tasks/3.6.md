## NLP Intake Flow
```
Step 1: User describes request (free text)

Step 2: NLP Intake
  a. Read HANDOFF.json.control_point.active (boost BUG if true)
  b. Scan description for signal keywords
  c. Calculate confidence:
     ≥85%: auto-classify → inline confirm
           "Nhận diện: 🐛 Bug — [Enter để tiếp / 'n' để đổi]"
     60-84%: propose + 1 targeted question
           "Có vẻ là Enhancement. Đây là bug đang xảy ra không?"
     <60%: show top-3 options (not 6-item menu)
           "1. Bug  2. Feature  3. Enhancement"

Step 3 (Gap C): UI direction check
  if description contains UI keywords (UI/UX/giao diện/landing/prototype/design):
    "Nhận thấy yêu cầu liên quan đến UI/UX.
    Route qua /vp-brainstorm --ui để direction trước? (y/n)"

Step 4 (Horizon-aware): XL scope check
  if request scope > current milestone:
    read ROADMAP.md Post-MVP section
    "Yêu cầu này có vẻ thuộc Post-MVP scope.
    Thêm vào backlog Post-MVP thay vì current milestone? (y/n)"
```

## Signal Keywords
```yaml
BUG_signals:      ["lỗi", "bug", "error", "crash", "không hoạt động", "broken", "exception"]
FEATURE_signals:  ["thêm", "tính năng mới", "add feature", "new feature", "cần có"]

```

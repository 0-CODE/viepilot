## File-Level Plan

### `workflows/crystallize.md` — Step 1F modification (Gap 6)

**Current behavior:** `⚠️ Coverage gap: Feature "{name}" has no coverage in Architect OR UI Direction.` — non-blocking.

**New behavior:** Split into two cases:
```
Case A — Feature is in phases_inventory (scoped for implementation):
  → BLOCKING gate:
    "⛔ Coverage gap BLOCKED: Feature '{name}' is scoped to Phase {N} but has no design coverage.
     Must resolve before ROADMAP generation:
       (a) Add to UI Direction now → /vp-brainstorm --ui
       (b) Create implementation task with explicit 'design-TBD' note
       (c) Remove from scope with reason"
  → Step 7 is blocked until resolved

Case B — Feature is NOT in phases_inventory (out of scope):
  → Non-blocking warning only (unchanged)
  → "⚠️ Coverage gap (out-of-scope): Feature '{name}' has no coverage — currently not in phases_inventory"
```

Update the gate language in Step 1F to reflect this split.

### `workflows/crystallize.md` — Step 7 cross-check (Gap 2)

After generating all phase tasks from `phases_inventory`, add a mandatory UI completeness check:
```
## Step 7 — UI Pages → Component Map Completeness Check

After generating ROADMAP from phases_inventory:
  For each row in the UI Pages → Component Map (built in Step 1A):
    1. Check if a task exists in the generated ROADMAP that implements the target component
    2. If NO task found:
       → Add a task to the appropriate phase:
           Title: "Implement {target_component} from {prototype}"
           UI Prototype Reference: {path_to_prototype_html}
           Source: UI Pages → Component Map (auto-added)
    3. Update the component map row status: pending → assigned (Phase N, Task N.X)
  
  After cross-check: ALL rows in component map must have status != pending
  If any rows remain pending after exhausting phases_inventory: create a new phase task for them
```

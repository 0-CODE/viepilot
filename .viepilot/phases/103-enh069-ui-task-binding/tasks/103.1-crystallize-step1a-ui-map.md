### `workflows/crystallize.md` — Step 1A additions

**After the MANDATORY READ GATE section (around the existing lines that read pages/*.html), add three new sub-steps:**

#### Sub-step 1A-i: Build UI Pages → Component Map

After reading all `pages/{slug}.html` files, produce a working-notes table:

```markdown
## UI Pages → Component Map
| Prototype | Target component | Phase | Source | Status |
|-----------|-----------------|-------|--------|--------|
| pages/landing.html | resources/js/Pages/Home.vue | Phase 1 | page | pending |
| pages/dashboard.html | resources/js/Pages/Dashboard.vue | Phase 2 | page | pending |
```

Rules:
- One row per `pages/{slug}.html` file found
- Target component = stack-appropriate path (Inertia → `resources/js/Pages/{Name}.vue`; Next.js → `app/{slug}/page.tsx`; generic → `src/pages/{Name}.{ext}`)
- Phase = best estimate from pages content or phases_inventory match
- Source = `page` (direct), `arch_to_ui_sync` (Gap 5), `ux_walkthrough` (Gap 9)
- Status = `pending` initially

#### Sub-step 1A-ii: Process `## UX walkthrough log` (Gap 9)

After building the component map, scan `notes.md → ## UX walkthrough log`:
- For each entry with severity `P0`: add a UX-fix row to the component map with `Source: ux_walkthrough`, mark as `ux-fix-required`; generate a task description `fix-ux-p0: {pain description} on {page}.html`
- For each entry with severity `P1`: add to component map with `Source: ux_walkthrough`, mark as `ux-fix-p1`; generate task description
- `P2` entries: list as warnings only (non-blocking, do not add to component map)
- If `## UX walkthrough log` is absent or empty: no action

#### Sub-step 1A-iii: `## Background extracted ideas` resolution gate (Gap 10)

After UX walkthrough processing, scan `notes.md → ## Background extracted ideas`:
- For each idea entry, check if it maps to an existing row in the UI Pages → Component Map or phases_inventory
- If NOT covered: add to a `## Unresolved Background Ideas` gate list
- Before Step 7 runs (ROADMAP generation), this gate must be cleared:

  - Option (b): add as implementation task with `design-TBD` note
  - Option (c): descope with reason logged in working notes
- Record resolutions in working notes

### `workflows/autonomous.md` — UI Coverage Gate at phase completion (Gap 4)

Add a new gate block before the existing phase PASS check:

```
#### UI Coverage Gate (ENH-069)

Before marking phase PASS:
  1. Read PROJECT-CONTEXT.md ## UI Pages → Component Map
  2. Filter rows where Phase = current phase AND Source ≠ arch_to_ui_sync (i.e. has a real prototype)
  3. For each filtered row:
     a. Check if target component file exists
     b. If file exists: check for stub heuristic:
          - File length < 20 lines AND
          - Contains only routing/title text (no real layout/components)
          → STUB detected
     c. If file missing OR stub detected:
          → ⚠️ UI Coverage Warning: "{prototype}" → "{component}" is still a stub
          → List all unimplemented pages
          → Options:
              (a) Implement now (recommended — continue phase)
              (b) Defer to next phase (update component map status + log reason)
              (c) Mark as design-only (no implementation required — remove from gate)
  4. Phase cannot receive ✅ PASS until all warnings are resolved (Option b or c clears the warning)
```

## Implementation Notes
- `PROJECT-CONTEXT.md` is written by crystallize — it will receive a new `## UI Pages → Component Map` section (crystallize Step 7 output). This is the single source of truth at runtime.
- Stub heuristic thresholds (20 lines, routing/title text) are conservative — they can be tightened later.
- Gap 4 only warns; it does not auto-fail. The gate offers deferral/design-only to unblock if needed.

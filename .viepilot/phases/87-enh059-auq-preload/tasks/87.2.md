# Task 87.2 — Add AUQ preload step to 3 workflow files

## Objective
Add a `ToolSearch` preload step at the top of each workflow's process section so that when a workflow runs on Claude Code adapter, AUQ is guaranteed to be schema-loaded before any interactive prompt is attempted.

## Paths
- `workflows/request.md`
- `workflows/evolve.md`
- `workflows/autonomous.md`

## Implementation Notes

### Standard preload step to insert

In each workflow, find the `<process>` opening tag (or the first `## 1.` / `## Step 1` heading inside the process section). Insert the following block **immediately after** the opening tag or before the first step:

```markdown
**AUQ preload — Claude Code adapter:**
At session start, before any interactive prompt:
```
ToolSearch { query: "select:AskUserQuestion" }
```
This loads the deferred `AskUserQuestion` schema. Required on Claude Code (terminal).
Skip only if ToolSearch returns an error → use text fallback for that session.
```

### Per-file guidance

#### `workflows/request.md`
- Location: after `<process>` tag, before Step 1 "Detect Request Type"
- AUQ is used in Step 1 (type detection), Step 2B (feasibility gate), Step 7 (continuation)

#### `workflows/evolve.md`
- Location: after `<process>` tag or before Step 2 "Ask User Intent"
- AUQ is used in Step 2 (mode selection), Step 3A (complexity/brainstorm), Step 5 (continuation)

#### `workflows/autonomous.md`
- Location: at the top of the process section or before Step 1 "Initialize"
- AUQ is used at control points (blockers, conflict resolution, quality gate failures)

## Acceptance Criteria
- [ ] `workflows/request.md` — ToolSearch preload step present before Step 1
- [ ] `workflows/evolve.md` — ToolSearch preload step present before Step 2
- [ ] `workflows/autonomous.md` — ToolSearch preload step present in init/process
- [ ] Each file retains existing text fallback guidance
- [ ] `grep -l "select:AskUserQuestion" workflows/*.md | wc -l` ≥ 3

## Sections to update

### 1. Session type / mode question

wrap with adapter-aware pattern:
- AUQ spec: question "What type of brainstorm session?" with options like:
  - New Feature (describe and explore a new capability)
  - Improvement (deepen an existing feature)
  - Architecture (explore structural changes)

- multiSelect: false

### 2. Topic gathering question
The initial "What do you want to explore?" prompt:
- AUQ spec: open-ended — no structured options (use free-text note field)
- Since AUQ requires 2–4 options, either:
  - Skip AUQ for fully free-text questions (keep as text prompt)
  - OR use AUQ with broad category options + "Other" automatic option
- Decision: **use AUQ only when ≥2 meaningful constrained options exist**;
  purely open-ended questions remain as text prompts

### 3. Goals/scope question
After topic is set, brainstorm typically asks about scope:
- AUQ spec: question "What scope for this brainstorm?" with options:
  - Quick (15 min — key decisions only)
  - Standard (30 min — full exploration)
  - Deep Dive (60 min — architecture + edge cases)
- multiSelect: false

### 4. Related feature question
"Is this related to existing feature or new area?" — typical continuation question:
- AUQ spec: 3 options (Extend existing / Completely new / Not sure yet)
- multiSelect: false

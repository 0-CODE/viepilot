# Task 79.1 — workflows/audit.md: Tier 4 Silent Patches

## Objective
Apply 3 targeted edits to `workflows/audit.md` so Tier 4 output is completely hidden when the check is skipped or passes cleanly. Output only surfaces when `TIER4_ISSUES > 0`.

## Paths
- `workflows/audit.md`

## File-Level Plan

### Edit 1 — Step 4 skip block: remove echo
**Location**: `## 4. Tier 4: Framework Integrity` section, the `if IS_VIEPILOT_FRAMEWORK != true` guard block.

**Current:**
```bash
if [ "$IS_VIEPILOT_FRAMEWORK" != "true" ]; then
  echo "→ Tier 4 skipped (not a viepilot framework repo)"
  # Jump to step 5 (report)
fi
```

**Change to:**
```bash
if [ "$IS_VIEPILOT_FRAMEWORK" != "true" ]; then
  # Tier 4 skipped silently — not a viepilot framework repo
fi
```

### Edit 2 — Step 4f: wrap Tier 4 report block in conditional
**Location**: `### 4f. Report Tier 4 results` block.

Wrap the entire `TIER 4: Framework Integrity` report output in a guard:
```
if [ "$TIER4_ISSUES" -gt 0 ]; then
  # ... existing Tier 4 report block ...
fi
```
The `TIER4_ISSUES` variable must already be set/incremented in steps 4a–4e whenever a gap/mismatch is found (it is, per existing workflow logic — just needs the guard here).

### Edit 3 — Step 5 All Clear banner: remove Tier 4 line
**Location**: `### All Clear` → the summary banner block.

**Remove this line:**
```
 Tier 4: Framework          {✅ In sync | ℹ️ Skipped}
```

### Edit 4 — Step 5 Issues banner: Tier 4 line conditional
**Location**: `### Issues Found` → the summary banner block.

The Tier 4 line in the issues banner should only render when `TIER4_ISSUES > 0`:
```
{if TIER4_ISSUES > 0}
 Tier 4: Framework          ⚠️ {N} issues
{/if}
```
(Keep the existing markdown conditional comment pattern used elsewhere in the file.)

## Implementation Notes
- Do NOT change the actual Tier 4 check logic (4a–4e) — only change what gets printed.
- The `ℹ️ Skipped` state in the All Clear banner is removed entirely (silent skip — no message at all).
- Pattern is consistent with ENH-047 gap detection: gap tiers only appear in output when non-zero.

# Task 93.1 — Update `workflows/crystallize.md` — skill decision gate

## Objective
Add skill gate step to the crystallize workflow: read `## skills_used` from brainstorm notes.md → AUQ confirm required/optional/exclude → write `## Skills` to `PROJECT-CONTEXT.md`.

## Paths
- `workflows/crystallize.md`

## Implementation Notes

### Where to insert

After **scope lock** (Step 1 / Step 1A) and **before** SPEC generation (Step 2). New step: `### Step 1E — Skill Decision Gate (FEAT-020)`.

### Content to add

```markdown
### Step 1E — Skill Decision Gate (FEAT-020)

**Trigger**: `## skills_used` section exists in the brainstorm session's `notes.md`.
**Skip condition**: If no `## skills_used` found → silently skip this step (no error, no prompt).

**Step — Load skills_used:**
Read `## skills_used` from `.viepilot/ui-direction/{session-id}/notes.md` (or `.viepilot/architect/{session-id}/notes.md`).

**Step — AUQ confirm (Claude Code adapter):**

> **Claude Code (terminal) — REQUIRED:** Call `AskUserQuestion` tool.
> AUQ spec:
>   - question: "The following skills were used in this brainstorm. Mark each as required, optional, or exclude:"
>   - For each skill: show id + capabilities + best_practices[]
>   - multiSelect: false per skill (required | optional | exclude)

> **Text fallback (other adapters):** Present numbered list with required/optional/exclude options.

**Step — Write `## Skills` to `PROJECT-CONTEXT.md`:**

```markdown
## Skills

| Skill | Source | Required | Phases | Rationale |
|-------|--------|----------|--------|-----------|
| frontend-design | npm:@vp-skills/frontend-design | required | 1, 2 | UI-Direction HTML generation — design token best practices |
```

- `Required`: skill best_practices injected by vp-auto for matching phases — **no re-asking**
- `Optional`: skill context available but not auto-injected
- `Exclude`: skill not used; entry omitted from PROJECT-CONTEXT.md

**Lock semantics**: once written, this section is the authoritative skill decision for the project. `vp-auto` reads it and never re-prompts.
```

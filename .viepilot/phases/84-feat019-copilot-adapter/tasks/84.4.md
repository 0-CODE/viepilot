```markdown
## GitHub Copilot

**Status**: ✅ Supported (FEAT-019, v2.21.0)

### Supported Surfaces

| Surface | Status | Notes |
|---------|--------|-------|
| VS Code Copilot Chat | ✅ Supported | Type `/vp-status` in Chat panel |
| GitHub Copilot CLI | ✅ Supported | via `gh copilot` command |
| GitHub.com cloud agent | ⚠️ Preview | Third-party custom agents in preview (April 2026) |
| JetBrains Copilot | ⚠️ Pending | IDE parity not yet available |

### Installation

```bash
npx viepilot install --adapter copilot
```

Skills are installed to: `~/.config/gh-copilot/skills/`
Framework files: `~/.config/gh-copilot/viepilot/`

### Prerequisites
- [GitHub CLI (`gh`)](https://cli.github.com/) installed
- GitHub Copilot subscription active
- VS Code with GitHub Copilot extension (for Chat surface)

### Usage
In VS Code Copilot Chat, type:
```
/vp-status
```

### Limitations
- `AskUserQuestion` interactive prompts not supported in Copilot Chat → text fallback menus
- MCP server registration (ViePilot tools-as-MCP) planned for Phase 2
- Cloud agent custom agents still in preview — may change

### Config Directory
Copilot uses the standard `gh` CLI config location:
```
~/.config/gh-copilot/
├── skills/          ← vp-* SKILL.md files
└── viepilot/        ← framework workflows, templates, agents
```

On Windows: `%APPDATA%\gh-copilot\` (auto-detected by installer)
```

# BUG: brainstorm không hỏi user về cả Architect Design và UI Direction Mode — thiếu unified mode-selection prompt

- **Reporter**: User
- **Assignee**: AI

## Summary
Brainstorm tự động activate Architect Design Mode khi heuristic fire (≥3 components hoặc ≥1 stack mention) và KHÔNG hỏi user về UI Direction Mode. UI Direction chỉ surface thụ động (background accumulation). User không được offer cả hai modes cùng một lúc và không biết UI Direction là một option song song.

## Details

### Actual behavior
1. User mở brainstorm session mới.
2. Session detect ≥1 stack mention (e.g. React, Node.js) → 🏗️ Architect banner fires → hỏi "Yes/No activate Architect?"
3. User chọn Yes → Architect workspace được tạo.
4. **UI Direction KHÔNG được offer** tại bước này.
5. UI Direction chỉ surface nếu: (a) initial message có UI keyword → 🎨 banner; (b) ≥2 unique UI signals accumulated + /save hoặc /topic change. Nếu user chưa dùng UI keyword → không thấy option nào.
6. User phải tự biết dùng `--ui` flag hoặc chờ passive surface.

### Expected behavior
Sau khi Architect Mode được confirm (hoặc ngay khi session start nếu project scope rõ), brainstorm nên:
- **Option A**: Hỏi upfront "Which modes do you want to activate?" với multi-select: Architect / UI Direction / Both / Neither
- **Option B**: Sau khi Architect confirmed → immediately offer "Also activate UI Direction Mode?" (vì ENH-061 defines them as a pair)

### Root Cause
Hai modes có activation triggers **hoàn toàn độc lập**:
- **Architect**: heuristic-based auto-prompt (≥3 components OR ≥1 stack mention)
- **UI Direction**: keyword-based passive (background accumulation, early-session scan)

Không có **unified mode-selection prompt** nào kết nối cả hai. ENH-061 `Recommended Breakdown Ordering` (Step 4 Architect → Step 6 UI Direction) định nghĩa flow nhưng không có enforcement tại session start.

### Files affected
- `workflows/brainstorm.md` — Architect activation (line ~301), Background UI Extraction (line ~510), Recommended Breakdown Ordering (line ~246)
- `skills/vp-brainstorm/SKILL.md` — objective description

### Fix options

#### Fix A — Unified mode prompt after scope lock (Recommended)
Sau khi `## Phases` được set (scope lock), trước khi bắt đầu detail work, thêm AUQ step:
```
question: "Which design workspaces would you like to activate for this session?"
options:
  - label: "Architect Design only (🏗️)"
    description: "System architecture, data-flow, ERD, APIs HTML workspace"
  - label: "UI Direction only (🎨)"
    description: "HTML prototype direction for screens/pages"
  - label: "Both Architect + UI Direction (Recommended)"
    description: "Full coverage: architecture + UI per ENH-061 Breakdown Ordering"
  - label: "Neither — text-only"
    description: "Continue as text brainstorm, activate workspaces manually later"
multiSelect: false
```

#### Fix B — Post-Architect offer (simpler)
Sau khi Architect Mode confirmed (user chọn "Yes" tại 🏗️ banner), immediately thêm:
```
Great, Architect workspace created. Would you also like to activate UI Direction Mode?
1. Yes — create UI Direction workspace alongside architect
2. Later — I'll use /vp-brainstorm --ui when ready
3. No — architect only for this session
```

Fix A là recommended vì nó cũng handles trường hợp user muốn UI Direction mà không cần Architect.

## Summary
When any `vp-*` skill is invoked, check if a newer version of the ViePilot framework is available. If yes, display a non-blocking notice banner immediately after the greeting banner so the user knows an update is available and how to upgrade. The check is cached (max once per 24h) to avoid latency on every skill call.

## Current Behavior
1. User calls `/vp-auto`, `/vp-brainstorm`, etc.
2. Greeting banner shows installed version: `VIEPILOT ► VP-AUTO v0.2.2 (fw 2.40.0)`
3. No check against latest available version
4. User has no way to know a new version was released

## Desired Behavior
1. User calls any `vp-*` skill
2. Greeting banner shown (same as today)
3. After greeting: skill calls `node {envToolDir}/bin/vp-tools.cjs check-update --silent`
   - If up to date: silent, no output
   - If new version available: display notice banner:
     ```
     ┌──────────────────────────────────────────────────────────────────┐
     │ ✨ ViePilot v{latest} available  (installed: v{current})         │
     │    npm i -g viepilot && vp-tools install --target {adapter}      │
     └──────────────────────────────────────────────────────────────────┘
     ```
4. Skill continues normally (banner is non-blocking)

## Detailed Design

### `lib/viepilot-update.cjs` — `checkLatestVersion()`

New exported function:

```js
async function checkLatestVersion(opts = {}) {
  // 1. Read installed version from package.json in envToolDir
  // 2. Check cache: ~/.viepilot/update-cache.json
  //    If cache.checked_at is within 24h → return cached result
  // 3. Fetch latest: https://registry.npmjs.org/viepilot/latest
  //    On network error: return { upToDate: true } (fail silently)
  // 4. Compare installed vs latest (semver)
  // 5. Write cache: { checked_at, installed, latest, has_update }
  // 6. Return { upToDate, installed, latest }
}
```

Cache file: `~/.viepilot/update-cache.json`
```json
{
  "checked_at": "2026-04-23T10:00:00Z",
  "installed": "2.40.0",
  "latest": "2.41.0",
  "has_update": true
}
```

TTL: 24 hours. If cache is fresh → skip network call, return cached result.

Failure modes (all silent, return `{ upToDate: true }`):
- Network unavailable
- npm registry timeout (>3s)
- Malformed registry response
- Cache file unreadable

### `bin/vp-tools.cjs` — `check-update` subcommand

```
vp-tools check-update [--silent] [--json]

Options:
  --silent    Exit 0 if up-to-date (no output). Exit 1 + print new version if update available.
  --json      Print JSON: { installed, latest, has_update }
  --force     Bypass 24h cache and check now

Exit codes:
  0  — up to date (or check failed silently)
  1  — update available (latest version printed to stdout)
```

## Discussion
Currently `vp-tools` has a `viepilot-update.cjs` module in `lib/` — this enhancement extends it with an actual check-latest-version function using the npm registry, with a 24h cache so the check is invisible to the user in day-to-day use but surfaces a notice on the first call after a new release. The banner is intentionally minimal and non-interactive — it does not ask the user to update now, just informs them. The `{adapter_id}` in the update command resolves to the current adapter (claude-code, cursor, antigravity, etc.) at invocation time.

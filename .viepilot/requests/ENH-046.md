## Summary
`workflows/documentation.md` hardcodes GitHub-specific slug parsing (`github.com` regex, `GITHUB_OWNER`/`GITHUB_REPO` variables). Additionally `workflows/crystallize.md` Step 0 prompts for "GitHub username". Most other workflows are already platform-agnostic (they use plain `git push`/`git remote get-url`). This enhancement generalizes the affected workflows to support GitHub, GitLab, Bitbucket, Azure DevOps, Gitea, and self-hosted Git remotes.

## Details

### Current Behavior

**`workflows/documentation.md` (problematic section):**
```bash
GITHUB_SLUG=$(echo "$REMOTE_URL" | sed 's|.*github\.com[:/]||; s|\.git$||')
GITHUB_OWNER=$(echo "$GITHUB_SLUG" | cut -d'/' -f1)
GITHUB_REPO=$(echo "$GITHUB_SLUG" | cut -d'/' -f2)
```
- Regex anchors to `github.com` — fails silently for GitLab, Bitbucket, etc.
- Variable names `GITHUB_OWNER` / `GITHUB_REPO` are forge-specific and confusing.

**`workflows/crystallize.md` Step 0 prompt:**
- Asks user for "GitHub username? (optional)" — GitHub-specific wording.

### Desired Behavior

**`workflows/documentation.md`:**
1. Parse remote URL with a forge-agnostic extractor that handles:
   - `https://github.com/owner/repo.git`
   - `git@github.com:owner/repo.git`
   - `https://gitlab.com/owner/repo.git`
   - `git@gitlab.com:owner/repo.git`
   - `https://bitbucket.org/owner/repo.git`
   - `https://dev.azure.com/org/project/_git/repo`
   - `https://gitea.example.com/owner/repo.git`
   - Self-hosted patterns (any `host:owner/repo` SSH or `https://host/owner/repo`)
2. Extract into neutral variables: `GIT_HOST`, `GIT_OWNER`, `GIT_REPO`
3. Replace all references to `GITHUB_OWNER`/`GITHUB_REPO` with the new names.
4. For forge-specific features (e.g., GitHub Pages, GitHub Actions links), conditionally branch on `GIT_HOST` or degrade gracefully.

**`workflows/crystallize.md`:**
1. Change "GitHub username? (optional)" label in Step 0 → "Git remote host / username (optional, e.g. github.com/johndoe)"
2. Update any `github_username` field in the resulting session stub to `git_remote_info`.

# BUG: `/vp-auto` does not reliably enforce final commit/push, causing missing commits

## Meta
- **ID**: BUG-003
- **Type**: Bug
- **Status**: triaged
- **Priority**: high
- **Created**: 2026-03-31
- **Reporter**: User
- **Assignee**: AI

## Summary
Khi chạy `/vp-auto`, bước commit/push cuối không luôn được enforce nhất quán nên có tình huống task/phase đã báo complete nhưng git history thiếu commit hoặc chưa push.

## Details

### Actual behavior
- Một số lượt `/vp-auto` có commit/push đầy đủ, một số lượt lại bỏ sót.
- Dẫn đến trạng thái docs/state có thể cập nhật nhưng git remote chưa phản ánh đủ thay đổi.

### Expected behavior
- Mỗi task/phase PASS phải có rule rõ ràng và nhất quán:
  1. enforce `git add` + `git commit` theo logical unit
  2. enforce push theo policy đã định (task-level hoặc phase-level)
  3. fail fast/cảnh báo nếu commit hoặc push không thành công
- Không được mark complete khi chưa đạt git safety gate.

### Steps to reproduce
1. Chạy `/vp-auto` qua nhiều task liên tiếp.
2. So sánh trạng thái `PHASE-STATE/TRACKER` với `git log` và remote branch.
3. Quan sát có trường hợp thiếu commit hoặc chưa push dù flow đã tiến tiếp.

### Affected area
- `workflows/autonomous.md`
- `skills/vp-auto/SKILL.md`
- `bin/vp-tools.cjs` (nếu dùng helper commit/push checks)
- tài liệu user/dev liên quan execution guarantees

### Severity
- high

## Acceptance Criteria

- [ ] Nếu commit/push fail, flow dừng ở control point (không mark done).
- [ ] State update logic nhất quán với git status thực tế.
- [ ] Docs mô tả rõ chính sách commit/push trong `/vp-auto`.

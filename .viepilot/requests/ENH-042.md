## Summary
Slides hiện tại thiếu hình ảnh minh hoạ, trông khô khan. Đề xuất tự động nhúng ảnh chụp màn hình (screenshot) từ các HTML artifact đã có trong session — cụ thể là ui-direction prototype (`index.html`, `pages/*.html`) và architect workspace (`architecture.html`, `erd.html`, `sequence-diagram.html`, v.v.) — vào đúng slide tương ứng trong file `.pptx`.

## Details

### Current Behavior
- `gen-proposal-pptx.cjs` không có `addImage()` call nào.
- Slides chỉ có text: heading + bullets + speaker notes.
- Không có visual nào giúp khán giả hình dung sản phẩm/kiến trúc.

### Desired Improvement
Khi `/vp-proposal` phát hiện session đã có artifact HTML (ui-direction hoặc architect), workflow:

1. **Screenshot các trang HTML liên quan** → lưu tạm thành `.png`
2. **Nhúng ảnh vào slide phù hợp**:
   - UI mockup slide → screenshot `index.html` (ui-direction)
   - Architecture slide → screenshot `architecture.html`
   - ERD slide → screenshot `erd.html`
   - Sequence slide → screenshot `sequence-diagram.html`
3. **Fallback** nếu không có artifact HTML hoặc không cài puppeteer: styled placeholder box (màu accent, text "Add screenshot of UI here")

### Proposed Technical Approach

#### Screenshot tool options (ranked by simplicity):
| Option | Dep | Zero-install? | Quality |
|--------|-----|---------------|---------|
| `puppeteer` (headless Chrome) | npm optional | No | Best |
| `playwright` (headless) | npm optional | No | Best |
| `pageres` / `capture-website` | npm optional | No | Good |
| **Placeholder box** (pptxgenjs shape) | none | Yes | N/A — fallback |

#### Recommended approach:
- `puppeteer` hoặc `playwright` là **optional peer dep** — vp-proposal tự detect: `try { require('puppeteer') } catch { fallback }`
- Nếu có: chụp HTML artifact → temp `.png` → `addImage({ path: tmpPng, ... })` vào slide
- Nếu không: thêm styled shape placeholder vào vị trí ảnh trên slide
- Workflow Step 4c (mới): `detect_visual_artifacts` — scan `.viepilot/ui-direction/{session}/` và `.viepilot/ui-direction/{session}/pages/` + architect workspace

#### Slide → Image mapping (AI xác định trong manifest):
```json
{
  "visualSlides": [
    { "slideIndex": 3, "artifactType": "ui-direction", "page": "index" },
    { "slideIndex": 6, "artifactType": "architect", "page": "architecture" },
    { "slideIndex": 7, "artifactType": "architect", "page": "erd" }
  ]
}
```

### Why Valuable
- Khán giả (client/partner) hình dung được sản phẩm ngay trong slide thay vì đọc chữ
- Tái dụng artifact đã có sẵn từ vp-brainstorm → zero extra work từ user
- Tăng credibility của proposal pack (professional, visual-first)

### Breaking Changes
- Không — fallback đảm bảo backward compatible khi không cài puppeteer

## Summary
`vp-tools scan-skills` is referenced in FEAT-020 skill registry documentation across SKILL.md files but the command was never implemented in `bin/vp-tools.cjs`. Invoking it results in "Unknown command: scan-skills".

## Details

### Actual Behavior
```
$ node bin/vp-tools.cjs scan-skills
✖ Error: Unknown command: "scan-skills"
  Hint: Available commands: init, current-timestamp, phase-info, task-status, commit,
  progress, version, reset, clean, checkpoints, tag-prefix, git-persistence, info,
  update, conflicts, save-state, hooks, config, get-registry, help
```

### Expected Behavior
`vp-tools scan-skills` scans installed skills at `~/.claude/skills/vp-*/SKILL.md` (and `~/.cursor/skills/vp-*/SKILL.md`), extracts each skill's capabilities and best practices, and writes `~/.viepilot/skill-registry.json` with structured entries. Output:
```
✔ Scanned 8 skills → ~/.viepilot/skill-registry.json
```

### Affected Files
- `bin/vp-tools.cjs` — add `scan-skills` subcommand routing
- `lib/` — implementation module `viepilot-scan-skills.cjs` (or inline in vp-tools)
- `skills/vp-brainstorm/SKILL.md` — documents `vp-tools scan-skills` (FEAT-020 registry section)
- `skills/vp-crystallize/SKILL.md` — documents `vp-tools scan-skills` (FEAT-020 registry section)

## Summary

Claude Code `plan` permission mode blocks all tool execution — AI can read and plan but not write/execute. This maps perfectly to ViePilot's doc-first gate (BUG-001): before any implementation commit, the AI should review the task contract in a non-destructive read-only pass. Currently doc-first gate is enforced by workflow instruction only; Plan mode provides a structural enforcement mechanism.

## Source

Reference: `for-learn-claude-cli-main/reports/REPORT-permission-system.md` (plan mode), `REPORT-lessons-learned.md` (Bài học 14).

## Mechanism

```
plan mode: Claude sees tool schemas but CANNOT execute
  → all tool invocations return 'ask' (blocked)
  → Claude adjusts plan based on rejections
  → user reviews plan → switches to default/acceptEdits → execution begins
```

Plan mode can be set via:
- Claude Code settings (`permissionMode: "plan"`)
- `--plan` flag at invocation
- `/plan` slash command (if registered)

## Proposed Fix

### autonomous.md — Explicit plan mode guidance

In the doc-first gate section, add guidance:

```
#### Doc-first gate enforcement options:

Option A (recommended): Before starting task implementation, switch to Plan mode:
  → Claude can read all files, fill task contract, validate Paths + File-Level Plan
  → No commits possible until you switch back to default/acceptEdits mode
  → Switch: Settings → Plan mode ON → review task file → switch OFF → /vp-auto continues

Option B (existing): Doc-first checklist enforced by workflow instruction (current behavior)
  → Lower structural guarantee, higher flexibility
```

### docs/user/features/autonomous-mode.md

Add section: "Using Plan Mode for Task Contract Review":
- When to use: before L/XL tasks, first time running a new phase, when task touches compliance paths
- How to activate in Claude Code
- What to review in plan mode: Paths section, File-Level Plan, write_scope

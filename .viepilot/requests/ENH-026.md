## Problem

### 1. vp-crystallize bỏ qua ui-direction brainstorm
`/vp-crystallize` hiện đọc `ui-direction` artifacts (Step 1A) nhưng **chỉ xử lý khi artifacts đã tồn tại**. Nếu user chưa chủ động chạy `--ui` hoặc chưa sinh HTML, crystallize tiếp tục mà không cảnh báo nghiêm túc và bỏ mất toàn bộ vision UI/UX của user. Quy trình ui-direction vì thế bị **opt-in thủ công** thay vì được nhúng vào luồng mặc định.

### 2. vp-brainstorm UI mode thiếu background extraction
Hiện tại, UI Direction Mode (`workflows/brainstorm.md`) chỉ kích hoạt khi user **chủ động** chuyển sang `--ui` hoặc đề cập UI/UX. Khi user thảo luận tự nhiên (không dùng flag), các ý tưởng UI ẩn trong hội thoại **không được trích xuất** và không cập nhật `notes.md` / `ui-direction` artifacts.

## Goal

### 2.1 Background UI Idea Extraction (brainstorm.md)
Thêm quy trình ngầm trong mọi chế độ brainstorm:
- **Signal detection**: nhận diện các tín hiệu UI/UX từ hội thoại (màu sắc, layout, tính năng có giao diện, luồng người dùng, mobile/web, v.v.)
- **Silent accumulation**: gom các ý tưởng này vào buffer tạm trong session
- **Periodic surfacing**: định kỳ (hoặc khi chốt topic) hỏi user xác nhận / làm rõ các ý tưởng UI đã phát hiện
- **Auto-write to ui-direction**: khi có đủ signal, tự động tạo / cập nhật `notes.md` và gợi ý user có muốn tạo HTML direction không

### 2.2 Crystallize UI Direction Gate (crystallize.md)
Tăng cường Step 1A thành **hard gate** khi dự án có UI scope:
- Nếu brainstorm session chứa UI signal nhưng **không có** `ui-direction` artifacts → dừng, hỏi user muốn:
  1. Quay lại `/vp-brainstorm --ui` để tạo direction trước
  2. Tiếp tục với assumptions (ghi rõ limitations)
- Nếu `notes.md` tồn tại nhưng không có `## Pages inventory` và scope là multi-page → cảnh báo tương tự
- Output crystallize phải liệt kê **tất cả pages** từ inventory (hoặc explicit "single-page / no UI artifacts")

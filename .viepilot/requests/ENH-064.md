# ENH: Unified HUB for Architect + UI Direction workspaces + Crystallize mandatory read gates

- **ID**: ENH-064
- **Type**: Enhancement
- **Status**: new
- **Priority**: high
- **Created**: 2026-04-21
- **Reporter**: User
- **Assignee**: AI

## Summary
Hai workspaces (`.viepilot/architect/{session-id}/` và `.viepilot/ui-direction/{session-id}/`) hiện tại là 2 folder riêng biệt không có cross-link. Crystallize có hard gate cho UI Direction khi artifacts vắng, nhưng **không có hard gate khi architect workspace TỒN TẠI** — có thể bị đọc sơ hoặc bỏ qua âm thầm. Cần: (1) unified HUB index.html nối cả hai, (2) crystallize mandatory read gate bắt buộc đọc hết từ A-Z khi workspace tồn tại.

## Details

### Current state

**Directory structure (correct — already separate):**
```
.viepilot/
  architect/{session-id}/
    index.html              # Architect HUB (12 pages)
    architecture.html
    data-flow.html
    decisions.html
    tech-stack.html
    tech-notes.html
    feature-map.html
    erd.html
    user-use-cases.html
    sequence-diagram.html
    deployment.html
    apis.html
    style.css
    notes.md                # YAML: decisions[], tech_stack{}, open_questions[], erd, use_cases, apis, deployment
  ui-direction/{session-id}/
    index.html              # UI HUB (multi-page)
    pages/*.html
    style.css
    notes.md                # YAML: ui_idea_buffer[], background extracted ideas, arch_to_ui_sync, Pages inventory
```

**Problem 1 — No cross-workspace navigation:**
- User must open `.viepilot/architect/` and `.viepilot/ui-direction/` separately
- No single entry point to see both workspaces at a glance
- arch_to_ui_sync links exist in notes.md but no HTML cross-link

**Problem 2 — Crystallize mandatory read gaps:**
- `Step 1A` (UI Direction): has hard gate when `ui_scope_detected = true` + artifacts MISSING → STOP
- `Step 1D` (Architect): reads if `.viepilot/architect/` exists — but NO hard gate when workspace EXISTS → could skim or skip files silently
- No enforcement rule: "if architect workspace exists → MUST read ALL files from A-Z before proceeding"
- No enforcement rule: "if ui-direction workspace exists → MUST read ALL notes.md + all pages/*.html"
- Risk: crystallize reads only `notes.md` (decisions section) and skips `feature-map.html`, `tech-notes.html`, `arch_to_ui_sync` entries, etc.

### Proposed fix

#### A — Unified cross-workspace HUB (`viepilot-session-hub.html`)

When BOTH architect AND ui-direction workspaces exist for the same session:
create `.viepilot/session-hub/{session-id}/index.html` as a unified dashboard:

```html
<!-- viepilot-session-hub — links to both workspaces -->
<!-- Left panel: Architect workspace (12 pages) -->
<!-- Right panel: UI Direction workspace (N pages) -->
<!-- Center: Coverage matrix table (Feature | Architect page | UI screen) from ## Coverage in notes.md -->
<!-- Bottom: arch_to_ui_sync log (decisions with UI implications) -->
```

OR simpler: add cross-links to each workspace's existing `index.html`:
- Architect `index.html` → add "🎨 View UI Direction workspace" link
- UI Direction `index.html` → add "🏗️ View Architect workspace" link

**When to create/update cross-links:**
- When arch_to_ui_sync runs (ENH-061 step 5)
- When both workspaces are active in the same session

#### B — Crystallize mandatory read gate for existing architect workspace

**New rule in Step 1D (after "If `.viepilot/architect/` exists"):**

```
MANDATORY GATE — if architect workspace EXISTS:
1. Read notes.md front-to-back (ALL sections, not just decisions[])
   Sections to capture: decisions[], tech_stack{}, open_questions[], erd, use_cases, apis, deployment,
   feature_map, arch_to_ui_sync[], coverage[], skills_used[]
2. Read index.html → confirm all 12 linked pages exist on disk
3. For each page that EXISTS on disk:
   Read it and extract key content:
   - architecture.html → components, C4 diagram intent, external integrations
   - data-flow.html → high-level flows, async indicators
   - decisions.html → ADR entries not yet in notes.md
   - tech-stack.html → layer-by-layer stack (authoritative)
   - tech-notes.html → assumptions, risks, open questions
   - feature-map.html → features with phase, priority, status tags
   - erd.html → entities, relationships (if not in notes.md)
   - user-use-cases.html → actors, use cases (if not in notes.md)
   - sequence-diagram.html → (skip — not an architecture artifact per design)
   - deployment.html → infra, environments (if not in notes.md)
   - apis.html → endpoints, service contracts (if not in notes.md)
   - admin.html → admin capabilities (if exists — ENH-063)
4. STOP if notes.md is missing or unreadable → ask user to fix architect artifacts
5. Record: "architect_read_complete: true" in crystallize working notes before proceeding
```

**No silent skip allowed**: if `.viepilot/architect/` exists but ANY mandatory read fails → surface error + pause.

#### C — Crystallize mandatory read gate for existing UI Direction workspace (strengthen existing)

**Strengthen existing Step 1A rule:**

Current: hard gate fires only when `ui_scope_detected = true` AND artifacts MISSING.
Problem: if artifacts EXIST, gate passes — but crystallize may only read `notes.md` partially.

**New rule when `.viepilot/ui-direction/` EXISTS:**

```
MANDATORY GATE — if ui-direction workspace EXISTS:
1. Read notes.md front-to-back (ALL sections):
   Sections to capture: ui_idea_buffer[], background extracted ideas,
   Pages inventory, arch_to_ui_sync[], ## skills_used[], coverage[]
2. If pages/ directory exists:
   Read ALL pages/*.html one by one — do NOT skip any page
   For each page: extract layout intent, components, interactions, annotations
3. Read index.html (hub) → verify nav links match pages/ contents
4. Verify Pages inventory in notes.md matches actual pages/*.html set
   If mismatch → STOP and ask user to run /vp-brainstorm --ui to sync
5. Record: "ui_direction_read_complete: true" in crystallize working notes
```

#### D — Cross-reference check before crystallize proceeds

After Step 1A + Step 1D both complete:

```
Cross-reference gate (if BOTH workspaces present):
- Read ## Coverage from architect notes.md (or ui-direction notes.md)
- For each Phase 1 feature in coverage matrix:
  - Confirm architect page exists and was read
  - Confirm UI screen page exists and was read
- Features with "none yet" in both columns → surface warning (non-blocking):
  "⚠️ Feature '{name}' has no coverage in Architect OR UI Direction.
   Consider adding before /vp-crystallize."
- Record: "coverage_check_complete: true"
```

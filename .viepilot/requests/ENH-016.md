## Summary
Làm rõ và (nếu cần) chuẩn hóa hành vi khi user **đã cài bản cũ** rồi chạy lại installer: có xung đột không, file cũ có bị xóa không, khác nhau giữa `install.sh` và `dev-install.sh`.

## Details (hiện trạng đã rà soát)

| Luồng | Trước khi cài mới | Ghi chú |
|--------|-------------------|---------|
| **`dev-install.sh`** | Xóa toàn bộ `~/.cursor/skills/vp-*` và **`rm -rf ~/.cursor/viepilot`** | Không có merge conflict; bản cũ bị thay hoàn toàn. |
| **`install.sh`** | **Không** xóa skills / viepilot trước; dùng `cp -r` đè lên | File trùng tên được ghi đè; **file chỉ có ở bản cũ** (skill đổi tên / bỏ file) có thể **còn sót** trong thư mục copy. Workflows/templates/ui-components: `cp -r` tương tự — orphan ít gặp hơn nhưng vẫn có thể. |
| **Symlink skills** (`VIEPILOT_SYMLINK_SKILLS=1`) | `ln -sfn` | Trỏ mới, không merge thư mục. |
| **`npx viepilot install`** | **Luôn** chạy **`install.sh`** (mọi target: `cursor-agent`, `cursor-ide`, `claude-code`) — xem `bin/viepilot.cjs` `handlerForTarget` | **Không** dùng `dev-install.sh`. Hành vi = **`install.sh`**: không xóa trước, `cp -r` đè; có thể sót file orphan. Chỉ set thêm env `VIEPILOT_AUTO_YES=1`, `VIEPILOT_INSTALL_PROFILE=<target>`. |
| **`vp-tools update`** | npm upgrade global/local | Khác hẳn: không đụng `~/.cursor/skills` copy từ repo cũ (trừ khi user cài qua npm và tool chain đồng bộ). |

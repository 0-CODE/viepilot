### B. High-Value Missing Tools for ViePilot

#### 1. `Monitor` — Background Log/Process Watcher
**What it does:** Runs a command in the background; every output line is fed back to Claude.
Claude can react to log entries, file changes, or polled status mid-conversation.

**ViePilot use cases:**
- `vp-auto`: watch `npm test --watch` output during implementation
- After spawning a dev server, Monitor can tail its logs and alert on errors
- Quality gate: run `jest --watchAll` and react when tests turn red
- CI polling: `until ci-status done; do sleep 30; done`

**How to document:** Add `Monitor` to Claude Code adapter tool list in all SKILL.md files.
Add usage patterns to `workflows/autonomous.md` quality gate section.

---

#### 2. `CronCreate` / `CronDelete` / `CronList` — In-Session Scheduling
**What it does:** Schedule a prompt to re-run at intervals within the current session.
Tasks survive `--resume` / `--continue`.

**ViePilot use cases:**
- `vp-auto`: after spawning a long build, schedule a check every 5 min
- `vp-auto`: schedule a "check git push status" reminder
- Can replace the `ScheduleWakeup` pattern for non-/loop scenarios

**How to document:** Add to Claude Code adapter, mention alongside `ScheduleWakeup`.

---

#### 3. `EnterWorktree` / `ExitWorktree` — Isolated Git Worktree
**What it does:** Creates an isolated git worktree so the subagent works on a copy of the repo
without affecting the main branch. Auto-cleaned on exit if no changes.

**ViePilot use cases:**
- `vp-task-executor`: each task runs in a worktree → zero branch conflicts during parallel fan-out
- Replaces the manual `git worktree add` + `git worktree remove` Bash pattern
- Agent tool already supports `isolation: "worktree"` — this is the underlying primitive

**How to document:** Add `EnterWorktree`/`ExitWorktree` to Claude Code adapter. Document
in `workflows/autonomous.md` → parallel task execution section.

---

#### 4. `LSP` — Language Server Protocol Code Intelligence
**What it does:** Jump to definition, find references, type errors/warnings after each edit
(no separate build step). Requires a code intelligence plugin.

**ViePilot use cases:**
- `vp-task-executor`: after editing a file, LSP auto-reports type errors → self-correcting loop
- Replaces the pattern of running `tsc --noEmit` or `eslint` after every edit
- Quality gate: fail-fast on type errors without waiting for full test suite

**How to document:** Add `LSP` to Claude Code adapter tool list. Note plugin requirement.

---

#### 5. `EnterPlanMode` / `ExitPlanMode` — Plan Mode Gate
**What it does:** `EnterPlanMode` switches Claude to plan-only mode (no file edits, no
shell commands). `ExitPlanMode` presents the plan for approval and exits.

**ViePilot use cases:**
- `vp-evolve`: after phase planning, use ExitPlanMode to show the plan before committing
- `vp-auto` doc-first gate: enter plan mode for contract validation, exit to implement
- User can reject the plan without any files being touched

**How to document:** Add to Claude Code adapter. Document ExitPlanMode as an alternative
approval gate in `workflows/evolve.md` and `workflows/autonomous.md`.

---

#### 6. `PushNotification` — Long-Task Completion Alerts
**What it does:** Sends a desktop notification + phone push (when Remote Control connected).
Anthropic-infra only (not Bedrock/Vertex).

**ViePilot use cases:**
- `vp-auto` phase complete → push notification "Phase 144 done, 21 tests pass"
- `/vp-auto` running overnight → notify on error or when paused at control point

**How to document:** Add to Claude Code adapter. Add optional notification at phase
complete in `workflows/autonomous.md`.

---

#### 7. `SendMessage` + `TeamCreate` / `TeamDelete` — Agent Teams (Experimental)
**What it does:** Multiple Claude Code sessions communicate via message passing.
Requires `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`.

**ViePilot use cases:**
- Future: `vp-auto` orchestrator as team lead, task-executor agents as teammates
- More structured than fire-and-forget Agent spawning — allows back-and-forth
- Better fit for phases with complex inter-task dependencies

**How to document:** Add experimental section in `docs/dev/agents.md`.
Document as planned upgrade to ENH-096/097 orchestration model.

---

#### 8. `RemoteTrigger` — claude.ai Routines
**What it does:** Create scheduled tasks (Routines) that run on claude.ai infra even when
your computer is off. Backs the `/schedule` command.

**ViePilot use cases:**
- Schedule `vp-auto` runs nightly or on weekdays
- Auto-trigger quality gates on CI events
- Periodic docs drift check (DEBT-001 type tasks)

**How to document:** Document in `docs/user/features/autonomous-mode.md` → scheduling.

### C. Hooks Inventory — New Events Since ViePilot's Initial Hook Docs

ViePilot currently documents: `PreToolUse`, `PostToolUse`, `UserPromptSubmit`.

**New hooks discovered (not yet in ViePilot docs):**

| Hook | Fires When | Key ViePilot Use |
|------|-----------|-----------------|
| `SubagentStart` | Subagent spawned | Log which agent starts + context injection |
| `SubagentStop` | Subagent finishes | Block premature stop / collect output |
| `PostToolBatch` | After parallel tools resolve | Validate batch results before next model call |
| `TaskCreated` | TaskCreate tool used | Enforce naming conventions, block bad tasks |
| `TaskCompleted` | Task marked complete | Trigger quality gate automatically |
| `FileChanged` | Watched file changes | React to test file saves → rerun tests |
| `PreCompact` | Before context compaction | Block compaction during critical phase steps |
| `PostCompact` | After compaction | Re-inject phase context |
| `WorktreeCreate` | Worktree created | Custom setup (copy env files, install deps) |
| `WorktreeRemove` | Worktree removed | Cleanup custom setup |
| `TeammateIdle` | Agent team teammate idle | Keep agent alive / assign next task |
| `ConfigChange` | Config file changes | Block unexpected config changes during runs |
| `UserPromptExpansion` | Slash command expands | Audit/block vp-* skill invocations |
| `Stop` | Claude finishes turn | Continue autonomous loop automatically |

**Most impactful for ViePilot:**
- `TaskCreated` + `TaskCompleted` → hook-driven quality gates (no manual trigger)
- `SubagentStart` + `SubagentStop` → better orchestration logging
- `PreCompact` + `PostCompact` → protect against mid-phase context loss
- `Stop` → enable truly autonomous `/vp-auto` without user re-prompting
- `PostToolBatch` → validate parallel fan-out results (phase 144 pattern)

### D. `TodoWrite` Deprecation Impact

ViePilot SKILL.md files currently list `TodoRead` / `TodoWrite` in tool sections.
Since Claude Code v2.1.142 (Jan 2026), `TodoWrite` is disabled by default.

**Affected files to audit:**
- All SKILL.md files mentioning `TodoRead` / `TodoWrite`
- `workflows/autonomous.md` if it references these tools

**Fix:** Replace `TodoRead/TodoWrite` references with `TaskCreate/TaskGet/TaskList/TaskUpdate`.

## Summary
`workflows/autonomous.md` và `workflows/documentation.md` vừa được thêm các bước update `docs/skills-reference.md` (ENH-005) và đọc `node bin/vp-tools.cjs version` (ENH-001). Những bước này **chỉ có nghĩa trong viepilot framework repo**, không phải trong dự án người dùng.

## Affected Code (từ M1.5)

### `workflows/autonomous.md` — Step 5a Sync ROADMAP.md
```bash
ACTUAL_VERSION=$(node bin/vp-tools.cjs version get --raw 2>/dev/null | ...)
```
→ `bin/vp-tools.cjs` không tồn tại trong user project.

### `workflows/autonomous.md` — Step 5b Update skills-reference.md
```bash
NEW_SKILLS=$(git diff vp-p{N}..HEAD --name-only | grep 'skills/.*/SKILL\.md')
```
→ User project không có `skills/` directory với SKILL.md files.

### `workflows/autonomous.md` — Step 6a Sync README.md badges
```bash
ACTUAL_SKILLS=$(ls skills/*/SKILL.md 2>/dev/null | wc -l | tr -d ' ')
ACTUAL_WORKFLOWS=$(ls workflows/*.md 2>/dev/null | wc -l | tr -d ' ')
```
→ User project không có `skills/` hay `workflows/` directory.

Update: README.md `skills-{N}` badge, `workflows-{N}` badge
→ User README không có những badges này.

### `workflows/documentation.md` — Step 3B Generate/Update skills-reference.md
Toàn bộ step này scan `skills/*/SKILL.md` và update `docs/skills-reference.md`.
→ Không áp dụng cho user project.

### `workflows/documentation.md` — Step 0 Resolve Project Context
```bash
ACTUAL_SKILLS=$(ls skills/*/SKILL.md 2>/dev/null | wc -l | tr -d ' ')
ACTUAL_WORKFLOWS=$(ls workflows/*.md 2>/dev/null | wc -l | tr -d ' ')
```
→ Sẽ return 0 cho user project, nhưng vẫn được sử dụng.

## Desired Improvement

### Pattern: Conditional execution với framework detection

Tất cả các bước liên quan đến viepilot framework internals nên được wrap bằng detection guard:

```bash
# Pattern: Detect if running inside viepilot framework repo
IS_VIEPILOT_FRAMEWORK=$([ -f "bin/vp-tools.cjs" ] && [ -d "skills" ] && echo "true" || echo "false")
```

Hoặc đơn giản hơn — thêm comment/instruction cho AI:
> If `bin/vp-tools.cjs` does not exist in current project, skip this step.
> If `skills/` directory does not exist, skip skills-related steps.

### Cụ thể trong `workflows/autonomous.md`:

**Step 5b** — wrap với guard:
```markdown
#### Post-phase: Update skills-reference.md (viepilot framework only)
> Skip this step if `skills/` directory does not exist in the project.
```

**Step 6a** — rewrite để generic hơn:
```markdown
#### Sync README.md on milestone complete
Update README.md to reflect current project state:
- If project has `package.json`: sync version from `package.json`
- If project is viepilot framework (has `skills/` dir): also sync skills/workflows badge counts
- Update any version references in README.md
- Commit: `docs: sync README.md to v{version}`
```

### Cụ thể trong `workflows/documentation.md`:

**Step 0** — remove skills/workflows count:
```bash
# Only get GitHub context and project version — no framework-specific counts
REMOTE_URL=$(git remote get-url origin 2>/dev/null || echo "")
# ... extract GITHUB_OWNER, GITHUB_REPO as before
# Remove: ACTUAL_SKILLS, ACTUAL_WORKFLOWS (viepilot-specific)
```

**Step 3B** — add conditional guard:
```markdown
### Step 3B: Update skills-reference.md (viepilot framework only)
> Only run this step if `skills/` directory exists and contains `*/SKILL.md` files.
> Skip entirely for non-framework projects.
```

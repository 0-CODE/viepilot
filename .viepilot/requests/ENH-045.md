## Summary
Hiện tại vp-proposal PPTX bị cố định theo stock template với số trang tối đa cứng nhắc, khiến các dự án lớn hơn bị nén thông tin vào số slide cố định. Bên cạnh đó, design của PPTX cần được nâng cấp để mặc định tạo ra bản trình bày chuyên nghiệp, bắt mắt, phù hợp với ý tưởng dự án — đủ chất lượng để thuyết trình cho khách hàng.

## Details

### Current Behavior
- **Cố định template**: `scripts/gen-proposal-pptx.cjs` sử dụng 4 stock template (project-overview, technical-detail, commercial-detail, team-expertise) mỗi cái hard-code số slide
- **Giới hạn trang**: Dù dự án có bao nhiêu phases, features, hay components, thông tin vẫn bị ép vào số slide cố định của template
- **Design tĩnh**: PPTX output có layout + màu sắc cố định không phản ánh đặc thù của từng dự án
- **Thiếu tính scalable**: Dự án 3 phases và dự án 10 phases dùng cùng số slide → thông tin bị cắt bớt hoặc quá nén

### Desired Improvement

#### 1. Dynamic Slide Count
- `workflows/proposal.md` Step 4a (`generate_slide_manifest`) phải tính số slide **dựa trên nội dung thực tế** của dự án:
  - Số phases → số slide timeline tương ứng
  - Số team members → rows trong team slide (hoặc tách thành nhiều slides)
  - Số risk items → expand risk section nếu cần
  - Số features → feature breakdown slides
  - **Không có hard max** — AI quyết định số slide tối ưu cho dự án
- Step 4a phải trả về `slideManifest[]` với số entries phù hợp (không cố định 10/12/15 slides)

#### 2. AI-Driven Visual Design
- Trước khi generate PPTX, AI chọn **color palette + layout style** phù hợp với ngành/loại dự án:
  - Tech/SaaS → Modern, gradient, dark accent
  - Finance/Enterprise → Professional, navy/gold, clean
  - Healthcare → Trust colors (blue/white), accessible
  - Creative/Startup → Vibrant, bold accent, dynamic layout
- `scripts/gen-proposal-pptx.cjs` nhận `designConfig` từ Step 4a:
  ```json
  {
    "colorPalette": { "primary": "#1a1a2e", "accent": "#16213e", "highlight": "#e94560" },
    "layoutStyle": "modern-tech",
    "fontPair": { "heading": "Montserrat", "body": "Inter" },
    "slideRatio": "16:9"
  }
  ```

#### 3. Rich Visual Layouts per Slide Type
- **Cover slide**: Full-bleed background image area, large project title, tagline
- **Executive Summary**: 2-column layout (key stats left, narrative right)
- **Timeline slide**: Visual Gantt-bar style rows (not just table)
- **Team slide**: Card-based layout with role/avatar placeholder per member
- **Investment slide**: Visual breakdown bar or donut chart (not just table)
- **Architecture/Technical slide**: Dedicated visual zone for screenshots/diagrams
- **CTA/Closing slide**: Strong call-to-action with next steps prominently styled

#### 4. Content-Aware Slide Splitting
- Nếu `technicalNarrative` dài → tự động split thành 2–3 slides
- Nếu có >4 team members → tách thành 2 team slides
- Nếu có >3 phases → expand timeline across 2 slides
- AI kiểm soát split logic trong Step 4a

### Affected Files
- `workflows/proposal.md` — Step 4a: dynamic manifest generation rules + designConfig field
- `scripts/gen-proposal-pptx.cjs` — apply designConfig, dynamic slide rendering, rich layouts
- `lib/proposal-generator.cjs` — potentially add `getDesignConfig(projectType)` helper
- `tests/unit/` — new test file for dynamic manifest + design config contracts

### Breaking Changes
No — PPTX output changes are additive; existing stock template fallback retained when no designConfig provided

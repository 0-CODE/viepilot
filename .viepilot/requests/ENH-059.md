# ENH: AskUserQuestion ToolSearch preload — prevent deferred-tool fallback on skill init

## Summary
In Claude Code (terminal), `AskUserQuestion` is a **deferred tool** — its JSON schema is not loaded at conversation start. Before it can be called, the model must first call `ToolSearch` with `select:AskUserQuestion` to load the schema. If this preload step is skipped, the model falls back to plain numbered-list text even though the adapter is Claude Code (where AUQ is REQUIRED).

## Details

### Observed behavior
When invoking a vp-* skill (e.g. `vp-request`, `vp-evolve`) in Claude Code terminal, the very first interactive prompt in the session falls back to text output instead of the AUQ dialog. The model later explains: *"AskUserQuestion is a deferred tool — I have to call ToolSearch with `select:AskUserQuestion` first."*

### Expected behavior
Every skill that uses AUQ must call `ToolSearch { query: "select:AskUserQuestion" }` as the **first action** in its init step, before any interactive prompt. Once loaded, AUQ remains available for the rest of the session.

### Root cause
The SKILL.md files and workflow files (request.md, evolve.md, autonomous.md) declare that AUQ is REQUIRED on Claude Code, but they do not instruct the model to preload it via `ToolSearch` at init time. The model therefore attempts `AskUserQuestion` directly, fails with a schema-not-found error, and silently falls back to text.

### Proposed fix
Add a **ToolSearch preload block** to the init section of each skill that uses AUQ:

**In SKILL.md `<process>` / `## Initialize` (or equivalent):**
```
**AUQ preload (Claude Code adapter — do before first interactive prompt):**
Call ToolSearch with query "select:AskUserQuestion" to load the schema.
Only skip if ToolSearch itself returns an error.
```

**Affected files (skills with AUQ prompts):**
- `skills/vp-request/SKILL.md` — uses AUQ at Step 1 (type detection)
- `skills/vp-evolve/SKILL.md` — uses AUQ at Step 2 (mode selection)
- `skills/vp-auto/SKILL.md` — uses AUQ at control points
- `skills/vp-brainstorm/SKILL.md` — uses AUQ
- `skills/vp-crystallize/SKILL.md` — uses AUQ

**Affected workflow files:**
- `workflows/request.md` — Step 1 / Step 2B
- `workflows/evolve.md` — Step 2 / Step 5
- `workflows/autonomous.md` — control points

### Why this is not a simple fallback issue
The adapter compatibility table correctly marks Claude Code as "REQUIRED". The problem is operational: the skill never instructs the model to run the prerequisite `ToolSearch` call. Adding an explicit preload instruction to each skill's init block will fix it permanently without runtime hacks.

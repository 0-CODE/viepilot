# BUG: hooks install writes wrong path — brainstorm-staleness.cjs not found → Stop hook non-zero exit

## Meta
- **ID**: BUG-031
- **Type**: Bug
- **Status**: triaged
- **Priority**: high
- **Created**: 2026-05-25
- **Reporter**: User (Windows dev environment)
- **Assignee**: AI

## Summary
`vp-tools hooks install` writes a Stop hook command pointing to
`~/.viepilot/hooks/brainstorm-staleness.cjs` (hardcoded `home/.viepilot/hooks/`),
but the actual file is installed at `{adapter.viepilotDir}/lib/hooks/brainstorm-staleness.cjs`
(e.g. `~/.claude/viepilot/lib/hooks/...`). The path `~/.viepilot/hooks/` never gets created.

Every time the Claude Code `Stop` event fires, `node` exits with non-zero (file not found)
→ `/doctor` reports "non-blocking status code" warning indefinitely.

## Details

### Root Cause
In `bin/vp-tools.cjs` lines 1010 and 1027, the hook command path is hardcoded:

```js
// WRONG — hardcoded to ~/.viepilot/hooks/ which doesn't exist
command: `node ${path.join(home, '.viepilot', 'hooks', 'brainstorm-staleness.cjs')}`
```

Actual install destination for Claude Code adapter:
```
~/.claude/viepilot/lib/hooks/brainstorm-staleness.cjs
```

The correct path should use `adapter.viepilotDir(home)`:
```js
// CORRECT
command: `node ${path.join(adapter.viepilotDir(home), 'lib', 'hooks', 'brainstorm-staleness.cjs')}`
```

### Affected Adapters
| Adapter | Wrong path (written to settings.json) | Correct path |
|---------|--------------------------------------|--------------|
| Claude Code | `~/.viepilot/hooks/brainstorm-staleness.cjs` | `~/.claude/viepilot/lib/hooks/brainstorm-staleness.cjs` |
| Cursor | `~/.viepilot/hooks/brainstorm-staleness.cjs` | `~/.cursor/viepilot/lib/hooks/brainstorm-staleness.cjs` |

### Symptom
1. User runs `vp-tools hooks install`
2. settings.json is updated with wrong path
3. Every Claude Code session → Stop event fires → `node` can't find file → exits non-zero
4. `/doctor` perpetually reports: `"non-blocking status code"` on Stop hook
5. No actual brainstorm staleness checking happens (hook silently fails)

### Windows-specificity
On Windows, `path.join(home, '.viepilot', ...)` produces `C:\Users\dev\.viepilot\...`
which makes the failure immediately obvious. On macOS/Linux the wrong path is
`/Users/dev/.viepilot/hooks/...` — also broken but may be less visible to users.

### Secondary issue: existing installs
Users who already ran `vp-tools hooks install` have the wrong path persisted in
`settings.json`. Fix must also update (re-write) existing hook entries, not just
prevent new wrong entries.

## Files to Fix
- `bin/vp-tools.cjs` — lines 1010 and 1027: change `path.join(home, '.viepilot', 'hooks', ...)` → `path.join(adapter.viepilotDir(home), 'lib', 'hooks', ...)`
- `vp-tools hooks install` — when re-run after fix, idempotent check must also detect and replace the old wrong-path entry (not just skip if any entry exists)

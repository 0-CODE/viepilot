## Summary

Claude Code hooks (`PostToolUse`, `Stop`) can fire shell commands after tool execution. ViePilot's BUG-005 class (AI skips state updates) could be partially solved at infrastructure level — a `PostToolUse` hook on Edit/Write detects stale HANDOFF.json/PHASE-STATE and alerts or auto-syncs.

## Source

Reference: `for-learn-claude-cli-main/reports/REPORT-lessons-learned.md` (Bài học 14 / Hooks), `CODEMAP-claude-source.md`.

## Mechanism

Claude Code hooks fire on tool events via settings:

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{ "type": "command", "command": "node bin/vp-tools.cjs handoff-sync --check" }]
    }],
    "Stop": [{
      "hooks": [{ "type": "command", "command": "node bin/vp-tools.cjs handoff-sync --force" }]
    }]
  }
}
```

`handoff-sync` command checks if HANDOFF.json position matches latest PHASE-STATE and TRACKER state — warns if stale.

## Proposed Fix

### Fix A — `vp-tools` new command: `handoff-sync`

```
node bin/vp-tools.cjs handoff-sync [--check | --force]

--check: Compare HANDOFF.json position vs PHASE-STATE current task. Exit non-zero + warn if stale.
--force: Write HANDOFF.json position from PHASE-STATE + TRACKER (sync without AI involvement).
```

### Fix B — `templates/project/CLAUDE.md` (or install-time hooks config)

Document recommended hooks setup for ViePilot projects in crystallize output or install script. Projects that opt in get automatic stale-state detection.

### Fix C — `autonomous.md` Post-task hook note

Add note in State Update Checklist: "If hooks are configured, `handoff-sync --check` fires automatically after each Edit — AI still responsible for explicit state writes but hook provides safety net."

# BUG: Workflow skill context steps use non-executable JS function call

## Summary
`workflows/autonomous.md` (Skill Context Load) và `workflows/brainstorm.md` (Skill Registry Integration) cùng hướng dẫn agent "Call `loadRegistry()` from `lib/skill-registry.cjs`" — đây là JS function, agent (LLM) không thể gọi trực tiếp. Cần thay bằng lệnh shell thực thi được.

## Details

### Actual behavior
Workflow viết:
```
Call loadRegistry() from lib/skill-registry.cjs (read ~/.viepilot/skill-registry.json)
```
Agent không có cơ chế gọi hàm JS nội bộ — chỉ có thể chạy bash / shell commands.

### Expected behavior
Workflow cung cấp lệnh shell thực thi được. Hai phương án:

**Option A (đơn giản nhất):** đọc trực tiếp file JSON:
```bash
cat ~/.viepilot/skill-registry.json 2>/dev/null
```
Agent đọc và parse JSON bằng LLM reasoning.

**Option B (CLI command):** thêm `vp-tools get-registry` subcommand xuất JSON — agent gọi `node bin/vp-tools.cjs get-registry 2>/dev/null`.

### Affected files
- `workflows/autonomous.md` — Skill Context Load (FEAT-020 block)
- `workflows/brainstorm.md` — Skill Registry Integration (FEAT-020 block)

### Steps to reproduce
1. Chạy `/vp-auto` trên project có `## Skills` trong PROJECT-CONTEXT.md
2. Agent cố gắng thực thi "Call `loadRegistry()`" → không có executable command tương ứng

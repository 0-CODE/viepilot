## Summary
Add `vp-tools ask` CLI command using `@clack/prompts` to replace plain-text numbered Q&A during brainstorm sessions. Users can use arrow keys to select options instead of typing "Q1. C" style responses.

## Problem
Brainstorm Q&A currently requires users to type answer codes like "Q1. C / Q2. A / Q3. B" in chat messages. This is awkward and error-prone. BMAD Method demonstrates that a TUI-based approach (arrow-key selection) dramatically improves the Q&A experience.

## Proposed Solution
Implement `node bin/vp-tools.cjs ask` sub-command:

```bash
# Single-select question
node bin/vp-tools.cjs ask --single \
  --question "Which approach for entity manifest?" \
  --options "A=Write to SPEC.md" "B=Separate entity-manifest.md" "C=Both"

# Multi-select question
node bin/vp-tools.cjs ask --multi \
  --question "Which gaps to fix?" \
  --options "A=Entity extraction" "B=Dependency validation" "C=Manifest output"
```

Output: selected key(s) to stdout (for script capture).

**Library**: `@clack/prompts` npm package (lightweight, no heavy dependencies).

**Integration**: `vp-brainstorm` skill calls `vp-tools ask` for each Q&A step when running in Claude Code CLI context; falls back to text Q&A in non-terminal/non-interactive contexts.

## Discussion
BMAD Method research: their TUI (React/Ink based) is for the CLI installer, not for AI-to-user interaction during brainstorm. The `@clack/prompts` approach is lighter and fits within vp-tools.cjs without a full React/Ink dependency.

The key insight is that `vp-tools ask` must be callable from within a Claude Code skill (shell command execution) and return the selection to stdout so the skill can proceed with the chosen answer.

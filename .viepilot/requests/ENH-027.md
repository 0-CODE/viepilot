## Summary
Bổ sung trang `erd.html` vào Architect Design Mode workspace, cho phép brainstorm trực tiếp ra Database ERD (Entities, Attributes, Relationships) với Mermaid `erDiagram` — tích hợp vào hub navigation và crystallize Step 1D.

## Details

### Current behavior
Architect Design Mode (FEAT-011) có 7 pages: index, architecture, data-flow, decisions, tech-stack, tech-notes, feature-map.
Không có trang ERD — khi brainstorm database schema phải viết text trong tech-notes hoặc không capture được.

### Desired improvement
1. **`templates/architect/erd.html`** — trang mới:
   - Mermaid `erDiagram` diagram với placeholder entities
   - Table: Entity | Attributes | Primary Key | Foreign Keys | Notes
   - Relationship summary: Entity A | Relationship type (one-to-one/many-to-one/many-to-many) | Entity B | Notes
2. **`templates/architect/index.html`** — thêm link `erd.html` vào hub card grid + sidebar nav
3. **`templates/architect/style.css`** — không cần thay đổi (style hiện tại đủ dùng)
4. **`workflows/brainstorm.md`** — Architect Design Mode section:
   - Thêm `erd.html` vào workspace layout list
   - Thêm trigger: khi user nhắc đến "database", "entity", "table", "schema", "relation", "foreign key", "primary key" → update `erd.html`
   - Diagram type: `erDiagram` (Mermaid)
5. **`workflows/crystallize.md`** — Step 1D `consume_architect_artifacts`:
   - Nếu `erd.html` tồn tại → extract entities + relationships → feed vào `.viepilot/ARCHITECTURE.md` section `## Database Schema (from Architect ERD)`
   - Nếu `notes.md` có `## erd` YAML section → parse entities/relationships
6. **`notes.md` YAML schema** — thêm section `## erd`:
   ```yaml
   ## erd
   entities:
     - name: User
       attributes: [id, email, name, created_at]
       primary_key: id
     - name: Order
       attributes: [id, user_id, total, status]
       primary_key: id
   relationships:
     - from: User
       to: Order
       type: one-to-many
       label: places
   ```
7. **`skills/vp-brainstorm/SKILL.md`** — bump version, thêm ERD mention
8. **`tests/`** — contract tests cho ERD page + crystallize extraction

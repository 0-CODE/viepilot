# Bug: BUG-005 — vp-* skills hardcode ~/.cursor/viepilot/ path, broken khi cài cho Claude Code

## Summary
Khi cài ViePilot cho `claude-code` target, skills được mirror vào `~/.claude/skills/` đúng, nhưng toàn bộ `execution_context` trong skill files vẫn hardcode `$HOME/.cursor/viepilot/` (Cursor path). Máy không có Cursor hoặc môi trường Claude Code standalone không tìm được `bin/vp-tools.cjs`, workflows, hay templates → mọi skill bị broken.

## Details

### Actual behavior
- `vp-info` trong Claude Code gọi `@$HOME/.cursor/viepilot/bin/vp-tools.cjs` → file không tồn tại trên máy không có Cursor.
- Tương tự với tất cả skills: `@$HOME/.cursor/viepilot/workflows/*.md`, `@$HOME/.cursor/viepilot/templates/project/`.

### Expected behavior
- Khi cài cho `claude-code` target → skill files phải trỏ vào đúng path của môi trường đó (ví dụ: `~/.claude/viepilot/` hoặc npm global bin path).
- Mỗi install target có `BIN_PATH` và `VIEPILOT_DIR` riêng.

### Root cause
`lib/viepilot-install.cjs` line ~266 comment xác nhận: `"Cursor paths always; Claude Code skills when target includes claude-code"` — install chỉ mirror skill files, không rewrite `execution_context` paths theo target.

| `claude-code` | `~/.claude/skills/` | `~/.claude/viepilot/` ❌ **chưa được mirror** |

## Proposed Fix

**Option A (recommended):** Khi install cho `claude-code` target, thêm bước mirror **workflows/ + bin/ + templates/** vào `~/.claude/viepilot/`; sau đó rewrite `execution_context` trong mirrored skill files để thay `$HOME/.cursor/viepilot` → `$HOME/.claude/viepilot`.

**Option B:** Dùng env variable `VIEPILOT_DIR` resolved at runtime — skills dùng `${VIEPILOT_DIR}/workflows/...` thay vì hardcode Cursor path. Nhưng option này phức tạp hơn vì Claude Code và Cursor agent không expand shell env trong `execution_context`.

**Option A là đủ và ít rủi ro nhất.**

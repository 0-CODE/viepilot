# BUG: vp-auto không thực hiện state updates sau mỗi task

## Summary

Sau khi vp-auto thực thi một task (pass), hệ thống **không** thực hiện đầy đủ các state update được yêu cầu bởi workflow. Vấn đề được phát hiện ở phase 12 (complete) và phase 13 (in-progress) của dự án `smart-track-platform`.

## Actual Behavior (observed)

- Task file `## Meta → Status` vẫn là `not_started` dù task đã pass
- Tất cả checklist boxes (`[ ]`) trong task files không được tick — Pre-execution doc gate, Acceptance Criteria, Best Practices, State Update Checklist
- `## Files Changed` section trong task files vẫn là placeholder `(Auto-populated after completion)`
- `## Files Changed` table trong PHASE-STATE.md trống sau khi phase complete
- `execution_state.status` trong PHASE-STATE.md không được cập nhật (phase 13: còn `not_started` dù tasks 13.1 và 13.5 đã `pass`)
- HANDOFF.json `status` field không được cập nhật sau mỗi task
- Git tags (`vp-p{N}-t{N}` và `vp-p{N}-t{N}-done`) không được tạo

## Expected Behavior (per autonomous.md spec)

Sau mỗi task PASS, workflow yêu cầu atomically:
1. Update task file Meta.Status → `done` + tick toàn bộ checklist boxes
2. Update PHASE-STATE.md: task row → `done`, populate Files Changed table, update `execution_state.status`
3. Update HANDOFF.json với position/recovery/meta fields
4. Tạo git tags (`-done` suffix)
5. Update TRACKER.md và ROADMAP.md

## Root Cause Analysis

### RC-1: Không có enforcement checkpoint
`autonomous.md` mô tả state updates như prose trong bước "Handle Result → PASS". Không có mechanism buộc verify rằng tất cả state updates đã được thực hiện trước khi advance sang task tiếp theo.

### RC-2: AI ưu tiên implementation code
Khi context window lớn (phase 13 với nhiều files), AI hoàn thành implementation code nhưng skip hoặc truncate các "housekeeping" edits (task file meta, checkbox ticks, PHASE-STATE tables).

### RC-3: HANDOFF.json schema mismatch
`autonomous.md` references `control_point.active`, `position.task`, `recovery.l1_attempts`, `meta.last_written` — nhưng HANDOFF.json thực tế dùng flat schema (`status`, `task`, `next_action`, `context_notes`). Schema divergence khiến nhiều update instructions không thể apply được.

### RC-4: Files Changed tracking yêu cầu extra step
Populate Files Changed table yêu cầu chạy `git diff` sau commit — step này bị skip vì không có explicit reminder trong execution loop.

### RC-5: Markdown checkbox edits bị bỏ qua
Ticking `[ ]` → `[x]` trong task files yêu cầu Edit calls riêng cho từng file. Không có explicit instruction nào trong workflow nói "edit task file để tick checkbox X, Y, Z".

## Proposed Fix (high-level)

### Fix A — autonomous.md: State Update Checklist Block
Sau "Handle Result → PASS", thêm explicit mandatory block:

```
#### State Update Checklist (mandatory — verify each before advancing)

For each item: edit the file, make the change, verify the change was written.

TASK FILE (.viepilot/phases/{phase}/tasks/{task}.md):
- [ ] Meta.Status: not_started → done
- [ ] Pre-execution doc gate boxes: all → [x]
- [ ] Acceptance Criteria boxes: mark each met criterion [x]
- [ ] Best Practices boxes: all applied → [x]
- [ ] State Update Checklist boxes: all → [x]
- [ ] ## Implementation Notes: fill with actual implementation summary
- [ ] ## Files Changed: populate from `git diff {start-tag}..HEAD --name-status`

PHASE-STATE.md:
- [ ] Task row Status → done, Completed timestamp → now, Git Tag → actual tag
- [ ] execution_state.status → pass (if all tasks done) or executing (if more tasks)
- [ ] Files Changed table: append each file from this task

HANDOFF.json:
- [ ] task → {next_task_id}
- [ ] status → not_started
- [ ] lastUpdated → {ISO8601}

TRACKER.md: update current task line

If any edit fails: → control_point("state update failed: {item}")
```

### Fix B — HANDOFF.json schema alignment
Sync HANDOFF.json schema với autonomous.md expectations, hoặc update autonomous.md để reference đúng flat schema.

### Fix C — vp-tools: state-sync command
Add `node bin/vp-tools.cjs state-sync --phase {N} --task {T}` command để:
1. Read completed task
2. Auto-populate task file and PHASE-STATE.md from git history
3. Can be run manually after a session where state updates were missed

## Discussion

Vấn đề này ảnh hưởng đến **khả năng resume** của vp-auto: khi resume sau session bị ngắt, hệ thống không thể biết chính xác state vì HANDOFF.json và PHASE-STATE.md không accurate. Bản chất bug là **workflow spec giả định AI sẽ thực hiện tất cả state updates**, nhưng không có mechanism để enforce hoặc verify điều đó.

Fix quan trọng nhất là Fix A — làm state updates thành explicit, atomic, verifiable checklist thay vì prose description.

# BUG: Git tag collision across projects due to generic checkpoint names

## Meta
- **ID**: BUG-002
- **Type**: Bug
- **Status**: done
- **Priority**: high
- **Created**: 2026-03-31
- **Reporter**: User
- **Assignee**: AI

## Summary
Checkpoint tags do `vp-auto` tạo ra đang dùng format chung (ví dụ `vp-p{N}-...`), dễ trùng khi nhiều dự án trên cùng máy cùng dùng ViePilot, gây lỗi khi tạo tag hoặc khó phân biệt nguồn gốc tag.

## Details

### Actual behavior
- Workflow tạo git tags theo mẫu generic, không kèm định danh project/repo.
- Khi nhiều repo áp dụng cùng naming convention, tag naming có thể conflict theo ngữ cảnh vận hành và gây nhầm lẫn trong quản trị checkpoint.

### Expected behavior
- Tag phải có project-scoped prefix ổn định, ví dụ:
  - `{projectSlug}-vp-p{N}-t{M}`
  - hoặc `vp-{repoName}-p{N}-...`
- Vẫn giữ backward compatibility cho các flow hiện tại (list/restore/checkpoint lookup).

### Steps to reproduce
1. Dùng ViePilot cho nhiều repository khác nhau với checkpoint tags cùng pattern generic.
2. Chạy các bước tạo checkpoint/tag theo workflow `vp-auto`.
3. Quan sát khó phân biệt hoặc nguy cơ collision với tag naming hiện tại.

### Affected area
- `workflows/autonomous.md`
- `skills/vp-auto/SKILL.md`
- `bin/vp-tools.cjs` (nếu có logic list/filter/restore tag)
- docs liên quan checkpoint/recovery

### Severity
- high

## Discussion
Đề xuất của user là hợp lý: thêm tiền tố theo dự án là cần thiết để giảm collision risk và tăng khả năng truy vết checkpoint.

## Resolution
Completed in M1.18 / Phase 22:
- Introduced project-scoped checkpoint prefix strategy (`{project}-vp-...`) to avoid cross-project collisions.
- Added `vp-tools tag-prefix` and updated checkpoint discovery to support both new and legacy tag formats.
- Updated workflow/docs/templates to use project-prefixed tags with compatibility notes.

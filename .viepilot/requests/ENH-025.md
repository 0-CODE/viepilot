## Summary

Claude Code Agent tool supports `isolation: "worktree"` — spawns agent in an isolated git worktree copy of the repo. Ideal for L/XL complexity tasks: if task fails halfway, working code is never corrupted. User reviews diff before merging. ViePilot has no equivalent; currently L/XL tasks that fail midway leave broken intermediate state.

## Source

Reference: `for-learn-claude-cli-main/reports/REPORT-lessons-learned.md` (Bài học 7 — Worktree Isolation), `REPORT-multi-agent-coordinator.md`.

## Mechanism

```typescript
AgentTool({
  prompt: "Implement task 9.3 in worktree",
  isolation: "worktree"
})
// → git worktree add .claude/worktrees/<agentId>
// Agent does all work in isolated copy
// Must commit before returning result
// Returns: { worktree_path, branch, changes_summary }
// Worktree cleaned up if no changes made
```

## Proposed Fix

### autonomous.md — Worktree escalation for L/XL tasks

In Execute Task section, add worktree path for L/XL complexity:

```
#### Worktree Isolation (for complexity=L or XL)
When task.complexity in (L, XL):
  Offer: "This is a large task. Run in isolated worktree for safety? (y/n)"
  If yes:
    Spawn Agent with isolation="worktree"
    Agent implements in isolated copy
    On complete: present diff summary → user approves → merge OR discard
  If no: proceed in-place (existing behavior)
```

### TASK.md template

Add optional field:
```yaml
worktree_isolation: auto    # auto | always | never
                            # auto = offer for L/XL (default)
                            # always = always use worktree regardless of complexity
                            # never = skip offer
```

## Acceptance Criteria

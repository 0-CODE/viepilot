## Summary
Nang cap quy trinh implement (dac biet `vp-auto` va cac skill co coding) de phan bo task chi tiet hon va cap nhat state ngay sau tung task/sub-task thay vi doi den cuoi phase. Muc tieu la tranh mat dau vet khi dung dot xuat, tang do chinh xac va kha nang resume.

## Current Feature
Hien tai task decomposition trong mot so phase con chung chung, thieu granularity cho implementation details va state updates co the bi don den cuoi dot thuc thi.

## Current Behavior
- Task mo ta high-level, chua bat buoc:
  - path cu the
  - implementation files cu the
  - best practices can ap dung theo stack
  - output artifact contract
- State files (`ROADMAP`, `TRACKER`, `PHASE-STATE`, `HANDOFF`) co luc duoc cap nhat batch sau cung, de gay mat trace khi session bi ngat.

## Desired Improvement

### 1) Task planning chi tiet, co the action ngay
Voi moi task implementation:
- Bat buoc co:
  - `Task ID` + objective ro rang
  - Danh sach file paths can sua/tao
  - Description theo tung file (what + why)
  - Best practices can dung (stack-specific + coding conventions)
  - Verification checklist (commands + expected result)
- Neu task lon thi tach sub-task 30-90 phut de de theo doi.

### 2) Operational contract cho implementation skills
Voi `vp-auto` va bat ky skill nao co implement:
- Truoc khi code: ghi plan itemized vao task file/phase state.
- Sau moi sub-task PASS:
  - update ngay `PHASE-STATE.md` (status/progress/files changed)
  - update ngay `TRACKER.md` current task/progress delta
  - update ngay `HANDOFF.json` context/checkpoints
  - neu co thay doi milestone summary thi update `ROADMAP.md` ngay o muc lien quan
- Tuyet doi khong doi den cuoi phase moi sync state.

### 3) Guard against abrupt stop
- Moi step quan trong tao checkpoint nho (logical checkpoints/tags).
- Neu bi interrupt, `HANDOFF.json` phai du de resume chinh xac den task/sub-task dang do.
- Co quy tac "state-first then continue": cap nhat state truoc khi bat dau sub-task tiep theo.

### 4) Best-practice aware execution template
Tao mau task template gom:
- `Paths`
- `Implementation notes`
- `Best practices to apply`
- `Do/Don't`
- `Verification`
- `State update checklist`

## Value Proposition
- Giam rui ro mat dau vet khi ngat session.
- Tang quality va reproducibility khi autonomous coding.
- De review va debug vi co trace tung buoc + tung file.
- Resume nhanh va chinh xac hon.

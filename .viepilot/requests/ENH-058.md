# ENH: Workflow continuation prompt — AskUserQuestion at vp-evolve/vp-request completion

- **Assignee**: AI

## Summary
Instead of showing a static "Next:" text list at the end of `vp-evolve` and `vp-request`,
use `AskUserQuestion` to actively ask the user if they want to continue immediately to the
next step (`/vp-evolve` or `/vp-auto`). This removes friction and makes the workflow feel
continuous rather than requiring the user to re-type commands.

## Current Behavior
After `vp-evolve` completes, the output shows:
```
Next:
 - /vp-auto    Execute Phase 83
 - /vp-request Create another request
```
User must manually type `/vp-auto` or nothing happens.

After `vp-request` creates a request, same pattern — static list, no prompt.

## Desired Behavior

### At vp-evolve completion:
Use `AskUserQuestion` to ask:
```
Phase plan ready. What would you like to do next?
  1. Execute now → /vp-auto  (Recommended)
  2. Create another request → /vp-request
  3. Done for now
```
If user selects "Execute now" → immediately invoke `/vp-auto` in the same session.

### At vp-request completion:
Use `AskUserQuestion` to ask:
```
Request {ID} logged. What next?
  1. Plan phase + tasks → /vp-evolve  (Recommended)
  2. Create another request → /vp-request
  3. Done for now
```
If user selects "Plan phase + tasks" → immediately invoke `/vp-evolve`.

# BUG: vp-evolve/vp-request Step 5/6 thiếu AUQ call trong `<process>` — chỉ có text template

## Summary
`skills/vp-evolve/SKILL.md` Step 5 và `skills/vp-request/SKILL.md` Step 6 chỉ có text template trong `<process>` mà không có AUQ call instruction. AUQ spec đầy đủ nằm trong `workflows/evolve.md` / `workflows/request.md` nhưng **không được load** vì Claude Code không auto-expand `@` file reference trong `<execution_context>`.

## Root Cause

### Layer analysis

| Layer | Tình trạng |
|-------|-----------|
| `workflows/evolve.md` line 337–348 | ✅ AUQ spec đầy đủ ("Phase N planned. What next?") |
| `skills/vp-evolve/SKILL.md` Adapter Compatibility | ✅ Khai báo: "Workflow continuation (Step 5 — AUQ)" |
| `skills/vp-evolve/SKILL.md` `<process>` Step 5 | ❌ Chỉ có text template — không có AUQ call |
| `<execution_context>` `@$HOME/.../workflows/evolve.md` | ❌ Không được Claude Code auto-expand |

### Cơ chế

Khi `/vp-evolve` được gọi, Claude Code load `skills/vp-evolve/SKILL.md` vào system context. Agent đọc `<process>` → Step 5 và thấy:

```
### Step 5: Confirm & Suggest Next
✓ Evolution complete
...
Next action: /vp-auto --from {new_phase}
```

Không có AUQ instruction nào trong Step 5 body. Agent output text và kết thúc.

`<execution_context>@$HOME/.claude/viepilot/workflows/evolve.md</execution_context>` là một reference tag — Claude Code **không expand** nội dung file vào context. Nên AUQ spec ở `workflows/evolve.md` line 337 không bao giờ được agent thấy.

ENH-058 đã add AUQ vào workflow file nhưng **không update** SKILL.md `<process>` step body.

## Actual behavior
Step 5 output là plain text:
```
Next action: /vp-auto --phase 95
```

## Expected behavior
Step 5 gọi AUQ với options:
- "Execute now → /vp-auto"
- "Create another request → /vp-request"  
- "Done for now"

## Affected files
- `skills/vp-evolve/SKILL.md` — Step 5 `<process>` body thiếu AUQ
- `skills/vp-request/SKILL.md` — Step 6 `<process>` body thiếu AUQ

## Fix

Thêm AUQ call instruction trực tiếp vào Step 5 body của `skills/vp-evolve/SKILL.md`:

```markdown
### Step 5: Confirm & Suggest Next

[display summary banner]

**Claude Code (terminal) — REQUIRED:** After displaying summary, call `AskUserQuestion`:
- question: "Phase {N} planned. What would you like to do next?"
- options:
  - "Execute now → /vp-auto" (Recommended)
  - "Create another request → /vp-request"
  - "Done for now"

**On selection:**
- "Execute now": invoke /vp-auto skill
- "Create another request": invoke /vp-request skill
- "Done for now": print confirmation and exit

**Text fallback** (Cursor/Codex/other): show plain-text next-actions menu.
```

Tương tự cho `skills/vp-request/SKILL.md` Step 6.

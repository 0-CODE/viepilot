# ENH-029: Remove MVP Concept — Full-Arc ROADMAP with Priority + Release Target

## Meta

- **Type**: Enhancement
- **Status**: new
- **Priority**: high
- **Created**: 2026-04-03
- **Reporter**: User
- **Assignee**: AI

## Summary
Remove the MVP/Post-MVP/Future tier system from ViePilot. Replace with `priority` (P1/P2/P3) + `release_target` (v1.0/v1.1/backlog) fields per phase. crystallize generates a full project arc ROADMAP upfront. Phases are never archived — done phases collapse to AutoDream summaries. `vp-evolve --milestone` deprecated.

## Problem
The MVP concept causes severe context loss:
1. Post-MVP restart archives ROADMAP → brainstorm decisions lost
2. `vp-evolve --milestone` forces a fresh start with no phase history
3. "Post-MVP" implies the first product was complete, but Autonomous Vibe Coding means continuous build
4. Teams lose entity manifest, phase rationale, dependency chains when archiving
5. Product horizon with MVP/Post-MVP/Future tiers in brainstorm gets mapped to ViePilot phases incorrectly

## Proposed Solution

### Replace tier tags with phase-level fields

**Before (current):**
```yaml
## Product horizon
- Tenant Management API (MVP)
- Vehicle Fleet Service (Post-MVP)
- Analytics Dashboard (Future)
```

**After (ENH-029):**
```yaml
phases:
  - id: 3
    name: Tenant Management API
    priority: P1
    release_target: v1.0
  - id: 7
    name: Vehicle Fleet Service
    priority: P2
    release_target: v1.1
  - id: 11
    name: Analytics Dashboard
    priority: P3
    release_target: backlog
```

### Full-Arc ROADMAP
- crystallize generates all phases upfront (P1 through P3)
- No "scope" boundary — the entire product vision is planned
- `vp-auto` executes in priority order (P1 first, then P2, etc.)
- ROADMAP is never archived — it is the single source of truth

### Done Phase AutoDream
When a phase completes:
1. Phase row in ROADMAP.md collapses to one-line summary (status=done, link to SUMMARY.md)
2. Key learnings pushed to `~/.viepilot/profiles/<project>.md` (AutoDream entry)
3. Entity manifest and dependency data preserved in STACKS.md / architecture artifacts

### Deprecate vp-evolve --milestone
- `vp-evolve --milestone` → replaced by `vp-evolve --add-phases` (append new phases to ROADMAP)
- No archiving, no fresh starts, no context loss
- Version bumps (MAJOR) still happen at release_target boundaries

### brainstorm Product Horizon Update
- `## Product horizon` section in brainstorm sessions uses `priority: P1/P2/P3` + `release_target`
- Drops `(MVP)` / `(Post-MVP)` / `(Future)` tier tags
- crystallize maps brainstorm priority/release_target directly to phase fields

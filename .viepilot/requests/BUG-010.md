# BUG-010: ENH-033 — Approve/Edit buttons missing on Mermaid diagram cards

## Summary
The ENH-033 Approve/Edit feature added `data-arch-id` only to `<tr>` rows inside `<tbody>` sections.
Pages that contain Mermaid diagram blocks (`<div class="mermaid-wrap">` inside `<div class="card">`)
received no IDs or action buttons. The diagrams are the primary visual artifacts in several pages
and are a critical target for Approve/Edit interaction.

## Affected Pages and Missing Diagram IDs

| Page | Diagram(s) | Missing IDs |
|------|-----------|-------------|
| `architecture.html` | C4 Context Diagram, System Diagram | ARCH-DIAG1, ARCH-DIAG2 |
| `data-flow.html` | Service Flow, Event Flow | DF-DIAG1, DF-DIAG2 |
| `erd.html` | ERD | ERD-DIAG1 |
| `user-use-cases.html` | Use Case Diagram | UC-DIAG1 |
| `sequence-diagram.html` | Scenario 1 diagram, Scenario 2 diagram | SEQ-DIAG1, SEQ-DIAG2 |
| `deployment.html` | Infrastructure Diagram | DEP-DIAG1 |

**Total missing**: ~9 diagram sections across 6 pages.

## Actual Behavior
Hovering over a Mermaid diagram card shows no badge, no Approve button, no Edit button.
The card wrapping the diagram has no `data-arch-id` attribute.

## Expected Behavior
Each `<div class="card">` containing a `<div class="mermaid-wrap">` should:
- Have a `data-arch-id` (e.g., `ARCH-DIAG1`, `ERD-DIAG1`, `SEQ-DIAG1`)
- Have a `data-arch-title` matching the card's `<h2>` heading
- Receive badge + Approve/Edit buttons via `architect-actions.js` on hover

## Proposed Fix

### 1. HTML templates — add `data-arch-id` to diagram cards
For each `<div class="card">` that contains a `<div class="mermaid-wrap">`:
- Add `data-arch-id="{PAGE-DIAG}{n}"` to the **outer card div**
- Add `data-arch-title="{diagram heading text}"` for clean title extraction

### 2. architect-actions.js — ensure div branch handles card-level injection cleanly
The existing `inject()` function already handles non-TR elements via the `div` branch.
The actions div should be inserted after the `<h2>` heading inside the card (not before it).

### 3. ID scheme for diagram items
Use a `DIAG` prefix scoped per page to distinguish from table-row IDs:
- architecture: `ARCH-DIAG1`, `ARCH-DIAG2`
- data-flow: `DF-DIAG1`, `DF-DIAG2`
- erd: `ERD-DIAG1`
- user-use-cases: `UC-DIAG1`
- sequence-diagram: `SEQ-DIAG1`, `SEQ-DIAG2`
- deployment: `DEP-DIAG1`

### 4. CSS — ensure `.card:hover .arch-item-actions` reveal works
The existing hover rule `tr:hover .arch-item-actions` covers rows; the card hover rule
`.card:hover > .arch-item-actions` was written but inject position may need verification.
The actions div should be inserted as **first child after `<h2>`**, not as last child, so
it appears in the header area of the card rather than after the diagram.

## Root Cause
The Python script in Phase 50 task execution only queried `<tr>` elements inside `<tbody>`,
and for tech-notes.html used `<div class="card">` direct children of `<main>`. It did not
process `<div class="card">` elements that contain `<div class="mermaid-wrap">`.

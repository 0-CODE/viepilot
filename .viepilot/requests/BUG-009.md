# BUG: vp-evolve generates task Paths using installed/external paths instead of repo-relative paths

## Meta
- **ID**: BUG-009
- **Type**: Bug
- **Status**: done
- **Priority**: high
- **Created**: 2026-04-06
- **Reporter**: User
- **Assignee**: AI

## Summary
When `vp-evolve` (and any workflow that generates implementation task files) builds `## Paths` blocks in task `.md` files, it uses **installed/external paths** (e.g., `~/.claude/viepilot/workflows/`, `~/.claude/skills/vp-*/`) instead of **repo-relative source paths** (e.g., `workflows/`, `skills/vp-*/`). This causes `vp-auto` to target the wrong files during implementation — editing the live installed copy rather than the source codebase.

## What happened
Phase 47 and Phase 48 task files (47.1–47.8, 48.4–48.7) were generated by `vp-evolve` with `## Paths` blocks pointing to:
```
~/.claude/viepilot/workflows/crystallize.md
~/.claude/skills/vp-audit/SKILL.md
~/.claude/viepilot/templates/project/AI-GUIDE.md
```

## What should happen
Task `## Paths` blocks must always use repo-relative paths:
```
workflows/crystallize.md
skills/vp-audit/SKILL.md
templates/project/AI-GUIDE.md
```

Paths relative to the repo root (where `package.json` / `Makefile` lives). These are the **source files** that get installed to `~/.claude/viepilot/` — editing them is the correct flow; editing the installed copy bypasses version control and the install pipeline.

## Root cause
The audit scan in the previous session (`/vp-audit`) detected language issues by reading files from the **installed location** (`~/.claude/viepilot/`, `~/.claude/skills/`). When `vp-evolve` generated task files based on those findings, it transcribed those installed paths verbatim into the `## Paths` blocks rather than converting them to the corresponding source repo paths.

The `vp-evolve` workflow (and `autonomous.md` which executes tasks) has **no guard** that enforces repo-relative paths in task definitions.

## Affected workflows
- `workflows/evolve.md` — generates phase task files; must enforce repo-relative paths
- `workflows/autonomous.md` — executes tasks by reading `## Paths`; must reject or warn on absolute/external paths
- `workflows/request.md` (indirectly) — when triaging, should note the path convention

## Proposed fix

### 1. `workflows/evolve.md` — task generation guard
Add an explicit rule to the task generation step:

```
TASK PATH RULE (BUG-009):
When writing ## Paths in any task .md file, ALWAYS use paths relative
to the repository root (where package.json / Makefile is located).

CORRECT:   workflows/crystallize.md
           skills/vp-audit/SKILL.md
           templates/project/AI-GUIDE.md
           lib/viepilot-config.cjs
           bin/vp-tools.cjs

INCORRECT: ~/.claude/viepilot/workflows/crystallize.md
           ~/.claude/skills/vp-audit/SKILL.md
           /absolute/path/to/anything

Exception: paths INSIDE code block content (describing what the
           code reads/writes at runtime) may use absolute paths
           (e.g., ~/.claude/viepilot/config.json inside a bash snippet).
           Only the ## Paths section header block must be repo-relative.
```

### 2. `workflows/autonomous.md` — preflight path validation
Add a preflight check before executing each task:

```
PREFLIGHT: validate task Paths block
- If any path in ## Paths starts with ~/ or / → WARN user and abort task
- Message: "Task {N} contains absolute path '{path}'. Expected repo-relative path.
            Fix the task file before continuing."
```

### 3. `skills/vp-evolve/SKILL.md` + `skills/vp-auto/SKILL.md`
Document the path convention in the skill context so the AI model internalizes it.

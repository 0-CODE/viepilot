## Summary
A new `/vp-persona` skill that creates a **user-level, cross-project identity** for ViePilot.
Allows multi-domain professionals ("ki√™m nhi·ªám") to customize ALL ViePilot workflows ‚Äî brainstorm topics,
phase templates, stack best practices, architect pages ‚Äî to match their domain, profession, and interests.
No re-declaring context per project. Supports multiple named personas with switch + merge.

## Current Behavior
- ViePilot has no user-level identity ‚Äî only `language.*` in `~/.viepilot/config.json`
- Global profiles (FEAT-009) are project-specific, not cross-project
- Brainstorm topics are general (Admin, Content, User Data) ‚Äî irrelevant for embedded/data-science/devops
- Phase templates are hardcoded ‚Äî wrong for ML pipeline, firmware, mobile
- Stack best practices must be re-declared per project
- Multi-domain professionals must manually set context every time

## Desired Behavior

### `/vp-persona` skill with wizard
5-question setup wizard (domain, role, stacks, team size, output style).
Saves to `~/.viepilot/personas/{name}.json`. Active persona read by all vp-* skills.

### Multi-persona architecture
```
~/.viepilot/
  persona.json            ‚Üê active persona (pointer)
  personas/
    web-saas.json         ‚Üê named persona 1
    embedded-iot.json     ‚Üê named persona 2
    data-science.json     ‚Üê named persona 3
  context-map.json        ‚Üê directory ‚Üí persona bindings
```

### Layered resolution (3 layers)
```
Priority: project > auto-map > global
1. .viepilot/persona-override.json   ‚Üê per-project explicit
2. context-map.json pattern match    ‚Üê directory-based auto
3. ~/.viepilot/persona.json (active) ‚Üê global default
```

### Auto-detection from project files
| Signal file | Detected domain |
|------------|-----------------|
| `package.json` + `prisma/` | web-saas |
| `CMakeLists.txt` + `sdkconfig` | embedded |
| `requirements.txt` + `notebooks/` | data-science |
| `pubspec.yaml` / `*.xcodeproj` | mobile |
| `Dockerfile` + `terraform/` | devops |
| `pyproject.toml` + `langchain` | ai-product |

### Persona schema
```json
{
  "name": "Web SaaS Dev",
  "domain": "web-saas",
  "role": "full-stack",
  "stacks": ["nextjs", "nestjs", "postgresql"],
  "team_size": "solo",
  "output_style": "lean",
  "brainstorm": {
    "topic_priority": ["auth", "user-data", "api", "admin"],
    "topic_skip": []
  },
  "phase_template": "lean-startup",
  "interests": ["ai-integration", "saas"]
}
```

### persona-context.md injection
All vp-* skills read `persona-context.md` and inject it as context. Example:
```
## User Persona
- Role: Full-stack developer, solo founder
- Domain: Web SaaS, AI-integrated products
- Preferred stacks: Next.js / NestJS / PostgreSQL
- Output style: lean
- Phase template: lean-startup
```

Community packs: `npm install viepilot-domain-fintech` ‚Üí auto-register.

### Switch + Merge
- **Switch**: `/vp-persona --switch embedded` ‚Äî instant context switch
- **Merge**: `/vp-persona --merge web-saas embedded-iot` ‚Üí creates hybrid persona (e.g. IoT Dashboard project)
  - Topics: union of both domains
  - Phase template: hybrid (Board Bring-Up + Web Scaffold ‚Üí MQTT Bridge + REST API ‚Üí Dashboard UI + OTA)
  - Architect pages: union of both

### Workflow adaptations
- **vp-brainstorm**: topics reordered by `topic_priority`; `topic_skip` topics hidden; domain pack topics injected; session-start prompt for multi-domain users
- **vp-auto**: stack preflight uses persona stacks; `persona-context.md` injected per task
- **vp-crystallize**: `output_style: lean` ‚Üí concise docs; `enterprise` ‚Üí full docs + compliance sections
- **vp-evolve**: new phase suggestions follow `phase_template`

### `/vp-persona` commands
```
/vp-persona                      ‚Üí Show active persona + list
/vp-persona --create <name>      ‚Üí Wizard to create new named persona
/vp-persona --switch <name>      ‚Üí Set active persona
/vp-persona --list               ‚Üí List all saved personas
/vp-persona --bind . <name>      ‚Üí Bind current project dir to persona
/vp-persona --auto               ‚Üí Enable auto-detect from project files
/vp-persona --merge <a> <b>      ‚Üí Create hybrid persona from two
/vp-persona --edit               ‚Üí Edit active persona fields
/vp-persona --export / --import  ‚Üí Backup/share/onboard
/vp-persona --reset              ‚Üí Clear all personas
```

## Summary
A new `/vp-persona` skill that creates a **user-level, cross-project identity** for ViePilot.
Persona is **inferred automatically** from project files, git history, and usage traces — no setup wizard,
no manual declarations, no repeated questions. Supports multiple named personas with auto-switch on
project change and auto-merge when multiple domains detected.

**Core principle:** Persona exists because AI reads context — not because user declares it.
Wizard = escape hatch (opt-in), never the primary path.

## Current Behavior
- ViePilot has no user-level identity — only `language.*` in `~/.viepilot/config.json`
- Global profiles (FEAT-009) are project-specific, not cross-project
- Brainstorm topics are general — irrelevant for embedded/data-science/devops
- Phase templates are hardcoded — wrong for ML pipeline, firmware, mobile
- Stack best practices must be re-declared per project

## Desired Behavior

### Auto-boot — zero setup required
On first invocation of any `vp-*` skill in a new directory:
1. Scan project files → detect domain (see signal table)
2. Infer `team_size` from `git shortlog -sn | wc -l` (1 = solo, 2–5 = small, 6+ = team)
3. Default `output_style: lean`; learned from calibration traces over time
4. Write `~/.viepilot/personas/auto-{slug}.json` silently
5. Set as active persona — no prompt, no confirmation

User never sees a setup screen unless they opt into `/vp-persona --refine`.

### Multi-persona architecture
```
~/.viepilot/
  persona.json            ← active persona pointer (auto-written)
  personas/
    auto-web-saas.json    ← auto-detected persona
    auto-embedded.json    ← auto-detected persona
    custom-iot-dash.json  ← user-created via --refine
  context-map.json        ← directory → persona bindings (auto-populated)
  traces/
    {session-id}.json     ← session traces (always-on, async)
  overlays/
    {persona}/
      brainstorm.patch.json
  persona-reflections.json
```

### Layered resolution (3 layers, fully automatic)
```
Priority: project > auto-map > global
1. .viepilot/persona-override.json   ← per-project explicit (rare, escape hatch)
2. context-map.json pattern match    ← directory-based auto (populated by --bind or auto-detect)
3. ~/.viepilot/persona.json (active) ← global default
```
Layer 2 is updated automatically when auto-detect runs in a new directory. No `--bind` needed
in normal flow — `--bind` exists only to correct wrong auto-detections.

### Auto-detection from project files
| Signal file | Detected domain |
|------------|-----------------|
| `package.json` + `prisma/` | web-saas |
| `CMakeLists.txt` + `sdkconfig` | embedded |
| `requirements.txt` + `notebooks/` | data-science |
| `pubspec.yaml` / `*.xcodeproj` | mobile |
| `Dockerfile` + `terraform/` | devops |
| `pyproject.toml` + `langchain` | ai-product |

**Multi-domain auto-merge:** When 2+ domain signals detected in the same project
(e.g. `CMakeLists.txt` + `package.json` → embedded + web-saas), AI automatically
creates a merged hybrid persona — no user prompt required.

### Inferred fields (no wizard)
| Field | Inference method |
|-------|-----------------|
| `domain` | Project file signals |
| `role` | Git blame pattern — majority file type → frontend/backend/fullstack/embedded/data |
| `stacks` | `package.json` deps + `requirements.txt` + `CMakeLists.txt` target names |
| `team_size` | `git shortlog -sn \| wc -l` → solo/small/team |
| `output_style` | Default `lean`; auto-upgraded to `balanced` after calibration detects multi-stakeholder traces |

### Persona schema
```json
{
  "name": "auto-web-saas",
  "source": "auto",
  "domain": "web-saas",
  "role": "full-stack",
  "stacks": ["nextjs", "nestjs", "postgresql"],
  "team_size": "solo",
  "output_style": "lean",
  "brainstorm": {
    "topic_priority": ["auth", "user-data", "api", "admin"],
    "topic_skip": []
  },
  "phase_template": "lean-startup",
  "interests": ["ai-integration", "saas"],
  "inferred_at": "2026-04-24T10:00:00Z",
  "confidence": 0.92
}
```

`confidence` reflects auto-detect certainty. Below 0.6 → persona still applied silently
but flagged in `pending-review.md` for optional user review (not a blocking prompt).

### persona-context.md injection
All vp-* skills read `persona-context.md` and inject it as context. Example:
```
## User Persona
- Role: Full-stack developer, solo founder
- Domain: Web SaaS, AI-integrated products
- Preferred stacks: Next.js / NestJS / PostgreSQL
- Output style: lean
- Phase template: lean-startup
```

### Auto-switch on project change
When a `vp-*` skill detects `$PWD` changed to a different project directory:
1. Check `context-map.json` for existing binding
2. If found → switch silently
3. If not found → re-run auto-detect → write binding → switch silently
4. No notification unless persona confidence < 0.6

### Domain Packs (5 built-in + community)
| Pack | Extra brainstorm topics | Phase template | Extra architect pages |
|------|------------------------|----------------|----------------------|
| `web-saas` | Billing, Multi-tenant, Trial/freemium, Onboarding | Auth→Core→Monetize→Scale | `billing.html`, `tenant.html` |
| `data-science` | Dataset mgmt, Model versioning, Drift monitoring, Eval | Data→EDA→Train→Serve→Monitor | `ml-pipeline.html`, `data-lineage.html` |
| `mobile` | Push notifications, Offline sync, App store, Deep links | Auth→Core→Offline→Submit | `mobile-flow.html`, `push.html` |
| `devops` | IaC, CI/CD, SLO/SLA, Incident management | Infra→CI/CD→Observe→Runbooks | `infra-topology.html`, `slo.html` |
| `ai-product` | LLM integration, RAG pipeline, Prompt mgmt, Eval/feedback | Wrapper→Prompt→RAG→Eval→UX | `llm-arch.html`, `rag-flow.html` |

Community packs: `npm install viepilot-domain-fintech` → auto-register.

### Workflow adaptations
- **vp-brainstorm**: topics reordered by `topic_priority`; `topic_skip` topics hidden; domain pack topics injected
- **vp-auto**: stack preflight uses persona stacks; `persona-context.md` injected per task
- **vp-crystallize**: `output_style: lean` → concise docs; `enterprise` → full docs + compliance sections
- **vp-evolve**: new phase suggestions follow `phase_template`

### `/vp-persona` commands
```
/vp-persona                      → Show active persona + confidence + inferred fields
/vp-persona --refine             → Edit active persona fields (opt-in correction, not onboarding)
/vp-persona --list               → List all saved personas
/vp-persona --bind . <name>      → Override auto-detection for current dir (correction)
/vp-persona --merge <a> <b>      → Manually create hybrid persona (auto-merge happens by default)
/vp-persona --export / --import  → Backup/share/onboard
/vp-persona --reset              → Clear all personas
```

`--create` and `--switch` removed from primary API — replaced by auto-detection and auto-switch.
`--calibrate` removed as standalone command — calibration is always-on (see Phase 5).

## Phase 5 — Adaptive Persona Calibration (AI-driven, always-on)

Calibration runs **automatically after every session** (async, non-blocking). No manual trigger needed.
Results appear in `pending-review.md` only for 🟡/🔴 changes — 🟢 applied immediately.

### Session Trace Logging (passive, always-on)
After each session, async write to `~/.viepilot/traces/{session-id}.json`:
```json
{
  "skill": "vp-brainstorm", "persona": "auto-web-saas",
  "topics_offered": ["auth","content-mgmt","admin","user-data"],
  "topics_discussed": ["auth","admin","user-data"],
  "topics_skipped": ["content-mgmt"],
  "stacks_mentioned": ["nextjs","postgresql","stripe"],
  "duration_min": 42
}
```

### Reflexion Memory (`~/.viepilot/persona-reflections.json`)
Pattern detection across N traces (≥3 signals = proposal threshold):
- Topic skip_rate ≥ 0.70 → add to `topic_skip` (🟢 auto-apply)
- Stack mentioned in ≥ 0.80 of sessions → add to persona stacks (🟢 auto-apply)
- Phase always added manually → add to `phase_template` (🟡 pending-review)
- Rejected proposals stored in `guardrail_journal` → never re-proposed

### Risk-tiered gates — revised for minimal interruption
| Risk | Examples | Behavior |
|------|----------|---------|
| 🟢 Low | Add stack, reorder topic priority, add topic_skip | **Auto-apply + silent log** |
| 🟡 Medium | Set phase default, change output_style | **Auto-apply + append to `pending-review.md`** (user reads when ready, can revert) |
| 🔴 High | Modify overlay logic, restructure phase template | **Block + single confirm prompt per session max** |

Key change from original design: 🟡 no longer interrupts the session. Applied immediately,
logged for async review. User is never blocked by a calibration decision during active work.

### Workflow Overlays (`~/.viepilot/overlays/{persona}/`)
JSON Patch RFC 6902 overlays — base workflow files never modified:
```json
// ~/.viepilot/overlays/auto-web-saas/brainstorm.patch.json
[
  { "op": "skip_step", "step_id": "topic_content_management",
    "reason": "skipped in 4/5 sessions" },
  { "op": "set_default", "step_id": "stack_selection",
    "value": ["nextjs","postgresql","stripe"] }
]
```
ViePilot upgrades update base workflows; overlays compose on top — non-destructive.

### `pending-review.md` (`~/.viepilot/pending-review.md`)
Async log for 🟡 auto-applied changes and low-confidence detections. Not shown during sessions.
Shown as a one-line notice at the **start** of the next session only (not per-task):
```
ℹ  3 persona updates applied — review ~/.viepilot/pending-review.md to revert any
```

## Detailed Design

### Phase 1 — Foundation (auto-detect + zero-setup)

Key redesign decisions:
1. **No wizard** — inferPersona() replaces 5-question setup. First run is silent.
2. **No manual switch** — auto-switch on $PWD change replaces `--switch` command.
3. **Auto-merge** — 2+ domain signals → hybrid persona without user prompt.
4. **🟡 non-blocking** — medium-risk calibration applied immediately, logged async.
5. **Always-on calibration** — no `--calibrate` trigger needed; runs after every session.
6. **team_size from git** — `git shortlog -sn | wc -l` replaces wizard question.
7. **output_style from traces** — starts lean, self-adjusts; never asked.

Reflexion pattern (arxiv 2303.11366) for calibration memory.

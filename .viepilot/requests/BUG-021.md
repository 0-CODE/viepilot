# BUG: Antigravity adapter install path outdated тАФ skills must be at `~/.gemini/antigravity/skills/`

## Summary
After a recent Antigravity update, the IDE no longer discovers skills from `~/.antigravity/skills/`. The new discovery path is `~/.gemini/antigravity/skills/`. ViePilot's Antigravity adapter still installs to the old path, so all vp-* skills are silently inaccessible after a fresh install (or update) on the new version of Antigravity.

## Root Cause
The Antigravity IDE changed its skill discovery directory:

| | Old path (pre-update) | New path (post-update) |
|-|-----------------------|------------------------|
| Skills | `~/.antigravity/skills/` | `~/.gemini/antigravity/skills/` |
| ViePilot runtime files | `~/.antigravity/viepilot/` | `~/.gemini/antigravity/viepilot/` |

Evidence: Antigravity agent self-diagnosed and fixed by copying `~/.antigravity/skills/vp-*` тЖТ `~/.gemini/antigravity/skills/`. Skills were immediately recognized after the copy.

## Actual Behavior
1. User runs `vp-tools install --target antigravity` (or Node installer)
2. Skills installed to `~/.antigravity/skills/vp-*/`
3. Antigravity IDE does NOT discover them (new version looks in `~/.gemini/antigravity/skills/`)
4. User must manually copy as a workaround

## Expected Behavior
`vp-tools install --target antigravity` installs skills directly to `~/.gemini/antigravity/skills/` so they are immediately recognized without any manual steps.

## Files to Update

### `lib/adapters/antigravity.cjs`
```js
// BEFORE
skillsDir:            (home) => path.join(home, '.antigravity', 'skills'),
viepilotDir:          (home) => path.join(home, '.antigravity', 'viepilot'),
executionContextBase: '.antigravity/viepilot',
isAvailable: (home) => fs.existsSync(path.join(h, '.antigravity'))

// AFTER
skillsDir:            (home) => path.join(home, '.gemini', 'antigravity', 'skills'),
viepilotDir:          (home) => path.join(home, '.gemini', 'antigravity', 'viepilot'),
executionContextBase: '.gemini/antigravity/viepilot',
isAvailable: (home) => fs.existsSync(path.join(h, '.gemini', 'antigravity'))
             || fs.existsSync(path.join(h, '.antigravity'))   // fallback for old installs
```

## Discussion
The Antigravity IDE moved from a standalone `.antigravity/` directory structure to a Google Gemini-namespaced structure (`.gemini/antigravity/`), consistent with Antigravity being rebranded/integrated into the Gemini ecosystem. The fix is a simple path update in the adapter; the two-path `isAvailable` check provides graceful backward compatibility for users who haven't updated Antigravity yet.

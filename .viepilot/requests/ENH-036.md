## Summary
Remove `install.sh` and `dev-install.sh` from the repo. The codebase now uses a full Node-based installer (`bin/viepilot.cjs install`) that covers all functionality both scripts provided. Keeping them creates confusion and maintenance burden.

## Details

### install.sh (125 lines)
- Self-described as "thin wrapper around the Node installer (bin/viepilot.cjs)"
- Has hardcoded `.cursor/skills` / `.cursor/viepilot` paths — outdated since FEAT-013 (dynamic adapter system, Phase 53) and ENH-035 (`{envToolDir}` template, Phase 58)
- Prompts user about cloc; delegates all real work to Node CLI
- **Verdict:** Fully redundant. `bin/viepilot.cjs install` replaces it entirely.

### dev-install.sh (176 lines)
- Bash-level adapter routing (`VIEPILOT_ADAPTER` → `SKILLS_DIR` / `VIEPILOT_DIR`)
- Duplicates `lib/adapters/` registry (claude-code, cursor, antigravity cases)
- Updated in Phase 59 (FEAT-014) but the adapter logic now lives in `lib/adapters/`
- `bin/viepilot.cjs install --target <adapter>` handles all these cases
- **Verdict:** Redundant. Node installer is the authoritative path.

### Side effects to clean up
1. `package.json` `files` array — remove both entries
2. `README.md` line 72 (Installation table) and line 422 (Project Structure) — update references
3. `tests/unit/viepilot-adapters.test.js` Group 4 — 2 tests asserting `dev-install.sh` content; remove or migrate to Node-CLI equivalents
4. `tests/unit/vp-adapter-antigravity.test.js` — 2 tests asserting `dev-install.sh` has antigravity case; remove or migrate

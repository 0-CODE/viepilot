# BUG: Git tags thiếu project prefix — collision khi chạy nhiều dự án

## Meta
- **ID**: BUG-006
- **Type**: Bug
- **Status**: done
- **Priority**: high
- **Severity**: high
- **Created**: 2026-04-03
- **Reporter**: User
- **Assignee**: AI
- **Affects**: `workflows/crystallize.md`, `workflows/evolve.md`, `workflows/autonomous.md`, `templates/phase/TASK.md`

## Summary

`vp-auto` tạo git tags theo format `vp-p{N}-t{N}` (không có project prefix), dẫn đến **tag collision** khi chạy nhiều dự án trong cùng git hoặc các dự án có cùng phase/task numbering. Tag đúng theo spec phải là `{projectSlug}-vp-p{N}-t{N}` (e.g., `smart-track-platform-vp-p13-t13.1`).

## Actual Behavior

Task files được generate bởi `crystallize`/`evolve` có Git Tag field dạng:
```
- **Git Tag**: vp-p13-t13.1       ← WRONG (no prefix)
- **Git Tag**: vp-p6-t6.1         ← WRONG (Phase 06 viepilot project cũng bị)
```

Tags được tạo (hoặc AI hiểu cần tạo) theo format không prefix — ví dụ:
```bash
git tag vp-p12-t12.1       ← collision nếu dự án khác cũng có phase 12 task 1
git tag vp-p13-t13.1-done  ← collision
```

## Expected Behavior

Tags phải có project slug prefix (từ `node bin/vp-tools.cjs tag-prefix --raw`):
```bash
git tag smart-track-platform-vp-p13-t13.1
git tag smart-track-platform-vp-p13-t13.1-done
git tag smart-track-platform-vp-p13-complete
```

Task files phải show:
```
- **Git Tag**: smart-track-platform-vp-p13-t13.1
```

## Root Cause Analysis

### RC-1: crystallize.md + evolve.md không resolve `{projectPrefix}`

Khi generate task files, `{projectPrefix}` placeholder trong template không được resolve thành giá trị thực. Template có:
```
- **Git Tag**: {projectPrefix}-vp-p{{PHASE_NUMBER}}-t{{TASK_NUMBER}}
```

Nhưng AI không call `vp-tools tag-prefix --raw` để lấy giá trị, nên output ra:
```
- **Git Tag**: vp-p13-t13.1   ← prefix bị bỏ qua
```

### RC-2: autonomous.md dùng prose variable `{projectPrefix}`

Trong narrative text:
```
Create git tag: `{projectPrefix}-vp-p{phase}-t{task}`
```

Đây là AI-resolved variable — AI không nhất quán apply. Bash commands có `TAG_PREFIX=$(node bin/vp-tools.cjs tag-prefix --raw)` nhưng các instruction khác không.

### RC-3: `isCheckpointTag()` accept cả format không prefix (v1 legacy)

```js
// cli-shared.cjs line 50
return /^vp-p\d+-(?:t\d+(?:-done)?|complete)$/.test(value)   // ← matches no-prefix (v1)
    || /^[a-z0-9][a-z0-9-]*-vp-p\d+-(?:t\d+(?:-done)?|complete)$/.test(value);  // ← with prefix
```

Không có warning khi detect no-prefix tag — silent acceptance tạo false sense of correctness.

### RC-4: Tag fields trong Phase 06 task files cũng bị

Task files vừa tạo cho Phase 06 trong session này bị tương tự:
- `task-6.1-state-update-checklist.md`: `- **Git Tag**: vp-p6-t6.1` (thiếu `viepilot-vp-` prefix)

```js
// lib/cli-shared.cjs — tag prefix source of truth (đúng):
function getCheckpointTagPrefix(root) {
  return `${getProjectSlug(root)}-vp`;   // → "smart-track-platform-vp" or "viepilot-vp"
}
```

## Impact

- Multi-project developers: tag collision khi 2 projects đang đồng thời ở phase 1
- `git revert --no-commit $(git rev-list {TAG}..HEAD)` rollback logic bị sai project nếu tags trùng
- `vp-status` và `vp-rollback` có thể reference wrong tag từ wrong project
- Phase 06 task files vừa tạo cần fix thủ công hoặc sẽ tự-fix khi vp-auto chạy đúng

## Scope of Fix

### Fix A — `workflows/crystallize.md` và `workflows/evolve.md`

Khi generate task files, thêm explicit instruction:
```
Before populating Git Tag field:
  TAG_PREFIX=$(node bin/vp-tools.cjs tag-prefix --raw)
  # or: derive slug from git repo name = basename(git rev-parse --show-toplevel)
  Git Tag: {TAG_PREFIX}-p{phase}-t{task}
```

### Fix B — `workflows/autonomous.md`

Replace tất cả prose `{projectPrefix}-vp-p{N}-t{N}` bằng:
```
TAG_PREFIX=$(node bin/vp-tools.cjs tag-prefix --raw)
git tag "${TAG_PREFIX}-p{phase}-t{task}"
```

Đảm bảo **mọi** git tag creation trong autonomous.md dùng bash variable, không prose placeholder.

### Fix C — `templates/phase/TASK.md`

Thay `{projectPrefix}-vp-p{{PHASE_NUMBER}}-t{{TASK_NUMBER}}` bằng:
```
- **Git Tag**: `$(node bin/vp-tools.cjs tag-prefix --raw)-p{{PHASE_NUMBER}}-t{{TASK_NUMBER}}`
  # Resolved at runtime by vp-auto — do not hardcode
```

Hoặc thêm note:
```
> **Git Tag resolves at runtime**: AI fills this with `$(TAG_PREFIX)-p{N}-t{N}` where TAG_PREFIX = `node bin/vp-tools.cjs tag-prefix --raw`
```

### Fix D — `lib/cli-shared.cjs` (optional, low-priority)

Add warning khi `isCheckpointTag()` detects no-prefix v1 format:
```js
if (/^vp-p\d+-/.test(tag)) {
  console.warn('[vp-tools] Tag "%s" uses legacy no-prefix format. Expected: %s-p...', tag, getCheckpointTagPrefix());
}
```

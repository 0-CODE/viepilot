## Summary
`vp-tools info` (and `/vp-info` skill) crashes with "Could not locate viepilot package root" when run from any directory that is not the viepilot source repo.

## Details

### Actual Behavior
```
✖ Error: Could not locate viepilot package root
  Hint: Install viepilot globally, use npx from a project with viepilot, or run from a viepilot clone
Exit code 1
```

### Expected Behavior
`vp-tools info` succeeds regardless of CWD.

### Root Cause
`resolveViepilotPackageRoot()` in `lib/viepilot-info.cjs` finds the package root by walking up from:
1. `path.join(__dirname, '..')` → `~/.claude/viepilot/` — **no `package.json` here**
2. `process.cwd()` — only works if CWD happens to be the viepilot source repo

The install script (`lib/viepilot-install.cjs`, claude-code target block ~line 290) copies
`bin/`, `lib/`, `workflows/`, `templates/` to `~/.claude/viepilot/` but **does NOT copy `package.json`**.

So path #1 always fails for installed users, and path #2 is a lucky coincidence.

### Affected Files
- `lib/viepilot-install.cjs` — missing `copy_file` step for `package.json`

### Steps to Reproduce
1. Install viepilot with claude-code target (so files land in `~/.claude/viepilot/`)
2. `cd` to any directory that is NOT the viepilot source repo
3. Run `node ~/.claude/viepilot/bin/vp-tools.cjs info`
4. → Error: Could not locate viepilot package root

### Severity
High — `/vp-info` skill is broken for any user not actively working inside the viepilot source repo.

# BUG: vp-auto edit install path thay vì codebase source

## Meta
- **ID**: BUG-007
- **Type**: Bug
- **Status**: done
- **Priority**: high
- **Severity**: high
- **Created**: 2026-04-03
- **Reporter**: User
- **Assignee**: AI
- **Affects**: `workflows/autonomous.md`, `templates/project/AI-GUIDE.md`

## Summary

vp-auto/AI cố edit `~/.claude/viepilot/` (install path) thay vì project codebase working dir (`{project_cwd}`). Phát hiện trong session BUG-005/BUG-006 fix: AI đã bắt đầu edit `~/.claude/viepilot/workflows/autonomous.md` thay vì `/Users/sonicq12/DEV_PROJECTS/viepilot/workflows/autonomous.md`. User phải interrupt và correct thủ công.

## Actual Behavior

Trong framework development sessions (project codebase = ViePilot source):
- vp-auto reads workflow từ `~/.claude/viepilot/workflows/autonomous.md` (install path)
- AI infers "edit target = where I read from"
- Edit goes to install path → bypasses all testing, ships directly to production

## Expected Behavior

- Edit target ALWAYS = `{project_cwd}` (working directory of project being developed)
- Install path (`~/.claude/viepilot/`, `~/.cursor/viepilot/`) = READ ONLY for AI
- Attempt to write to install path → hard stop → control point

## Root Cause

### RC-1: autonomous.md không có working directory guard

`workflows/autonomous.md` Initialize section không có assertion:
```
Working directory: {project_cwd}
Edit targets: files under {project_cwd} ONLY
```

AI infers context from where it read the workflow file (install path) instead of asserting the project cwd.

### RC-2: AI-GUIDE.md template không có install path warning

`templates/project/AI-GUIDE.md` không có explicit note ngăn AI edit install path. Framework dev sessions không được warn.

### RC-3: Không có detection condition

Không có heuristic: "nếu file path starts with `~/.{tool}/viepilot/` → hard stop".

## Impact

- Framework development sessions có thể silently ship unreviewed code to production
- Install path edit bypasses git history (changes go directly to `~/.claude/`, not tracked)
- User cần manual vigilance — không có safety net

## Scope of Fix

### Fix A — `workflows/autonomous.md`: Working Directory Guard block

Thêm vào Initialize section:

```markdown
#### Working Directory Guard (mandatory — evaluate before any file edit)

**Source of truth**: `{project_cwd}` = current working directory (where TRACKER.md lives)
**Read-only paths**: `~/.claude/viepilot/`, `~/.cursor/viepilot/`, any install path outside `{project_cwd}`

Before EVERY file edit:
1. Confirm target path is under `{project_cwd}`
2. If target path starts with `~/` or absolute path outside `{project_cwd}` → STOP
3. Route to control point: "Edit target `{path}` is outside project working directory. Is this intentional?"

Install paths are loaded for context only — NEVER write to them.
```

### Fix B — `templates/project/AI-GUIDE.md`: Install path warning

Thêm vào Static Context section:

```markdown
> **Install path warning**: `~/.claude/viepilot/` and `~/.cursor/viepilot/` are READ-ONLY runtime paths.
> All edits must target files under this project's working directory only.
> Writing to install paths bypasses version control and ships untested code to production.
```

# ENH-056: Skill invocation greeting banner with version number

## Summary
When a skill is invoked in newer versions of Claude Code, the UI no longer shows whether the skill file was read or not. Users cannot confirm that the skill/workflow was actually loaded. Each skill invocation should output a greeting banner that includes the workflow/skill name and version number so the user knows exactly which version is running.

## Details
- **Affected**: All `vp-*` skills + their linked workflows
- **Root cause**: Claude Code UI change — the skill loading indicator is no longer shown to the user
- **Current behavior**: Skill is silently loaded; no confirmation that the workflow was read
- **Desired behavior**: On every skill invocation, the AI immediately prints a greeting such as:
  ```
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   VIEPILOT ► VP-AUTO  v0.2.2 (2.18.0)
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ```
  where the version shows both skill semver and framework semver.

## Acceptance Criteria
- [ ] Each `vp-*` SKILL.md has an explicit `<greeting>` block or `## Greeting` section instructing the AI to output the banner immediately on invocation
- [ ] Banner includes: skill name, skill version, framework version
- [ ] Banner is printed BEFORE any questions or work begins
- [ ] Pattern is consistent across all 17 skills
- [ ] Contract test verifies each SKILL.md has a greeting marker

## Discussion
The greeting serves a dual purpose:
1. **Debugging**: confirms the correct version was loaded from `~/.claude/skills/`
2. **UX**: gives the user immediate feedback that the skill was recognised and is running

The skill version (from `SKILL.md` frontmatter `version:` field) + framework version (from `package.json`) should both appear in the banner. The framework version can be read from `~/.claude/viepilot/` or hardcoded at install time.

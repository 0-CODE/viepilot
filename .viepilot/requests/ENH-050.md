# ENH: Git Tag Format — Include Branch + Version

## Summary
Git tags created by `vp-auto` currently follow the pattern `{projectPrefix}-vp-p{phase}-t{task}`. They should include the active branch and current version so tags are fully self-describing and sortable by context.

## Details

### Current Format
```
{projectPrefix}-vp-p{phase}-t{task}
{projectPrefix}-vp-p{phase}-t{task}-done
{projectPrefix}-vp-p{phase}-complete
```
Example: `viepilot-vp-p77-t4`, `viepilot-vp-p77-complete`

### Desired Format
```
{projectPrefix}-{branch}-{version}-vp-p{phase}-t{task}
{projectPrefix}-{branch}-{version}-vp-p{phase}-t{task}-done
{projectPrefix}-{branch}-{version}-vp-p{phase}-complete
```
Example: `viepilot-main-2.15.0-vp-p77-t4`, `viepilot-main-2.15.0-vp-p77-complete`

### Format Resolution at Runtime
```bash
PROJECT_PREFIX=$(node bin/vp-tools.cjs info --prefix 2>/dev/null || \
  cat .viepilot/PROJECT-META.md | grep -i "^prefix:" | awk '{print $2}')
BRANCH=$(git rev-parse --abbrev-ref HEAD)
VERSION=$(node -e "console.log(require('./package.json').version)" 2>/dev/null || \
  grep '"version"' package.json | head -1 | sed 's/.*"\([0-9.]*\)".*/\1/')
TAG="${PROJECT_PREFIX}-${BRANCH}-${VERSION}-vp-p${PHASE}-t${TASK}"
```

### Scope of Changes
- `workflows/autonomous.md` — update all 3 tag patterns (start, done, complete)
- `workflows/audit.md` — update tag detection regex patterns for `COMPLETE_TAGS` + `PREV_TAG`
- `workflows/rollback.md` — update tag grep pattern
- `lib/vp-tools.cjs` or `bin/vp-tools.cjs` — if tag creation is also implemented in CLI (check `git-tag` subcommand)
- `tests/unit/` — ≥5 contract tests on new format pattern + regex
- `CHANGELOG.md` + version bump

### Backward Compatibility
- Existing tags (legacy `viepilot-vp-p*`) remain in repo; no migration needed.
- Audit + rollback regex must match BOTH old and new formats during transition period.

### Why Valuable
- Branch in tag → immediately see which branch a checkpoint was taken on (useful for feature branches)
- Version in tag → immediately correlate a checkpoint with the package version at that point
- Fully self-describing → no need to cross-reference TRACKER/ROADMAP to understand what `viepilot-vp-p60-complete` refers to

## Discussion
Dots in version string (`2.15.0`) are valid in git tag names. Branch names with `/` (e.g. `feature/foo`) should be sanitized to `-` for safe tag names:
```bash
BRANCH_SAFE=$(git rev-parse --abbrev-ref HEAD | sed 's/[^a-zA-Z0-9._-]/-/g')
```

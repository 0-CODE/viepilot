## Summary
Add a `browser-audit-agent` Claude Code native agent that uses `vercel-labs/agent-browser` to audit a running web app — navigating routes, taking screenshots, checking visual regressions, validating forms and links, and running accessibility checks. Triggered via `vp-audit --visual` or `vp-audit --browser`.

## Problem Statement
Current `vp-audit` (via `file-scanner-agent`) is limited to static analysis: code quality, file structure, test coverage, documentation completeness. It has **no visibility into runtime behavior** — a page may pass all static checks but render broken in the browser.

Missing coverage:
- Visual layout / CSS regressions
- Broken navigation routes (404s, redirect loops)
- Form submission / interactive element behavior
- Accessibility violations (missing ARIA, poor contrast)
- JS console errors on page load

## Proposed Solution

### New agent: `browser-audit-agent` (agents/claude-code/)
```yaml
name: browser-audit-agent
model: claude-sonnet-4-6
tools: [Bash, Read, Write]
disallowedTools: [Edit, WebSearch, WebFetch]
# Prerequisite: npx skills add vercel-labs/agent-browser
```

**Operations:**
- `audit_routes` — open base URL, discover links, navigate each route, screenshot + record errors
- `visual_check` — compare current screenshot against baseline (if exists in `.viepilot/audit/baselines/`)
- `accessibility_check` — use `snapshot -i` to read ARIA structure, flag missing labels, poor semantic structure
- `form_check` — detect form elements via refs, attempt submit with test data, verify response

### Integration in `skills/vp-audit/SKILL.md`
New flag: `--visual` / `--browser <baseUrl>`
- Default base URL: `http://localhost:3000` (or from `.viepilot/config.json` → `audit.baseUrl`)
- Step 5 (new): Browser Audit → dispatch `Agent({ subagent_type: "browser-audit-agent", op: "audit_routes", baseUrl })`
- Output: `.viepilot/audit/visual-report-{timestamp}.md` with screenshots + findings

### Visual regression baseline
- First run: saves screenshots to `.viepilot/audit/baselines/{route-slug}.png`
- Subsequent runs: compares using `agent-browser screenshot` + pixel diff (if `pixelmatch` available) or Claude Vision comparison
- `--update-baseline` flag to accept current state as new baseline

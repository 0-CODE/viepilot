- **Breaking**: NO

## Summary
Tích hợp tool `AskUserQuestion` (Claude Code native tool) vào toàn bộ vp-* skills để thay thế plain-text numbered list khi hỏi user. Khi adapter không hỗ trợ tool này (Cursor, Codex, antigravity...), tự động fallback về text prompt.

## Current Feature
Tất cả vp-* skills (crystallize, brainstorm, request, evolve, auto, v.v.) hiện đặt câu hỏi cho user thông qua plain-text output dạng numbered list:
```
1. What is the project name?
2. Which stack?
3. ...
```
User phải gõ số hoặc text trả lời thủ công.

## Current Behavior
- `vp-crystallize`: hỏi project name, stack, UI direction bằng text block
- `vp-brainstorm`: hỏi topic, goals, constraints bằng text block
- `vp-request`: hỏi loại request, details bằng numbered list
- Không có structured choice UI, không có multi-select, không có descriptions cho từng option

## Desired Improvement
Khi chạy trong Claude Code (có `AskUserQuestion` available):
- Dùng `AskUserQuestion` tool với structured options, descriptions, multiSelect
- Nhóm tối đa 4 câu hỏi per call (giới hạn của tool)
- Dùng `preview` field cho các lựa chọn cần so sánh (stack options, UI layouts)

Khi chạy trong adapter không hỗ trợ (Cursor, Codex, antigravity):
- Fallback tự động về plain-text numbered list hiện tại
- Không break existing behavior

## Adapter Compatibility Research

_Web research conducted 2026-04-13._

### Claude Code (official)
- `AskUserQuestion` — **fully supported** (native tool, renders interactive UI component with click-to-select options, multi-select, preview panels)
- Source: Anthropic Claude Code system tools documentation
- Related: [anthropics/claude-code#12609](https://github.com/anthropics/claude-code/issues/12609) — feature request for VS Code extension parity (VS Code extension không có interactive UI như terminal)

### Cursor
- Tool set của Cursor agent: `list_dir`, `codebase_search`, `read_file`, `edit_file`, `run_terminal_cmd`, `web_search`, v.v.
- Cursor có **`AskQuestion` tool** — nhưng **chỉ hoạt động trong Plan Mode** (với composer-1.5), **không available trong Agent Mode**
- Đây là feature gap đang được request: [forum.cursor.com — Allow AskQuestion in Agent Mode](https://forum.cursor.com/t/allow-askquestion-tool-calls-in-agent-mode-or-any-mode/152517)
- Cursor 3 / v2.6.20 (Mar 2026) bổ sung interactive UIs trong agent chats — nhưng qua **MCP Apps** (Amplitude, Figma, tldraw), không phải AskQuestion natively
- **Verdict cho ViePilot**: vp-* skills chạy qua Cursor **Skills** (Agent Mode) → `AskQuestion` **NOT available** → **fallback required**

### OpenAI Codex CLI
- Codex CLI có **`ask_user_question` tool** (structured Q&A, constrained answers, tabbed UI)
- Giao diện: bottom input area với row of tabs — một tab per question + Submit tab, keyboard navigable
- Tuy nhiên tool này **không phải native Codex tool** — đây là **community skill/MCP server**:
  - [sanchomuzax/AskUserQuestion-tool-for-Codex](https://github.com/sanchomuzax/AskUserQuestion-tool-for-Codex) — standalone skill
  - [paulp-o/ask-user-questions-mcp](https://github.com/paulp-o/ask-user-questions-mcp) — MCP server
  - Issue: [openai/codex#9926](https://github.com/openai/codex/issues/9926) — enhancement request cho tabbed questionnaire UI
- Codex native tool set **không bao gồm** interactive question UI out-of-the-box
- **Verdict**: không thể dựa vào tool availability — **fallback required**

### Google Antigravity
- Agent-first IDE (VS Code fork), hỗ trợ Gemini 3, Claude Sonnet/Opus 4.6, GPT-OSS-120B
- Antigravity **không hỗ trợ MCP natively** (significant limitation vs Cursor/Claude Code)
- Common workflow: Antigravity (Gemini) → Claude Code extension bên trong Antigravity
- Khi dùng Claude Code extension bên trong Antigravity: `AskUserQuestion` **có thể hoạt động** (vì Claude Code extension chạy terminal-based)
- Khi dùng Antigravity agent trực tiếp (Gemini): **không rõ support** — agent dùng "Artifacts" model thay vì raw tool calls
- **Verdict**: không đủ thông tin confirm → **fallback required** (conservative)

### Windsurf / JetBrains / others
- Không trong scope hiện tại của ViePilot adapters
- **Verdict**: **fallback required** (conservative default)

### Fallback strategy
```
try: AskUserQuestion tool call
→ if tool available + response received: use structured answers
→ if tool NOT available / error: emit plain-text numbered list and wait for text reply
```

Since skills are markdown-based prompts (not code), the "try/fallback" logic must be expressed as **conditional instruction** in the workflow/skill file:

```markdown
**Interactive prompt (adapter-aware):**
- Claude Code (terminal): use `AskUserQuestion` tool with structured options
- Cursor, Codex, Antigravity, other adapters: use plain-text numbered list below
```

Apply to all skills that currently ask user questions:

## Value Proposition
- UX tốt hơn đáng kể trong Claude Code: user click chọn thay vì gõ số
- Multi-select hỗ trợ chọn nhiều option cùng lúc (e.g. features, stack components)
- Descriptions giúp user hiểu trade-off mà không cần đọc thêm docs
- Preview field cho phép so sánh visual (e.g. architecture options)
- Không break Cursor/other adapters qua graceful fallback

## Implementation Notes
- Tool schema: max 4 questions, 2-4 options each, optional multiSelect, optional preview
- Instruction pattern trong workflow files:
  ```
  > **Claude Code**: use AskUserQuestion tool
  > **Other adapters**: use text list below
  ```
- Ưu tiên crystallize + brainstorm trước (highest frequency của user interaction)
- request skill có thể dùng AskUserQuestion ngay lần đầu để detect type

## Summary
`vp-brainstorm` has two powerful visual modes — **Architect Design Mode** and **UI Direction Mode** — but they are surfaced to the user very differently. Architect Design shows a prominent activation banner when heuristics fire. UI Direction runs silently in the background until ≥5 signals accumulate or the user manually triggers `/save`. Users who would benefit from UI Direction Mode never get invited to use it.

## Details

### Current behavior gap

**Architect Design Mode (proactive):**
- Auto-activate heuristic: fires after ≥3 component/service mentions OR ≥1 stack suggestion
- Shows an immediate, visible banner:
  ```
  🏗️ I noticed you are designing an architecture with multiple components.
  Activate Architect Design Mode so I can create an HTML visualization?
  ```
- User sees the option and can choose → high discoverability

**UI Direction Mode (silent / passive):**
- Signal keywords detected silently in background
- Accumulation threshold: ≥3 unique keyword occurrences to start buffering
- Surface triggers: topic change (`/topic`), `/save`, `/review`, OR ≥5 accumulated signals
- No proactive banner at session start or on first UI keyword
- Result: users miss UI Direction Mode unless they know to ask for `--ui` explicitly

### Expected behavior (parity with Architect)
When `vp-brainstorm` detects UI/UX-relevant context (≥2 unique signal keywords OR first explicit UI phrase), it should show a **proactive suggestion** comparable to Architect:

```
🎨 Tôi nhận thấy session này liên quan đến UI/UX design.
Activate UI Direction Mode để tôi tạo HTML prototype direction?
  1. Yes — tạo workspace (.viepilot/ui-direction/{session-id}/)
  2. Save ideas to notes only (background)
  3. No, tiếp tục brainstorm thôi
```

This mirrors the Architect pattern: **detect → prompt → activate** instead of silent accumulation.

### Proposed changes to `workflows/brainstorm.md`

1. **Lower the surface threshold**: reduce from ≥5 accumulated signals to **≥2 unique signal keywords** before showing the activation prompt (matching Architect's lower bar)
2. **Add a proactive banner** in the "Surface triggers" section — triggered after ≥2 signals, not just at topic-end or `/save`
3. **Add an early-session check**: at the start of each brainstorm session where the user's description mentions UI/UX/design keywords, immediately suggest `--ui` mode (similar to how Architect suggests itself when `/vp-brainstorm --architect` is mentioned or heuristics fire early)
4. **Update `skills/vp-brainstorm/SKILL.md`** — document that UI Direction Mode auto-suggests itself just like Architect Mode

# ENH: vp-audit Tier 4 Silent Mode — Show Only When Issues Found

## Summary
Tier 4 (Framework Integrity) checks should run silently. If the check passes (✅) or is skipped (not a viepilot framework repo), the result must NOT appear in the audit report or banner. Only surface Tier 4 output when issues are found (⚠️).

## Details

### Current Behavior
In `workflows/audit.md` → Step 5 report, Tier 4 always appears in the summary banner:
```
 Tier 4: Framework          {✅ In sync | ℹ️ Skipped}
```
Even when the result is clean (✅) or not applicable (ℹ️ Skipped), the line is printed — adding noise for the user.

### Desired Behavior
- **✅ clean**: Tier 4 line omitted entirely from report banner and full report
- **ℹ️ Skipped** (not a framework repo): Tier 4 line omitted entirely — no output whatsoever
- **⚠️ issues found**: Tier 4 line shown normally (same as today) so user can act

Net effect: user only sees Tier 4 when it actually needs attention. Zero noise otherwise.

### Scope of Change
- `workflows/audit.md` — Step 4 (`→ Tier 4 skipped` echo removed) + Step 4f report block (conditional) + Step 5 All Clear banner (remove Tier 4 line) + Step 5 Issues banner (keep, conditional)
- `skills/vp-audit/SKILL.md` — update success criteria / behavior description if it mentions Tier 4 visibility
- Tests: `tests/unit/vp-audit*.test.js` or contract tests — assert Tier 4 line absent from clean-run output strings; assert present when issues exist

### Why
Tier 4 is an internal framework-health check. When running `vp-audit` from a user project, "Tier 4 Skipped" is expected and irrelevant — showing it every time trains users to ignore it. When running from the viepilot repo itself, a passing Tier 4 is also just noise. Surface only when action is needed.

## Acceptance Criteria
- [ ] `workflows/audit.md` Step 4: no `echo "→ Tier 4 skipped"` output when skipping
- [ ] `workflows/audit.md` Step 4f: Tier 4 report block wrapped in `if [ TIER4_ISSUES -gt 0 ]` guard
- [ ] `workflows/audit.md` Step 5 All Clear banner: Tier 4 line removed (not conditional — just gone)
- [ ] `workflows/audit.md` Step 5 Issues banner: Tier 4 line shown only when `TIER4_ISSUES -gt 0`
- [ ] `skills/vp-audit/SKILL.md`: description reflects silent-by-default behavior
- [ ] Contract tests verify: clean run → no "Tier 4" string in banner; issue run → "Tier 4" present

# BUG: .viepilot/ tracked by git despite being in .gitignore — vp-auto repeatedly stages & pushes it

## Meta
- **ID**: BUG-013
- **Type**: Bug
- **Status**: new
- **Priority**: high
- **Created**: 2026-04-10
- **Reporter**: User
- **Assignee**: AI

## Summary
`.viepilot/` is listed in `.gitignore` but its files are still **tracked** by git (they were committed before the ignore rule was added). As a result:
- `git status --porcelain` always shows `.viepilot/` files as modified/staged
- `vp-auto` commit steps blindly include them with `git add .viepilot/...`
- The git-persistence gate sees "porcelain not clean" and loops
- User has to intervene every session

## Root Cause
`git rm -r --cached .viepilot/` was never run after adding `.viepilot/` to `.gitignore`. Tracked files are never suppressed by `.gitignore` — only untracked files are.

## Expected Behavior
1. `.viepilot/` files should be **untracked** from the repository (one-time `git rm --cached`)
2. `vp-auto` commit step should **never** attempt to stage files that are listed in `.gitignore`
3. Git-persistence gate should treat `.viepilot/` as outside scope — only check tracked shipping files

## Fix Plan

### Part A — Immediate repo fix (one-time, can be done in Phase 57)
```bash
git rm -r --cached .viepilot/
git commit -m "chore: untrack .viepilot/ (gitignored internal state)"
```

### Part B — Framework guard in autonomous.md
Add a **GITIGNORE-AWARE STAGING RULE** to the commit section of `autonomous.md`:
- Before `git add <file>`, check if the file is gitignored: `git check-ignore -q <file>`
- If gitignored → skip; never stage gitignored files
- State files (TRACKER.md, ROADMAP.md, HANDOFF.json) that are in `.viepilot/` are framework-internal; they are NOT shipped artifacts

### Part C — Git-persistence gate clarification
The gate (`git status --porcelain` must be empty) must only evaluate **tracked** files. Untracked (`??`) and gitignored files must be excluded from the PASS check.

## Problem Statement

**Admin & Governance** (đã ship) = *ai vận hành hệ thống* — admin persona, RBAC, audit log, monitoring, billing.
**Content Management** (thiếu) = *nội dung gì được tạo và quản lý trong hệ thống* — CMS, articles, pages, media, catalog, taxonomy.

Các loại dự án thường cần Content Management:
- Blog / publishing platform
- eCommerce (product catalog, inventory)
- SaaS có landing pages / docs / knowledge base
- Marketplace (listing management)
- LMS (course, lesson, quiz)
- App có CMS-driven content (promotions, banners, notifications template)

Nếu không khai thác sớm, crystallize và ROADMAP sẽ thiếu hẳn content layer → discovered late → rework tốn kém.

## Desired Behavior

### brainstorm — Topic 7: Content Management

Thêm Topic 7 trong Topics Template:

```
7. **Content Management**
   - What types of content exist in this system? (articles, products, pages, media, courses, listings...)
   - Who creates content? (admin only, verified users, any user, API import)
   - Content lifecycle? (draft → review → published → archived)
   - Rich text or structured fields? (WYSIWYG, markdown, JSON schema, headless CMS)
   - Media management? (image upload, video, file attachments, CDN, storage limits)
   - Taxonomy & organization? (categories, tags, collections, folders)
   - Multi-language / localization? (which locales, fallback strategy)
   - Search & filtering? (full-text search, faceted filters, sort options)
   - Content versioning / history? (rollback, diff view, scheduled publish)
   - SEO fields? (slug, meta title, meta description, OG image, sitemap)
```

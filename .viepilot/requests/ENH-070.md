# ENH: vp-audit Auto-Log Gaps → Direct vp-evolve Routing

- **Created**: 2026-04-22
- **Reporter**: User
- **Assignee**: AI

## Summary
When `vp-audit` detects gaps or improvement points (Tier 1–4), it should **automatically log them as requests** (`.viepilot/requests/`) without requiring the user to manually run `/vp-request` afterward. After the audit report is shown, the user can run `/vp-evolve` directly to plan a fix phase — no intermediate request-creation step needed.

## Current Behavior

1. `vp-audit` runs Tier 1–4 checks and outputs a report of issues (⚠️/⛔)
2. User must manually invoke `/vp-request` to log each gap as a request
3. User then invokes `/vp-evolve` to plan a phase for the fix
4. Three separate steps for what is logically one action

## Desired Behavior

1. `vp-audit` runs Tier 1–4 checks
2. **After the report, vp-audit automatically logs each detected gap** as a `.viepilot/requests/BUG-XXX.md` or `ENH-XXX.md` file:
   - Tier 1 (state inconsistency) → `BUG-XXX`
   - Tier 2 (doc drift) → `BUG-XXX` (low/medium severity)
   - Tier 3 (stack violations) → `ENH-XXX` (improvement) or `BUG-XXX` (correctness issue)
   - Tier 4 (framework gaps) → `ENH-XXX`
3. TRACKER.md is updated with all newly-logged requests
4. User sees audit summary banner + request IDs created
5. User can immediately invoke `/vp-evolve {request-ids}` to plan fix phase — **no `/vp-request` step required**

## Detailed Design

### Auto-log behavior in `workflows/audit.md`

After each tier's report section, add an **Auto-Log Gate**:

```
For each ⚠️ or ⛔ item found in Tier N:
  1. Determine request type:
     - State inconsistency / correctness → BUG
     - Doc drift / missing docs → BUG (low)
     - Stack violation / anti-pattern → ENH (or BUG if blocking)
     - Framework gap → ENH
  2. Check if a request already exists with matching title (avoid duplicates):
     - Search .viepilot/requests/*.md for title match
     - If duplicate found: skip (log "already tracked as {ID}")
  3. Create .viepilot/requests/{TYPE}-{N}.md with:
     - Source: "auto-logged by vp-audit Tier {N}"
     - Severity from audit finding
     - Affected files from audit output
     - Summary from audit finding text
  4. Append to TRACKER.md backlog table
```

### Post-audit routing banner

Replace current "Fix now before starting next phase? (y/n)" prompt with:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 VP-AUDIT: {N} issue(s) found — {M} requests logged
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 Logged:
   BUG-XXX: {title}           [Tier 1]
   ENH-XXX: {title}           [Tier 3]
   ...

 Next actions:
   /vp-evolve {IDs}     Plan fix phase for logged requests
   /vp-auto --fix       Apply auto-fixable issues now
   skip                 Continue without fixing
```

**AUQ (Claude Code terminal):**
```
question: "{N} gaps logged. What would you like to do?"
options:
  - "Plan fix phase → /vp-evolve"        (Recommended)
  - "Auto-fix now → /vp-auto --fix"
  - "Skip — review later"
```

### Duplicate detection

Before creating a new request, check `.viepilot/requests/` for existing open requests with:
- Same source file(s) in `## Related → Files`
- Similar title (case-insensitive substring match ≥ 70%)

If match found: append a note to the existing request ("re-detected by vp-audit {date}") rather than creating a duplicate.

### `vp-audit --no-autolog` escape hatch

Add flag `--no-autolog` to disable auto-logging for users who prefer manual request management.

## Discussion
The current friction (audit → manually /vp-request each gap → /vp-evolve) discourages users from acting on audit findings. Auto-logging removes the "log it yourself" burden while still giving users a choice on whether to plan a fix immediately or skip. The duplicate check prevents noise on repeated audits.

## Summary
Comprehensive quality uplift for `/vp-proposal`. Current output (pptx, docx, AI content, workflow) is functional but low-quality. Needs improvement across all layers:

1. **Visual `.pptx` templates** — layouts too sparse, typography generic, visual hierarchy weak
2. **AI-generated content** — bullets shallow, speakerNotes thin, lacks domain context
3. **`.docx` structure** — sections placeholder-heavy, narrative insufficient
4. **Workflow** — insufficient information gathering before generation; misses key context

## Details

### Layer 1 — `.pptx` template quality
**Current:** Dark navy background with basic text boxes. No cover illustration area, no icon support, no visual variety between slides. All slides look nearly identical.

**Target:**
- Distinct layout per slide type: `cover` (full bleed, large title), `section` (accent sidebar), `two-column` (split), `data` (metric callouts with large numbers), `closing` (CTA block)
- Slide Master with defined text styles (H1, H2, body, caption)
- Consistent spacing grid (16px/24px rhythm)
- Accent bar, footer with logo + page number on all non-cover slides
- Better stock templates generated by `scripts/gen-proposal-pptx.cjs`

### Layer 2 — AI content quality
**Current:** Manifest generation prompt is generic; bullets are 3–5 words each; speaker notes are 1 sentence.

**Target:**
- Richer prompt with audience persona, tone, word count targets per bullet
- Bullets: 8–15 words, outcome-oriented
- Speaker notes: 3–5 sentences with talking points and transitions
- Cover slide: compelling subtitle, not just date
- Closing slide: concrete CTA, not "Thank you"
- manifest_generation step in `workflows/proposal.md` gets detailed prompt template

### Layer 3 — `.docx` quality
**Current:** Section headings + short placeholder paragraphs. Tables missing from timeline/budget sections.

**Target:**
- Executive Summary: 2–3 paragraphs from brainstorm session (not boilerplate)
- Problem & Solution: narrative with context from session
- Timeline: actual Gantt-style table (milestone | duration | owner)
- Budget: estimate table with line items
- Team: role + experience blurbs
- Appendix: full brainstorm session notes if available

### Layer 4 — Workflow / information gathering
**Current:** Brief mode asks only 5 generic questions; session mode auto-extracts but misses key fields (budget, timeline, audience seniority, competitor context).

**Target:**
- Add Step 2C: "Quality brief" — even when session loaded, ask 4 focused questions:
  1. What decision should this proposal drive? (CTA)
  2. Approximate budget range (if client-facing)?
  3. Timeline constraints?
  4. Who is the decision-maker (title/seniority)?
- Store answers in manifest `meta` field
- Use in content generation for sharper, more specific output

## Acceptance Criteria

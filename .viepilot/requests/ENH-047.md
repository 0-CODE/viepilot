## Summary
The current brownfield scanner (FEAT-018, Phase 75) handles monorepos at a basic level: Signal Category 1 detects npm workspaces, Nx/Turborepo, Maven multi-module, Cargo workspace, and Go workspace, then aggregates them into a `modules[]` array in the Scan Report. However, it lacks:

1. **Git submodules** — `.gitmodules` pulls external repos into the tree; they are invisible to the current scanner
2. **Multi-repo / polyrepo** — multiple independent git repos forming one system (frontend repo + backend repo + infra repo linked via docker-compose or service mesh); currently no detection or user prompt
3. **Per-module gap detection** — each module in a monorepo needs its own DETECTED / ASSUMED / MISSING tier; currently all modules are aggregated under a single root gap tier
4. **Cross-module dependency map** — which modules depend on which; enables change-impact analysis

---

## Details

### Current Behavior (FEAT-018 baseline)

Signal Category 1 detects these monorepo layouts and scans each module:
```
npm workspaces : package.json → "workspaces" field + packages/*/package.json
Nx / Turborepo : nx.json, turbo.json + apps/*/ and libs/*/
Maven multi-module : pom.xml with <modules>
Gradle multi-project : settings.gradle with `include`
Cargo workspace : Cargo.toml with [workspace]
Go workspace : go.work
```
Result: `modules[]` array in Scan Report (aggregate, no per-module gap tier).

---

### Gap A — Git Submodule Detection

**What to scan:**
- Read `.gitmodules` at repo root (if present)
- Parse each `[submodule "name"]` block → extract `path` + `url`
- For each submodule path that exists locally: run Signal Cat 1–4 scan on that path
- If submodule path is absent (not initialized): record as `initialized: false`

**Scan criteria (done = all checked):**
- [ ] `.gitmodules` file detected at repo root
- [ ] All `[submodule]` blocks parsed: `name`, `path`, `url` extracted
- [ ] For each initialized submodule: Signal Cat 1 (manifest), Cat 2 (framework), Cat 4 (DB signals) run on `{path}/`
- [ ] Uninitialized submodule paths recorded with `initialized: false` and `primary_language: MISSING`
- [ ] Each submodule entry added to `modules[]` with `type: submodule`

**Scan Report schema addition:**
```yaml
modules:
  - name: shared-lib
    type: submodule              # NEW: submodule | workspace | root
    path: libs/shared
    submodule_url: https://github.com/org/shared-lib.git
    initialized: true
    primary_language: TypeScript
    framework: null
    gap_tier: DETECTED
    open_questions: []
```

**Safety rule:** Never `git submodule update --init` during scan — read filesystem only; never trigger network calls.

### Gap B — Multi-Repo / Polyrepo Detection

**What to scan:**

| Signal file | Pattern | Meaning |
|-------------|---------|---------|
| `docker-compose.yml` / `docker-compose*.yml` | `build: ../` or `context: ../` (path starting with `../`) | Sibling repo referenced as build context |
| `docker-compose.yml` | Multiple services with `image:` + no `build:` but README/env mentions external repos | External services |
| `.github/workflows/*.yml` | `uses: org/other-repo/.github/workflows/` or `git clone … other-repo` steps | CI clones a sibling repo |
| `package.json` | `"dependencies"` or `"devDependencies"` with `"file:../sibling"` values | Local file: sibling dep |
| `README.md` / `CONTRIBUTING.md` | Lines matching `https?://github.com/…` or `gitlab.com/…` referencing a different repo (different owner or repo name) | Documentation link to related repo |
| `Makefile` / `justfile` | Targets containing `cd ../` followed by build commands | Cross-repo build tasks |

**Scan criteria (done = all checked):**
- [ ] `docker-compose*.yml` files read; `build:` and `context:` values extracted
- [ ] Relative `../` paths in `build:`/`context:` recorded as `polyrepo_hints`
- [ ] `package.json` `file:../` dependencies extracted
- [ ] CI workflow files scanned for cross-repo `git clone` or `uses:` of sibling repos
- [ ] `README.md`/`CONTRIBUTING.md` scanned for external repo URLs (different repo name from current)
- [ ] All hints deduplicated and stored in `polyrepo_hints[]`
- [ ] If `polyrepo_hints` non-empty → interactive prompt fired: "This repo appears to be part of a multi-repo system. List related repo URLs (optional, one per line):"
- [ ] User-supplied repo URLs stored as `related_repos[]` in Scan Report

**Scan Report schema addition:**
```yaml
polyrepo_hints:                    # NEW
  - source: docker-compose.yml
    hint: "build: ../api-service"
    inferred_repo: api-service
  - source: package.json
    hint: "\"utils\": \"file:../shared-utils\""
    inferred_repo: shared-utils
related_repos:                     # NEW (user-supplied after prompt)
  - url: https://github.com/org/api-service
    role: backend
  - url: https://github.com/org/shared-utils
    role: shared-library
```

**Gap-fill rule:** `polyrepo_hints` non-empty but `related_repos` empty = ASSUMED tier for system-level context fields (e.g. `deployment_topology`). User can skip the prompt — system proceeds with ASSUMED.

### Gap C — Per-Module Gap Detection

**What to scan (per module):**

Each entry in `modules[]` must run its own MUST-DETECT check independently. A module's gap tier is determined by the weakest field in its own scan result.

**Per-module MUST-DETECT fields:**
| Field | Tier if missing |
|-------|----------------|
| `primary_language` | MISSING → must ask before generating artifacts |
| `framework` (≥1) | ASSUMED (infer from deps) or MISSING if no deps found |
| `module_purpose` | ASSUMED (infer from directory name + manifest description) |
| `entry_point` | ASSUMED if `main`/`src/index.*` exists; MISSING otherwise |

**Per-module gap tier rules:**
```
DETECTED  — all MUST-DETECT fields found from file evidence
ASSUMED   — ≥1 MUST-DETECT field inferred (not read directly from files)
MISSING   — ≥1 MUST-DETECT field has no evidence and cannot be inferred
```

**Scan criteria (done = all checked):**
- [ ] Each `modules[]` entry carries: `gap_tier`, `open_questions[]`, `must_detect_status{}`
- [ ] `must_detect_status` records evidence source per field: `{ primary_language: { value: "TypeScript", source: "tsconfig.json", tier: "DETECTED" } }`
- [ ] If any module has `gap_tier: MISSING` → interactive prompt for that module before proceeding to artifact generation
- [ ] Root-level gap tier is the worst tier across all modules (e.g. if any module is MISSING → root is MISSING)
- [ ] Scan Report section header per module lists its gap tier clearly

**Scan Report schema — expanded `modules[]` entry:**
```yaml
modules:
  - name: api-service
    type: workspace
    path: apps/api
    primary_language: TypeScript
    framework: NestJS
    module_purpose: "REST API server"       # ASSUMED from dir name + package description
    entry_point: src/main.ts                # DETECTED
    gap_tier: DETECTED
    must_detect_status:
      primary_language: { value: TypeScript, source: tsconfig.json, tier: DETECTED }
      framework:        { value: NestJS,     source: package.json,  tier: DETECTED }
      module_purpose:   { value: "REST API", source: inferred,      tier: ASSUMED  }
      entry_point:      { value: src/main.ts, source: package.json, tier: DETECTED }
    open_questions: []
  - name: web-client
    type: workspace
    path: apps/web
    primary_language: TypeScript
    framework: React
    module_purpose: "Frontend SPA"
    entry_point: src/index.tsx
    gap_tier: DETECTED
    must_detect_status:
      primary_language: { value: TypeScript, source: tsconfig.json, tier: DETECTED }
      framework:        { value: React,      source: package.json,  tier: DETECTED }
      module_purpose:   { value: "SPA",      source: inferred,      tier: ASSUMED  }
      entry_point:      { value: src/index.tsx, source: vite.config.ts, tier: DETECTED }
    open_questions: []
```

### Gap D — Cross-Module Dependency Map

**What to scan:**

| Ecosystem | Evidence | Dependency type |
|-----------|---------|----------------|
| npm workspaces | `package.json` dep with `"workspace:*"` or `"*"` pointing to sibling name | compile-time |
| Maven multi-module | `<dependency>` in child `pom.xml` with `<groupId>` matching parent `<groupId>` | compile-time |
| Gradle multi-project | `implementation project(':sibling')` in `build.gradle` | compile-time |
| Go workspace | `go.work` `use` directives + `import` paths starting with sibling module path | compile-time |
| Cargo workspace | `[dependencies]` with `path = "../sibling"` | compile-time |
| Any | Dockerfile `COPY --from=sibling-stage` or `FROM sibling-image` | build-time |

**Scan criteria (done = all checked):**
- [ ] npm: `workspace:*` / `*` deps in each module's `package.json` cross-referenced against sibling module names
- [ ] Maven: `<dependency>` blocks in child poms cross-referenced against `<modules>` list
- [ ] Gradle: `project(':name')` references extracted
- [ ] Cargo: `path = "../"` dependencies extracted
- [ ] Each dependency edge recorded as `{ from, to, type, evidence_path }`
- [ ] Result stored in `module_dependencies[]` in Scan Report
- [ ] If dependency cycle detected → `open_questions[]` entry added at root level

**Scan Report schema addition:**
```yaml
module_dependencies:               # NEW (Gap D)
  - from: web-client
    to: shared-ui
    type: compile
    evidence_path: apps/web/package.json
  - from: api-service
    to: shared-types
    type: compile
    evidence_path: apps/api/package.json
dependency_cycles: []              # NEW — list cycle paths if found
```

## Done Scan Criteria — Complete Checklist

A brownfield scan of a multi-module or multi-repo project is **DONE** when:

**Monorepo root:**
- [x] Monorepo tool detected (npm workspaces / Nx / Turborepo / Maven / Gradle / Cargo / Go)
- [x] All workspace/module paths enumerated
- [x] Root-level Signal Cat 1–12 scan complete

**Per module (×N):**
- [ ] Signal Cat 1 run: manifest found → language + version extracted
- [ ] Signal Cat 2 run: framework identified (or ASSUMED/MISSING)
- [ ] Signal Cat 3 run: architecture layer pattern identified (or ASSUMED)
- [ ] Signal Cat 4 run: DB migration signals checked
- [ ] `gap_tier` assigned: DETECTED / ASSUMED / MISSING
- [ ] `must_detect_status` populated with evidence source per field
- [ ] `open_questions[]` populated for any MISSING field

**Git submodules (if `.gitmodules` present):**
- [ ] All submodule paths enumerated from `.gitmodules`
- [ ] Initialized paths: Signal Cat 1–4 run
- [ ] Uninitialized paths: recorded with `initialized: false`
- [ ] All submodule entries in `modules[]` with `type: submodule`

**Polyrepo (if hints detected):**
- [ ] `polyrepo_hints[]` built from docker-compose, package.json, CI, README signals
- [ ] Interactive prompt fired (or skipped if `--no-prompt`)
- [ ] `related_repos[]` populated from user input (or empty if skipped)

**Cross-module dependency map (Gap D — Phase 78):**
- [ ] All intra-workspace dependency edges extracted per ecosystem
- [ ] `module_dependencies[]` populated
- [ ] Cycles detected → `dependency_cycles[]` + `open_questions[]` entry

**Root-level aggregation:**
- [ ] Root `gap_tier` = worst tier across all modules
- [ ] All module `open_questions[]` rolled up to root `open_questions[]`
- [ ] Scan Report YAML complete with all new fields
- [ ] If any MISSING → pause-and-ask before artifact generation

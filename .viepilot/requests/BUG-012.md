# BUG: vp-auto edits production install (~/.claude/) instead of codebase (repo workflows/)

## Meta
- **ID**: BUG-012
- **Type**: Bug
- **Status**: new
- **Priority**: critical
- **Created**: 2026-04-10
- **Reporter**: User
- **Assignee**: AI

## Summary
`/vp-auto` khi thực thi task có `## Paths: workflows/brainstorm.md` (repo-relative) lại tự expand và edit file tại `~/.claude/viepilot/workflows/brainstorm.md` (production install) thay vì tại `{repo}/workflows/brainstorm.md` (codebase source).

**Incident**: Phase 55, Task 55.1 — LLM đọc `~/.claude/viepilot/workflows/brainstorm.md` và edit trực tiếp production file, không edit repo source.

## Details

### Vấn đề
ViePilot framework repo là **source code** tại `{repo}/` (e.g. `/Users/sonicq12/PROJECT_SCRATCH/viepilot/`). Sau khi `dev-install.sh` chạy, các files được copy sang `~/.claude/viepilot/` (production install).

Khi vp-auto thực thi task:
- Task file ghi đúng `## Paths: workflows/brainstorm.md` (repo-relative)
- BUG-009 path guard kiểm tra đúng (không có absolute path)
- **Nhưng** LLM khi tìm và đọc file, nó dùng absolute path `~/.claude/viepilot/workflows/brainstorm.md` (production) thay vì `{cwd}/workflows/brainstorm.md` (codebase)

### Root Cause
BUG-009 guard chỉ validate format của path trong `## Paths` block (repo-relative vs absolute). Nó **không enforce** rằng LLM phải resolve path tương đối từ `{repo_root}` chứ không phải từ `~/.claude/viepilot/`.

Khi LLM đọc context về "brainstorm.md", nó có thể biết đến cả hai copies. Không có rule tường minh nào nói:
> "Tất cả paths trong `## Paths` block phải được resolve từ `{cwd}` (repo root), KHÔNG phải từ `~/.claude/` hay bất kỳ install path nào khác."

### Ảnh hưởng
- LLM edit sai file → production bị thay đổi, codebase không thay đổi
- Sau khi `dev-install.sh` chạy lần tiếp theo → production bị overwrite về codebase (may mắn là self-healing), nhưng thay đổi đúng bị mất
- Nguy hiểm hơn: nếu production được dùng ngay sau khi edit nhầm, hành vi LLM bị ảnh hưởng trước khi ai phát hiện

### Context lịch sử
- BUG-009 (done 1.15.0): guard `## Paths` block cho absolute paths — NHƯNG chỉ guard format, không guard destination
- `dev-install.sh` copy source → `~/.claude/viepilot/` và `~/.claude/skills/vp-*/`
- Có 2 copies của mỗi workflow/skill file: source (codebase) và installed (production)

## Steps to Reproduce
1. Repo ViePilot đã chạy `dev-install.sh` → tồn tại cả `{repo}/workflows/brainstorm.md` và `~/.claude/viepilot/workflows/brainstorm.md`
2. Tạo task với `## Paths: workflows/brainstorm.md`
3. Chạy `/vp-auto` để execute task đó
4. → LLM tìm và edit `~/.claude/viepilot/workflows/brainstorm.md` thay vì `{repo}/workflows/brainstorm.md`

## Expected Behavior
- Tất cả file edits trong vp-auto phải resolve từ `{cwd}` (repo root)
- `{cwd}/workflows/brainstorm.md` được edit
- `~/.claude/viepilot/workflows/brainstorm.md` KHÔNG được đụng tới trong quá trình vp-auto

## Actual Behavior
LLM edit `~/.claude/viepilot/workflows/brainstorm.md` (production) — codebase không thay đổi, production bị pollute.

## Proposed Fix

### Option A — Tường minh hóa trong autonomous.md (preferred)
Thêm rule vào `workflows/autonomous.md` execution context:

```
⛔ PATH RESOLUTION RULE (BUG-012):
When implementing tasks, ALL file reads and edits MUST be resolved from {cwd} (the repository root where package.json lives).
NEVER read or edit files under ~/.claude/, ~/.cursor/, or any other install/home directory.
If a task path resolves to both a codebase location and an install location, ALWAYS use the codebase ({cwd}) location.
```

### Option B — Thêm vào task template
Mọi task `.md` file sinh ra bởi vp-evolve phải có note:
```
> ⚠️ All paths are repo-relative from {cwd}. Do NOT edit ~/.claude/ or any install path.
```

### Option C — Cả hai (recommended)
A + B để defense in depth.

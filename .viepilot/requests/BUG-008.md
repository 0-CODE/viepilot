# 🐛 BUG: vp-auto có cảm giác “loop break” sau mỗi task

## Meta
- **ID**: BUG-008
- **Type**: Bug (investigation / UX — không phải regression loop trong spec)
- **Status**: triaged
- **Severity**: medium
- **Created**: 2026-04-03
- **Reporter**: User
- **Affected**: `workflows/autonomous.md`, runtime Cursor/chat, optional gates

## Summary
Người dùng quan sát: mỗi lần chạy `/vp-auto`, sau khi hoàn thành **một** task thì không thấy tiếp tục task kế — cảm giác vòng lặp bị “break”.

## Actual Behavior
Một invocation `/vp-auto` trong chat thường kết thúc sau ~một task (hoặc tại control point).

## Expected Behavior (mental model)
Kỳ vọng: một lần gọi `/vp-auto` chạy liên tục nhiều task trong phase cho đến khi hết task hoặc gặp blocker.

## Root cause (đã rà soát repo)

### 1. Không phải lỗi spec “chỉ 1 task”
Workflow normative mô tả **vòng lặp theo task** trong phase (`### 3b. Execute Tasks Loop` — “For each task in phase”), và sau `pass` chuyển sang task tiếp hoặc `phase_complete`. **Không** có quy tắc “không arg ⇒ bắt buộc dừng sau mỗi task”.

### 2. Nguyên nhân chính: giới hạn **một lượt (turn) chat / Cursor**
Tài liệu sản phẩm đã ghi nhận rõ: một lượt AI thường đủ cho một task (implement → verify → cập nhật state); task tiếp theo cần **gọi lại** `/vp-auto`, `/vp-resume`, hoặc nhắc rõ tiếp task kế. Đây là hành vi **platform / usage**, không phải bug trong state machine ViePilot.

Tham chiếu:
- `README.md` — bảng `/vp-auto` và đoạn “Thực tế trong Cursor / chat”
- `docs/user/features/autonomous-mode.md` — mục “Tại sao vẫn thấy dừng sau mỗi task?”

### 3. Nguyên nhân phụ (có thể dừng có chủ đích)
- **Control points**: conflict, fail gate, recovery hết budget, Gap G / compliance menu.
- **Git persistence (BUG-003)**: chưa commit/push sạch → không được mark PASS / dừng tại điểm kiểm soát.
- **Token budget** (`workflows/autonomous.md` — Token budget check): `used_pct > 90` → **force pause**, exit session; `used_pct > 70` và không `--fast` → menu (có thể chọn pause).
- **Manual steps** trong task file.
- **Task contract** chưa đủ → `blocked`, không code tiếp.

## Steps to Reproduce (subjective)
1. Chạy `/vp-auto` trên phase có nhiều task chưa xong.
2. Để AI hoàn thành đủ một task (verify + state + git theo workflow).
3. Quan sát: response kết thúc; task tiếp không chạy trong cùng turn trừ khi model chủ động tiếp tục.

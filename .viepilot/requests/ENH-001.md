## Summary
`vp-auto` cần update `README.md` và `.viepilot/ROADMAP.md` khi milestone hoàn thành, thay vì chỉ update `TRACKER.md` và `PHASE-STATE.md`.

## Current Feature
`workflows/autonomous.md` — step `phase_complete` và `iterate`

## Current Behavior
Khi tất cả phases complete, `vp-auto` chỉ:
- Update `.viepilot/TRACKER.md`
- Update `.viepilot/phases/*/PHASE-STATE.md`
- Create git tag `vp-p{N}-complete`
- Bump version nếu cần

**Không** update:
- `README.md` badges (version, skills count, workflows count)
- `README.md` Skills Reference table
- `README.md` Workflows table
- `README.md` Project Structure section
- `README.md` Completion Status section
- `.viepilot/ROADMAP.md` Progress Summary table
- `.viepilot/ROADMAP.md` Phase status lines

## Desired Improvement

### Trong `workflows/autonomous.md` — step `phase_complete`:
Thêm sub-step sau khi phase complete:
```markdown
#### Sync ROADMAP.md
- Update phase status: Not Started → ✅ Complete
- Update Progress Summary row: 0% → 100%
- Update total completed count
```

### Trong `workflows/autonomous.md` — step `iterate` (milestone complete):
Thêm step "Sync ROOT documents":
```markdown
#### Sync README.md (milestone complete only)
Read current README.md và update:
1. Version badge: `version-{old}` → `version-{new}`
2. Skills badge: `skills-{old}` → `skills-{actual}`
3. Workflows badge: `workflows-{old}` → `workflows-{actual}`
4. Project Scale table: Skills, Workflows, CLI Commands counts
5. Skills Reference table: add missing skills
6. Workflows table: add missing workflows
7. Project Structure: update skills/, workflows/ lists
8. Completion Status: update percentage và skill list
```

### Trong `skills/vp-auto/SKILL.md` — `<objective>`:
Thêm vào **Updates** list:
```
- `README.md` (badges, counts, tables — on milestone complete)
- `.viepilot/ROADMAP.md` (phase status, progress summary)
```

## Value Proposition
Tránh manual sync sau mỗi milestone. README.md và ROADMAP.md luôn phản ánh đúng trạng thái thực tế mà không cần `/vp-debug` để phát hiện.

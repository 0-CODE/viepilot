## Summary
Khi áp dụng README auto-sync bằng `cloc`, installer cần hỗ trợ dependency `cloc` theo cách an toàn đa nền tảng để workflow chạy ổn định.

## Details

### Current behavior
- Cơ chế Phase 20 đang lên kế hoạch dùng `cloc` để tính LOC.
- Installer hiện chưa quản lý cài đặt `cloc`.

### Desired improvement
1. Không bundle binary `cloc` trực tiếp vào `bin/` của ViePilot (tránh lỗi cross-platform).
2. Trong installer, detect `cloc` availability.
3. Nếu thiếu, đưa option:
   - hướng dẫn cài theo OS (brew/apt/choco),
   - hoặc auto-install best-effort (nếu user đồng ý, có quyền phù hợp).
4. Workflow vẫn có fallback an toàn nếu `cloc` chưa sẵn sàng.

## Acceptance Criteria
- [ ] Installer có bước check `cloc`.
- [ ] Có UX rõ khi thiếu `cloc` (install guidance/optional auto-install).
- [ ] Không làm fail toàn bộ install/workflow nếu thiếu `cloc`.
- [ ] Docs cập nhật prerequisite cho README metric sync.

## Related
- Phase: `20-readme-sync-and-donate` (Task 20.3+)

## Resolution
Addressed in Phase 20:
- `install.sh` now detects `cloc`, provides OS-specific install guidance, and supports optional best-effort installation.
- `dev-install.sh` now checks `cloc` and provides guidance.
- Workflow remains non-blocking when `cloc` is unavailable.

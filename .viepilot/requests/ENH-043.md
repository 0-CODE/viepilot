## Summary
Mở rộng ENH-042 (PPTX visuals) và ENH-041 (docx Mermaid source text) sang file `.docx`:
1. **HTML artifact screenshots** (ui-direction/architect pages) nhúng vào docx dưới dạng `ImageRun`
2. **Mermaid charts** render thành PNG rồi nhúng inline thay vì chỉ hiện preformatted source text

## Details

### Current Behavior
- `.docx` Diagram Reference section: hiển thị `mermaidSource` dưới dạng preformatted text (monospace paragraph)
- `.docx` không có ảnh chụp từ ui-direction hay architect HTML
- Người xem phải đọc Mermaid source code — không trực quan

### Desired Improvement
1. **Mermaid → PNG**: render `mermaidSource` thành ảnh PNG, nhúng vào docx section tương ứng
2. **HTML artifact → PNG**: dùng lại `screenshotArtifact()` (từ ENH-042) để chụp ui-direction/architect HTML, nhúng vào docx section tương ứng (e.g. trước/sau Executive Summary, Solution, Technical Approach)
3. **Fallback** giống PPTX: nếu không render được → giữ nguyên preformatted text (không crash)

### Proposed Technical Approach

#### Mermaid rendering:
| Option | Dep | Quality |
|--------|-----|---------|
| `@mermaid-js/mermaid-cli` (`mmdc`) CLI | npm optional/global | Best — official |
| `mermaid` + jsdom/puppeteer | npm optional | Acceptable |
| **Keep preformatted text** | none | Fallback |

Recommended: try `mmdc` via `child_process.execSync('mmdc --version')` → if available, render `mmdc -i input.mmd -o output.png`; else fallback to text.

#### docx image embedding:
```js
// docx package supports ImageRun:
new ImageRun({
  data: fs.readFileSync(pngPath),           // Buffer
  transformation: { width: 500, height: 300 },  // EMU or px — use px
  type: 'png',
});
```

#### New sections in docx:
- `## Executive Summary` → prepend ui-direction `index.html` screenshot (if available)
- `## Technical Approach` / Diagram Reference → replace preformatted Mermaid with rendered PNG
- `## Glossary` / appendix → insert architect pages (architecture.html, erd.html, etc.)

#### Helpers needed:
- `renderMermaidToPng(mermaidSource, outputPath)` → calls `mmdc`; returns `string|null`
- `isMmdcAvailable()` → boolean guard
- `embedImageInDocx(pngPath)` → returns `ImageRun` or `null`
- Update `scripts/gen-proposal-docx.cjs` + `lib/screenshot-artifact.cjs` (add `isMmdcAvailable`, `renderMermaidToPng`)

### Breaking Changes
No — fallback ensures .docx still generates without `mmdc` or `puppeteer`

# 🔧 ENHANCEMENT: vp-docs — Keep skills-reference.md in sync with skills/ directory

## Meta
- **ID**: ENH-005
- **Type**: Enhancement
- **Status**: new
- **Priority**: high
- **Created**: 2026-03-30
- **Breaking**: No
- **Source**: Debug session debug-20260330-153747

## Summary
`vp-docs` (và bất kỳ workflow nào thêm skill mới) cần đảm bảo `docs/skills-reference.md` luôn có section cho **mỗi** skill trong `skills/` directory. Hiện tại file này chỉ được generate một lần và không tự cập nhật khi skill mới được thêm vào.

## Current Feature
`skills/vp-docs/SKILL.md` — Documentation generation skill

## Current Behavior
Khi `/vp-docs` chạy lần đầu, nó generate `docs/skills-reference.md` với các skills tồn tại tại thời điểm đó (10 skills). Khi 3 skills mới được thêm (`vp-debug`, `vp-rollback`, `vp-audit`) trong M1, `vp-docs` **không được chạy lại** và `skills-reference.md` không được cập nhật.

## Root Cause
Hai vấn đề riêng biệt:
1. **Incremental update**: `vp-docs` không có mode "update existing docs" — chỉ generate từ đầu
2. **New skill workflow**: Khi thêm skill mới (qua `vp-auto` hoặc manual), không có step nào update `docs/skills-reference.md`

## Desired Improvement

### Fix 1: Trong `workflows/documentation.md` — step generate skills-reference:
Thay vì generate static list, **scan `skills/` directory** để build danh sách:

```markdown
### Generate skills-reference.md

1. Scan `skills/` directory để lấy danh sách skills thực tế:
   ```bash
   SKILLS=$(ls skills/*/SKILL.md | sed 's|skills/||; s|/SKILL.md||' | sort)
   ```

2. Với mỗi skill, đọc `SKILL.md` để extract:
   - `<objective>` → Purpose
   - Flags từ `<context>` section
   - Output files từ `<objective>` Creates/Updates

3. Generate section cho mỗi skill theo template chuẩn

4. **Nếu `docs/skills-reference.md` đã tồn tại**: chỉ thêm sections cho skills mới,
   không overwrite sections đã có (preserve manual edits)
```

### Fix 2: Trong `workflows/autonomous.md` — khi phase complete và có skill mới:
Thêm check sau mỗi phase:
```markdown
#### Post-phase: Update skills-reference.md
Nếu phase vừa tạo skill mới (detect bằng `git diff --name-only | grep 'skills/.*/SKILL.md'`):
- Append section mới vào `docs/skills-reference.md`
- Commit: `docs: add {skill-name} to skills-reference.md`
```

### Fix 3: Trong `skills/vp-docs/SKILL.md` — `<objective>`:
Thêm vào **Updates** (không chỉ Creates):
```
- `docs/skills-reference.md` (incremental update — add missing skills)
```

## Value Proposition
`docs/skills-reference.md` luôn là source of truth cho tất cả skills, không bao giờ bị stale sau khi thêm skill mới.

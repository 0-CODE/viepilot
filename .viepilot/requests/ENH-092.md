## Summary
Upgrade the existing `research-agent` (agents/claude-code/research-agent.md) to use `vercel-labs/agent-browser` for deep research on JavaScript-rendered pages — competitor apps, Storybook/Docusaurus docs, product landing pages, pricing tables — beyond what WebFetch (static HTML only) can handle. Also surfaces this capability in `vp-brainstorm` when user provides reference URLs.

## Problem Statement
Current `research-agent` uses WebSearch + WebFetch. This fails for:
- **JS-rendered sites**: React/Next.js/Vue apps render empty HTML server-side, content injected by JS
- **Competitor apps**: pricing, feature comparison tables, interactive demos
- **Modern documentation**: Docusaurus, Mintlify, ReadTheDocs v2 — all client-side rendered
- **Product reference pages**: When user says "look at how Notion does X" → current agent gets mostly blank HTML

In `vp-brainstorm`, this means competitor research is shallow — user must paste text manually.

## Proposed Solution

### Upgrade: `research-agent` (agents/claude-code/research-agent.md)
Add optional agent-browser tool invocation when URL is provided:

```yaml
# Extended tool list (agent-browser available when skill installed)
# Pattern: if input has URL → use agent-browser; else → WebSearch + WebFetch
```

**New behavior flow:**
1. Input contains URL → use `agent-browser open <url>` + `snapshot -i` to extract structured content
2. Input is a search query → existing WebSearch + WebFetch path (unchanged)
3. Input is URL but agent-browser not available → WebFetch fallback + warning

### Integration in `vp-brainstorm` / `workflows/brainstorm.md`
Add research block in Step 3 (Explore phase):
- When user provides reference URL: `Agent({ subagent_type: "research-agent", url: "...", task: "extract features and UX patterns" })`
- Output: structured summary (feature list, UX patterns, pricing model, tech stack clues)
- Surfaced in brainstorm session as "Reference research" section

### New operations for research-agent
- `browse_url` — navigate URL, snapshot, extract structured text + key elements
- `compare_products` — browse multiple URLs, produce comparison table
- `extract_pricing` — navigate to pricing page, extract tiers/features/prices as structured data

## Summary
Bổ sung trang `user-use-cases.html` vào Architect Design Mode workspace để brainstorm và visualize User Stories / Use Cases trực quan bằng Mermaid.

## Details

### Current behavior
Architect Mode (FEAT-011 + ENH-027) không có trang dành riêng cho User Use Cases — user phải viết text trong tech-notes hoặc architecture mà không có diagram riêng.

### Desired improvement
1. **`templates/architect/user-use-cases.html`** — trang mới:
   - Mermaid `flowchart TD` hoặc custom ASCII-style use-case diagram (actors → use cases)
   - Table: **User Stories** (ID | As a... | I want to... | So that... | Priority | Status)
   - Table: **Use Cases** (Use Case | Actor | Preconditions | Main Flow | Alt Flow | Notes)
   - Section: **Actor summary** (Actor | Role | Goals)

2. **All existing pages + erd.html** — sidebar nav: thêm `user-use-cases.html`

3. **`templates/architect/index.html`** — thêm hub card

4. **`workflows/brainstorm.md`** — Architect Design Mode section:
   - Thêm `user-use-cases.html` vào workspace layout
   - Trigger keywords: `user story`, `use case`, `actor`, `persona`, `as a user`, `user flow`, `workflow`, `journey`, `role`, `permission`
   - Diagram: `flowchart TD` với actors + use case bubbles

5. **`workflows/crystallize.md`** — Step 1D:
   - Nếu `user-use-cases.html` hoặc `notes.md ## use_cases` tồn tại → append `## User Stories & Use Cases (from Architect Mode)` vào `.viepilot/PROJECT-CONTEXT.md`

6. **`notes.md` YAML schema** — thêm `## use_cases`:
   ```yaml
   ## use_cases
   actors:
     - name: Guest
       role: Unauthenticated visitor
     - name: User
       role: Registered member
   user_stories:
     - id: US001
       as_a: User
       i_want: to register an account
       so_that: I can access premium features
       priority: must-have
       status: open
   ```

7. **`skills/vp-brainstorm/SKILL.md`** — mention ENH-028

8. **`tests/`** — contract tests

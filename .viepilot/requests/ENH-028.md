## Summary
Split crystallize workflow into two phases: Extraction Phase (reads only, Plan Mode) and Generation Phase (writes). Add a Review Gate step between them so users can review extraction results (entity manifest, diagram list, phase skeleton) before committing to generation.

## Problem
Currently crystallize reads brainstorm → immediately generates phases, ROADMAP, SPEC, tasks in one pass. If the extraction step misidentifies entities or picks wrong diagrams, the user only discovers errors after files are written and must rollback.

## Proposed Solution

### Approach: Two-Phase crystallize with Review Gate

**Phase 1 — Extraction (Plan Mode reads)**
- Steps 1–6A run in Plan Mode (reads only, no file writes)
- AI extracts: entity manifest, architecture diagram list, phase skeleton
- Review Gate: present extraction results to user for approval

**Review Gate step** (`<step name="review_gate">` between Step 1C and Step 2):
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 CRYSTALLIZE ► REVIEW GATE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Extraction complete. Please review before generation:

[1] Entity Manifest (7 entities)
    └─ Tenant (core, needs_crud_api=yes, MISSING service phase)
    └─ User (core, needs_crud_api=yes, Phase 3 ✓)
    └─ Vehicle (core, needs_crud_api=yes, MISSING service phase)
    └─ ...

[2] Architecture Diagrams (4 selected)
    └─ system_context, container, entity_relationship, sequence_auth

[3] Phase Skeleton (8 phases planned)
    └─ Phase 1: Foundation, Phase 2: Auth, ...

Approve section by section or all at once:
  a) Approve all → proceed to generation
  b) Modify entity manifest → re-run entity step
  c) Modify diagram list → re-run diagram selection
  d) Modify phase skeleton → re-run phase planning
  e) Abort
```

**Phase 2 — Generation (normal writes)**
- Steps 7–12 run after approval
- Proceeds with confirmed extraction data

### Flags
- `--no-review`: skip review gate, run full crystallize in one pass
- `--extract-only`: stop after extraction phase (for inspection)

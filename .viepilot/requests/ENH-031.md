## Summary
Standardize all ViePilot framework content (skills, workflows, templates) to **English-primary** (Option B from language audit). Vietnamese is only permitted for: invocation triggers, CLI command aliases, and locale-specific identifiers.

## Context
A `/vp-audit` language mixing analysis identified 3 HIGH-severity and 17 MEDIUM-severity files with inconsistent Vietnamese/English mixing throughout skills, workflows, and templates. The framework's process/instruction content must be standardized so it is readable and maintainable regardless of the user's language.

## Details

### Scope of Changes

**HIGH severity — full refactor required:**
- `workflows/crystallize.md` — Vietnamese narrative mixed inline with English code (e.g., `"parse ## Phases từ brainstorm"`)
- `workflows/autonomous.md` — section headers Vietnamese, content English, scattered throughout
- `workflows/audit.md` — 50/50 split, no consistent language per section

**MEDIUM-HIGH severity — objective/context sections need cleanup:**
- `skills/vp-audit/SKILL.md` — `<objective>` block mixes Vietnamese bullets with English nested content
- `skills/vp-crystallize/SKILL.md` — `<objective>` block mixes language in same bullet items
- `skills/vp-brainstorm/SKILL.md` — Objective headers Vietnamese, descriptions English

**MEDIUM severity — metadata standardization:**
The following files have Vietnamese metadata (descriptions, comments) with English process content. The metadata must be converted to English:
- `skills/vp-request/SKILL.md`
- `skills/vp-auto/SKILL.md`
- `skills/vp-docs/SKILL.md`
- `skills/vp-pause/SKILL.md`
- `skills/vp-resume/SKILL.md`
- `skills/vp-rollback/SKILL.md`
- `skills/vp-ui-components/SKILL.md`
- `skills/vp-evolve/SKILL.md`
- `skills/vp-update/SKILL.md`
- `templates/project/AI-GUIDE.md`

### Language Policy (to be applied)

```
ALLOWED in English-primary files:
  - All prose, descriptions, instructions, comments
  - Section headers, labels, bullet items
  - Code block explanations

ALLOWED to remain in Vietnamese:
  - Skill invocation triggers: "Skill được gọi khi user mention..."
    (the trigger strings themselves: "kiểm tra", "audit", etc.)
  - cursor_skill_adapter invocation keywords (they must match user speech)
  - Inline user-facing aliases in CLI output (e.g., "Next: /vp-evolve")

NOT ALLOWED:
  - Mixed-language sentences ("parse ## Phases từ brainstorm sessions")
  - Vietnamese prose in the same section/block as English prose
  - Vietnamese section headers with English body content
```

### Templates (project/ and phase/)
- `templates/project/AI-GUIDE.md` — translate Vietnamese headers and labels to English
- `templates/project/README.md` — bilingual header format "Chỉ số / Metric" is intentional, keep as-is
- All `templates/phase/*.md` — already English-only, no changes needed

## Discussion
This is a refactor-only change — no behavior changes, no new features. The framework logic stays identical; only the prose language of instructions/descriptions/comments changes from Vietnamese → English.

The one exception: `cursor_skill_adapter` trigger keywords ("Skill được gọi khi user mention...", invocation keywords like `"kiểm tra"`) should stay Vietnamese because they match user speech patterns and are used for invocation detection.

## Summary
Sau khi **clear context** và chạy **`/vp-auto` từ đầu**, một lượt đọc tài liệu hướng dẫn đã tiêu **>13K tokens** (ước lượng từ runtime). Cần làm rõ nguồn gốc và giảm chi phí context mà không phá vỡ doc-first / recovery contract.

## Investigation (2026-04-03 — điều tra trong repo viepilot)

### Các khối context bắt buộc theo workflow hiện tại

1. **`skills/vp-auto/SKILL.md`** (~200+ dòng) — objective, process tóm tắt, trùng một phần với `autonomous.md`.
2. **`workflows/autonomous.md`** — **~886 dòng, ~37KB** — execution_context của skill; thường được load **toàn bộ** vào phiên.
3. **Initialize batch** (`autonomous.md` Step 1): `TRACKER.md` + `ROADMAP.md` + `AI-GUIDE.md`.
4. **Phase load** (Step 3a): `SPEC.md` + `PHASE-STATE.md` của phase hiện tại.
5. **Task batch** (Step 3b): lại `TRACKER`, `HANDOFF`, `PHASE-STATE`, **task file**, và **static**:
   - `.viepilot/AI-GUIDE.md` (~4.3KB)
   - `.viepilot/SYSTEM-RULES.md` (~10.3KB)

### Ước lượng kích thước (repo framework viepilot — `wc -c` tại thời điểm điều tra)

| Artifact | Bytes (gần đúng) |
|----------|------------------|
| `workflows/autonomous.md` | ~37,400 |
| `.viepilot/ROADMAP.md` | ~17,500 |
| `.viepilot/SYSTEM-RULES.md` | ~10,350 |
| `.viepilot/AI-GUIDE.md` | ~4,300 |

Chỉ bốn file trên đã ~**69KB** raw text (~**17–23k tokens** theo heuristic 3–4 byte/token — biến thiên theo tokenizer). **>13K tokens** cho một cold start là **nhất quán** với việc model đọc **full workflow + ROADMAP dài + SYSTEM-RULES + skill**.

### Nguyên nhân gốc (đã xác định)

1. **`autonomous.md` quá lớn** — một file duy nhất chứa state machine, guard, recovery, token budget, architecture ENH-018, git persistence, v.v.
2. **`ROADMAP.md` theo milestone** có nhiều phase/task bảng → rất dài; Initialize yêu cầu parse toàn file.
3. **Trùng lặp skill ↔ workflow** — `vp-auto` SKILL mô tả lại luồng đã có trong `autonomous.md` → rủi ro **đếm đôi** hướng dẫn trong cùng phiên.
4. **SYSTEM-RULES + AI-GUIDE** được batch-read **mỗi task** (workflow ghi “cached” nhưng **session mới = không có cache thật** trong model).
5. **Không có chế độ “cold start tối thiểu”** (vd. chỉ HANDOFF + TASK + link tới section workflow).

## Proposed directions (cho phase / design sau)

- **A — Workflow modularization**: tách `autonomous.md` thành include/logical files (state-machine, execute-task, recovery-only) và chỉ load **slice** theo bước (hoặc `@path#line` trong hướng dẫn agent).
- **B — ROADMAP index**: sinh `.viepilot/ROADMAP-INDEX.md` ≤N dòng cho Initialize; full ROADMAP chỉ khi `vp-evolve` / chỉnh phase.
- **C — Skill slim**: rút `vp-auto` SKILL xuống **routing only** (1 đoạn), bỏ trùng process dài — một nguồn sự thật là `autonomous.md`.
- **D — Flag `--lite` / tier resume**: bỏ qua SYSTEM-RULES đầy đủ cho task **S** đến khi vào bước implement hoặc sau khi contract pass (cần risk assessment).
- **E — Document “expected token budget”** trong `autonomous-mode.md` để user biết cold start tốn khoảng bao nhiêu.

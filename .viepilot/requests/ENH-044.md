## Summary
Hiện tại, Steps 4c và 7 trong `workflows/proposal.md` coi visual embedding là **optional** — nếu `puppeteer` / `mmdc` vắng mặt thì bỏ qua hoàn toàn không báo gì. Điều này khiến người dùng không biết ảnh bị bỏ qua dù artifacts đã có sẵn. Yêu cầu: **khi `detectVisualArtifacts()` trả về kết quả không rỗng (ui-direction hoặc architect pages tồn tại), visual embedding phải là BẮT BUỘC** — AI phải cố gắng chụp ảnh, và nếu tool thiếu thì phải cảnh báo rõ ràng + hướng dẫn cài.

## Details

### Current Behavior
- Step 4c (PPTX): `if no artifacts → skip silently` (log 1 dòng rồi tiếp tục)
- Step 7 (docx): `if mmdc/puppeteer absent → keep preformatted text / no image` (silent fallback)
- Người dùng không biết visuals bị bỏ qua

### Desired Improvement

**Logic mới** khi `detectVisualArtifacts()` trả về `uiPages.length > 0 || architectPages.length > 0`:

1. **PPTX Step 4c** — `visualSlides[]` PHẢI được populate (không để rỗng)
   - Nếu puppeteer available → screenshot → `addImage()`
   - Nếu puppeteer absent → `addPlaceholderVisual()` (styled box) + **log WARNING**:
     ```
     [visuals] ⚠ ui-direction/architect artifacts found but puppeteer not installed.
     Install puppeteer to get real screenshots: npm install puppeteer
     Using placeholder shapes instead.
     ```
   - `visualSlides[]` KHÔNG được rỗng khi artifacts exist

2. **docx Step 7** — Mermaid + UI/architect sections PHẢI được thêm vào document
   - Nếu mmdc absent → Mermaid preformatted text (hiện tại) + **log WARNING**:
     ```
     [visuals] ⚠ Mermaid diagrams found but mmdc not installed.
     Install mmdc to render PNG: npm install -g @mermaid-js/mermaid-cli
     Using source code text instead.
     ```
   - Nếu puppeteer absent → HTML artifact sections bị skip + **log WARNING**
   - Nếu cả hai absent → double WARNING, không crash

3. **`detectVisualArtifacts()` call là REQUIRED step** (không có `if` wrapper):
   - Hiện tại: `const artifacts = detectVisualArtifacts(); if (!artifacts...) skip`
   - Mới: `const artifacts = detectVisualArtifacts(); /* ALWAYS process if non-empty */`

### Affected Files
- `workflows/proposal.md` — Step 4c, Step 7: thay "optional/when available" → "mandatory when artifacts exist"
- `lib/screenshot-artifact.cjs` — add `warnMissingTool(toolName, installCmd)` helper
- `scripts/gen-proposal-pptx.cjs` — `addPlaceholderVisual()` must be called (not just defined) when artifacts exist but puppeteer absent

### Breaking Changes
No — fallback paths still work; only behavior changes from "silent skip" to "placeholder + warning"

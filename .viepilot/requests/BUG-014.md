## Summary
`workflows/rollback.md` Step 7 "Update State Files" parses the rollback target tag to
determine which phase/task to restore HANDOFF.json and TRACKER.md to. It only handles
two legacy patterns but is missing the enriched format introduced by ENH-050.

## Details

### Current patterns handled (Step 7)
```
- `vp-p{N}-t{M}`          → Phase N, Task M
- `{project}-vp-p{N}-t{M}` → Phase N, Task M
- `vp-p{N}-complete`       → Phase N+1
- `{project}-vp-p{N}-complete` → Phase N+1
```

### Missing: ENH-050 enriched format
```
- `{project}-{branch}-{version}-vp-p{N}-t{M}`        → Phase N, Task M
- `{project}-{branch}-{version}-vp-p{N}-t{M}-done`   → Phase N, Task M+1
- `{project}-{branch}-{version}-vp-p{N}-complete`     → Phase N+1
```

Since Phase 57 onward, `vp-auto` creates enriched-format tags exclusively.
A rollback to any tag from Phase 57+ will fail to extract phase/task and leave
HANDOFF.json in an incorrect state.

### File
- `workflows/rollback.md` — Step 7 (`update_state`), parse logic block

## Summary
The post-install "Next actions" printed by `bin/viepilot.cjs` after a successful install only lists Cursor and Claude Code. Antigravity is missing. The block should be adapter-driven so it stays correct automatically when new adapters are added.

## Details

### Current behavior (`bin/viepilot.cjs` lines 340-343)
```
Next actions:
- Cursor: open project and run /vp-status
- Claude Code: restart session if needed so ~/.claude/skills/vp-* is picked up; then /vp-status
- If needed, run /vp-brainstorm then /vp-crystallize
```

### Problem
- Hardcoded — only mentions Cursor and Claude Code
- Antigravity (added Phase 59) has no post-install hint
- Adding a new adapter in future will silently miss this message again

### Desired behavior
Generate per-adapter hints dynamically from the installed targets list. Each adapter object can carry an optional `postInstallHint` string, or the message is assembled from the adapter's `name` and `executionContextBase`.

**Example output when `--target all`:**
```
Next actions:
- Claude Code: restart session so ~/.claude/skills/vp-* is picked up; then /vp-status
- Cursor: open project and run /vp-status
- Antigravity: open project and run /vp-status
- If needed, run /vp-brainstorm then /vp-crystallize
```

**Example output when `--target antigravity`:**
```
Next actions:
- Antigravity: open project and run /vp-status
- If needed, run /vp-brainstorm then /vp-crystallize
```

### Implementation approach
Option A (simple): Add `postInstallHint` string field to each adapter object in `lib/adapters/*.cjs`; `bin/viepilot.cjs` iterates installed targets and prints their hint.

Option B (minimal): Inline per-adapter hint in `bin/viepilot.cjs` using `adapter.name` and `adapter.executionContextBase`, no adapter-file changes needed.

Prefer **Option A** — keeps adapter metadata self-contained.

## Acceptance Criteria

- [ ] Claude Code hint retains the "restart session" note

## Summary

Claude Code's Agent tool supports `run_in_background: true` — spawns agent fire-and-forget, returns immediately, notifies via `<task-notification>` XML when complete. This directly enables ViePilot's Fork State Updates epic (Tier 2): after task PASS, spawn background agent to update TRACKER/CHANGELOG/ROADMAP without blocking the execution loop.

## Source

Reference: `for-learn-claude-cli-main/reports/REPORT-multi-agent-coordinator.md` (Async Agent section), `REPORT-lessons-learned.md` (Bài học 6).

## Mechanism

```typescript
// Async agent — returns agentId immediately, runs in background
AgentTool({ prompt: "...", run_in_background: true })
// → agentId returned
// Agent completes → <task-notification> XML delivered to coordinator

// Notification format:
<task-notification>
  <task-id>agent-xyz</task-id>
  <status>completed</status>
  <summary>TRACKER.md updated to task 8.1, HANDOFF.json written</summary>
</task-notification>
```

## Proposed Fix

### autonomous.md — State Update fork pattern

After task PASS, instead of blocking for state writes:

```
#### Fork State Update (optional — when agent tool available)
Spawn background agent: "Update state files for task {task}:
  1. TRACKER.md current task → {next_task}
  2. PHASE-STATE.md task row → done, Completed={today}
  3. HANDOFF.json position.task → {next_task}, meta.last_written → now
  4. ROADMAP.md if phase progress changed
Fallback: if fork fails or not confirmed within 10s → do synchronous write"

# Main loop continues to next task without waiting
```

### Fallback guard

Background agent must confirm within reasonable window. If `<task-notification>` not received → synchronous write fallback (never allow state to drift silently).

## Summary
Create a lightweight **agents layer** for ViePilot that handles repetitive, parallelizable, or background tasks currently embedded in skill workflows. Agents are specialized sub-routines invoked by vp-* skills to improve throughput, reduce context bloat, and enable concurrent execution of heavy operations (research, file scanning, contract testing, doc generation).

## Problem Statement
Current skills handle everything inline — research web fetches, file scans, tracker updates, test generation, doc writes all happen sequentially in the main LLM context. This causes:
1. **Context bloat** — long skill runs exhaust context with intermediate data
2. **Sequential bottleneck** — research + write + verify runs one-at-a-time
3. **Skill coupling** — each skill re-implements common patterns (TRACKER update, CHANGELOG write, version bump)
4. **No parallelism** — vp-auto tasks that could run concurrently don't

## Proposed Agents

### Agent 1: `tracker-agent`
**Purpose**: Read/write `.viepilot/TRACKER.md` — request triaging, status updates, phase transitions
**Triggers**: vp-request (Step 5), vp-evolve (phase creation), vp-auto (task status)
**Replaces**: inline TRACKER update blocks in request.md, autonomous.md

### Agent 2: `research-agent`
**Purpose**: Web search + fetch + summarize for feasibility studies, tech research, dependency lookup
**Triggers**: vp-request (Copilot-style research gate), vp-brainstorm (external validation)
**Replaces**: ad-hoc WebSearch/WebFetch in skill prompts — now a dedicated context-isolated agent

### Agent 3: `file-scanner-agent`
**Purpose**: Glob + Grep across repo — find affected files, check adapter compatibility, detect stale refs
**Triggers**: vp-audit (Tier 1–4), vp-evolve (impact analysis), vp-rollback (state restore)
**Replaces**: repeated Glob/Grep patterns embedded in workflow steps

### Agent 4: `changelog-agent`
**Purpose**: Read CHANGELOG.md, append new entries, bump version in package.json + VIEPILOT-META.md atomically
**Triggers**: vp-auto (post-phase), vp-evolve (milestone ship)
**Replaces**: version bump instructions scattered across autonomous.md + evolve.md (see ENH-053)

### Agent 5: `test-generator-agent`
**Purpose**: Generate contract tests for new adapter/feature, run test suite, report pass/fail
**Triggers**: vp-auto (task 82.2 pattern — contract tests always last task per phase)
**Replaces**: manual test scaffolding in each phase

### Agent 6: `doc-sync-agent`
**Purpose**: Scan all 17 SKILL.md files, apply uniform table updates (adapter rows, version banners)
**Triggers**: adapter installs (FEAT-019), banner updates (ENH-056 pattern)
**Replaces**: 17× repeated file edit operations that always follow the same pattern

## Architecture

```
vp-* skill (orchestrator)
    │
    ├── tracker-agent      ← .viepilot/TRACKER.md read/write
    ├── research-agent     ← WebSearch + WebFetch (isolated context)
    ├── file-scanner-agent ← Glob + Grep (repo-wide)
    ├── changelog-agent    ← CHANGELOG + version bump
    ├── test-generator-agent ← tests/adapters/*.test.cjs
    └── doc-sync-agent     ← skills/*/SKILL.md bulk update
```

**Implementation approach:**
- Agents defined as `~/.claude/viepilot/agents/{name}.md` — Markdown files with `<agent>` prompt blocks
- Skills call agents via explicit invocation section in workflow `.md` files
- Claude Code: agents map to `Agent` tool calls (subagent_type: general-purpose / Explore)
- Cursor/Codex: agents run as scoped sub-prompts within the same session
- No new runtime required — uses existing adapter tool capabilities

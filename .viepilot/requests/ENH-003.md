## Summary
`vp-audit` cần detect khi `README.md` và `.viepilot/ROADMAP.md` bị "drift" so với thực tế (skills count, version, phase status), và report cụ thể những gì cần update.

## Current Feature
`skills/vp-audit/SKILL.md` và `workflows/audit.md`

## Current Behavior
`/vp-audit` hiện kiểm tra:
- Documentation vs implementation alignment
- Skill/workflow file structure validity
- TRACKER.md vs PHASE-STATE.md consistency

**Không** kiểm tra:
- README.md badges vs thực tế (version, skills count, workflows count)
- README.md Skills Reference table vs `skills/` directory
- README.md Workflows table vs `workflows/` directory
- ROADMAP.md phase status vs PHASE-STATE.md
- ROADMAP.md Progress Summary vs actual completion

## Desired Improvement

### Trong `workflows/audit.md` — thêm step "ROOT Document Drift Check":
```markdown
### ROOT Document Drift Check

#### 1. Count actual resources
```bash
ACTUAL_SKILLS=$(ls skills/*/SKILL.md 2>/dev/null | wc -l)
ACTUAL_WORKFLOWS=$(ls workflows/*.md 2>/dev/null | wc -l)
ACTUAL_VERSION=$(node bin/vp-tools.cjs version --raw 2>/dev/null || grep '"version"' package.json | head -1 | cut -d'"' -f4)
```

#### 2. Read README.md badges
Extract version, skills count, workflows count from badge URLs.

#### 3. Compare and report
If mismatch found:
```
⚠️ README.md DRIFT DETECTED:
  - version badge: 0.1.0 → should be 0.2.0
  - skills badge: 10 → should be 13
  - workflows badge: 8 → should be 11
```

#### 4. Check ROADMAP.md
For each phase in ROADMAP.md:
- Read corresponding PHASE-STATE.md
- Compare status (Not Started vs complete)
- Report mismatches

#### 5. Auto-fix option
```
Drift detected. Options:
1. Auto-fix all (update README.md + ROADMAP.md)
2. Show diff only
3. Skip
```
```

### Trong `skills/vp-audit/SKILL.md` — `<objective>`:
Thêm vào **Checks**:
```
- README.md badge drift (version, counts)
- README.md tables vs actual files
- ROADMAP.md phase status vs PHASE-STATE.md
```

## Value Proposition
`/vp-audit` trở thành safety net tự động phát hiện ROOT document drift, thay vì user phải manually nhận ra sau khi debug. Có thể chạy `/vp-audit` sau mỗi milestone như một health check.

## Summary
Extend Brainstorm Artifact Manifest schema (Phase 09) với 2 artifact types mới: `domain_entities` và `tech_stack` — cả hai `required: true`.

## Details

### Context
Resolved từ brainstorm session 2026-04-03 Part 4 (Topic 12). Phase 09 đang implement Artifact Manifest schema cơ bản (`ui_direction`, `product_horizon`, `research_notes`, `architecture_inputs`). ENH-030 extend schema với 2 types critical còn thiếu.

### `domain_entities` (required: true)

**Rationale**: Kết nối ENH-022 (entity extraction) + ENH-028 (review gate). AI không thể tự suy luận đúng entity list mà không có explicit data — proven drop point trong crystallize fidelity gap analysis.

**Schema**:
```json
{
  "type": "domain_entities",
  "session": "session-YYYY-MM-DD",
  "entities": [
    {
      "name": "Vehicle",
      "type": "core",
      "needs_crud_api": true,
      "service_phase": "OK | MISSING | N/A"
    }
  ],
  "required": true,
  "consumed": false,
  "consumed_at": null,
  "last_updated": "ISO8601"
}
```

**Integration**:
- brainstorm `/save` → auto-extract entities from `**Decisions**` + domain discussion → populate artifact
- crystallize Step 0A → consume `domain_entities` trước entity extraction step (ENH-022)
- Review Gate (ENH-028) → "Entity Manifest" section reads from this artifact

### `tech_stack` (required: true)

**Rationale**: Stack decisions ảnh hưởng trực tiếp Diagram Profile System (Topic 6). Nếu crystallize detect sai stack → diagram profile sai → wrong diagrams generated. Per-layer explicit mapping thay vì inference.

**Schema**:
```json
{
  "type": "tech_stack",
  "session": "session-YYYY-MM-DD",
  "stacks": {
    "backend": ["java", "spring-boot"],
    "database": ["postgresql", "postgis", "redis"],
    "frontend": ["react", "typescript"],
    "mobile": ["react-native"],
    "protocol": ["mqtt", "grpc"],
    "infra": ["docker", "kubernetes"]
  },
  "diagram_profile": "microservice-event-driven-spa",
  "required": true,
  "consumed": false,
  "consumed_at": null,
  "last_updated": "ISO8601"
}
```

**Integration**:
- brainstorm `/save` → auto-extract stack mentions → populate stacks map per layer
- crystallize Step 0A → consume `tech_stack` → use for Diagram Profile System stack detection (replaces inference)
- `diagram_profile` field: crystallize writes after profile selection → manifest records which profile was chosen

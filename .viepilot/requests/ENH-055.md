# ENH: AskUserQuestion enforcement — required on Claude Code, not optional

## Meta
- **ID**: ENH-055
- **Type**: Enhancement
- **Status**: new
- **Priority**: high
- **Created**: 2026-04-18
- **Reporter**: User observation
- **Assignee**: AI

## Summary
Workflows document `AskUserQuestion` (AUQ) as adapter-aware but treat it as
"nice to have" — the AI frequently skips it and falls back to plain-text even
when running on Claude Code (terminal) where AUQ is fully supported.
The tool needs to be enforced (not optional) for the Claude Code adapter.

## Details

### Current behavior
All workflow prompt blocks follow this pattern:
```
> **Adapter-aware prompt:**
> - **Claude Code (terminal):** use `AskUserQuestion` tool — spec: { ... }
> - **Cursor / other:** use text below
```

Because the fallback text always follows, the AI treats the whole block as
"use AUQ if you want, but text is fine too." In practice, the AI often uses
plain text even in Claude Code sessions, making the interactive UI non-functional.

### Desired behavior
For Claude Code adapter, AUQ must be called — the text menu is the fallback
ONLY if AUQ tool is unavailable or returns an error. The workflow phrasing must
make this unambiguous:

```
> **Claude Code (terminal) — REQUIRED:** call `AskUserQuestion` first.
>   If the tool is unavailable (error / not found), fall back to text menu below.
> **Cursor / Codex / Antigravity:** use text menu below (AUQ not available).
```

### Affected workflows (all have AUQ blocks)
- `workflows/evolve.md` — Step 2 (intent), Step 3A (complexity, brainstorm routing)
- `workflows/request.md` — Step 2 (type), Step 4A (severity), Step 4B (priority)
- `workflows/brainstorm.md` — Step 0 (mode), Step 2 (confirm scope lock), Step 5 (profile binding)
- `workflows/crystallize.md` — Step 0 (license), Step 0-B (overwrite confirm, polyrepo)

### Affected SKILL.md docs
All 4 skills that document AUQ prompts need the enforcement note.

## Acceptance Criteria
- [ ] Every AUQ block in all 4 workflows updated to: "Claude Code — REQUIRED; others — text fallback"
- [ ] Fallback condition made explicit: only when AUQ tool unavailable/errors
- [ ] SKILL.md for all 4 skills updated to reflect enforcement
- [ ] Contract test: each AUQ block in workflow files contains "REQUIRED" enforcement marker
- [ ] No regression: non-Claude-Code adapters still get plain-text menus

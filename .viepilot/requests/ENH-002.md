## Summary
`vp-docs` sau khi generate `docs/` directory cần update `README.md` để phản ánh cấu trúc docs mới và link đến các file đã tạo.

## Current Feature
`skills/vp-docs/SKILL.md` — Documentation generation skill

## Current Behavior
`/vp-docs` tạo ra:
- `docs/dev/` (architecture, contributing, testing, deployment, cli-reference)
- `docs/user/` (quick-start, features/, faq.md)
- `docs/videos/` (tutorial scripts)
- `docs/README.md` (index)

Nhưng **không** update `README.md` ở root để:
- Reflect cấu trúc `docs/` mới trong Project Structure section
- Thêm links đến docs mới vào Documentation table
- Update docs count nếu có badge

## Desired Improvement

### Trong `skills/vp-docs/SKILL.md` — `<process>`:
Thêm step cuối "Sync README.md":
```markdown
### Step N: Update Root README.md

Sau khi generate tất cả docs, update `README.md`:

1. **Project Structure section**: Update `docs/` tree để phản ánh
   các subdirectories mới (dev/, user/, videos/)

2. **Documentation table**: Thêm rows cho các docs mới:
   - Quick Start, FAQ, Troubleshooting (user docs)
   - CLI Reference, Architecture, Contributing, Testing (dev docs)
   - Feature guides (autonomous-mode, checkpoint-recovery, debug-mode)

3. **Commit**: `docs: update README.md to reflect new documentation structure`
```

### Trong `skills/vp-docs/SKILL.md` — `<objective>`:
Thêm vào **Creates/Updates**:
```
- `README.md` (Documentation table, Project Structure section)
```

## Value Proposition
User không cần manually tìm và link các docs mới vào README.md sau khi chạy `/vp-docs`.

## Summary
`workflows/brainstorm.md` Step 6 "Save Session" writes the session file to disk without
re-validating that features have been assigned to phases. The phase assignment gate exists
in Step 1 as a checklist, but nothing blocks the save if it was left incomplete.

## Details

### Current flow
1. Step 1: analyze_brainstorm → phase assignment gate (checklist items)
2. Step 2–5: brainstorm conversation, idea capture
3. Step 6: save session → writes `docs/brainstorm/session-*.md`

### Gap
Step 6 save logic does NOT validate:
- Every feature has a phase assignment
- `## Phases` section exists in the session file
- At least one feature is assigned to Phase 1

A user could proceed past Step 1 without completing the gate and still save. Then
`crystallize.md` Step 1 would stall with "no phases assigned" error — but the failure
occurs in the wrong tool (crystallize, not brainstorm).

### Desired behavior
Step 6 MUST check before writing:
```
[ ] session_data.phases is non-empty
[ ] session_data.phases.phase_1 has at least one feature
[ ] No features in `unassigned` list
```
If failed → block save, return to brainstorm Step 3 for phase assignment.

## Acceptance Criteria
- [ ] `brainstorm.md` Step 6 includes explicit pre-save validation block
- [ ] Save is blocked if `## Phases` section is empty or missing Phase 1 content
- [ ] User gets a clear prompt: "N features not assigned to phases — assign before saving"
- [ ] Edge case: pure exploratory session with no features → skip validation (add `--explore` or detect via scope lock flag)
- [ ] Contract test: brainstorm save blocked when phases missing

## Summary
The `copilot` adapter was added to `lib/adapters/index.cjs` (FEAT-019) but was never registered in the hardcoded `TARGETS` array in `bin/viepilot.cjs`. As a result, `viepilot install --target copilot` fails with "Unknown target" and `viepilot --list-targets` does not show `copilot`.

## Details

### Actual behavior
```
$ viepilot --list-targets
claude-code    Claude Code (default)
cursor-agent   Cursor Agent
cursor-ide     Cursor IDE
antigravity    Antigravity
codex          Codex
# copilot is missing
```

`viepilot install --target copilot` → likely throws "Unknown target: copilot".

### Expected behavior
```
$ viepilot --list-targets
claude-code    Claude Code (default)
cursor-agent   Cursor Agent
cursor-ide     Cursor IDE
antigravity    Antigravity
codex          Codex
copilot        GitHub Copilot
```

### Root cause (confirmed)
`bin/viepilot.cjs` line 20–26: `TARGETS` array is hardcoded without `copilot`.
`bin/viepilot.cjs` line 39: help text `--target` list is also hardcoded (no `copilot`).

The adapter registry (`lib/adapters/index.cjs`) correctly includes copilot, but the CLI layer was not updated in Phase 84.

## Affected files
- `bin/viepilot.cjs` — `TARGETS` array (line ~20) and `printHelp()` `--target` description (line ~39)

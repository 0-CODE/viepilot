# BUG: ui-direction path ambiguity — LLM reads {root}/ui-direction instead of {root}/.viepilot/ui-direction

## Meta
- **ID**: BUG-011
- **Type**: Bug
- **Status**: new
- **Priority**: high
- **Created**: 2026-04-10
- **Reporter**: User
- **Assignee**: AI

## Summary
Khi `crystallize`, `vp-auto`, và `vp-request` cần tham chiếu `ui-direction`, LLM thỉnh thoảng đọc nhầm `{root}/ui-direction/` (thư mục reference do user đặt) thay vì đúng path là `{root}/.viepilot/ui-direction/` (thư mục do ViePilot quản lý).

## Details

### Vấn đề
Trong dự án user có thể tồn tại 2 thư mục `ui-direction`:
1. `{root}/ui-direction/` — thư mục **reference tham khảo** do user tự đặt (ví dụ: reference designs, examples)
2. `{root}/.viepilot/ui-direction/` — thư mục **source of truth của ViePilot** (brainstorm sessions, frozen design contracts)

Khi LLM tìm kiếm `ui-direction` trong project (glob/search), nó tìm thấy `{root}/ui-direction/` trước hoặc đọc cả hai mà không phân biệt, dẫn đến:
- Crystallize đọc wrong artifacts
- vp-auto nhận design context sai
- vp-request brainstorm continuation bị nhiễu

### Root Cause (đã xác định)

**1. Ambiguous reference trong `brainstorm.md:416`**

Dialogue template trong Background UI Extraction confirmation có dòng:
```
1. Save to ui-direction/notes.md (background extraction)
```
→ Thiếu `.viepilot/` prefix → LLM có thể giải thích thành `{root}/ui-direction/notes.md`.

**2. Không có explicit disambiguation guard**

Các workflows (`crystallize.md`, `autonomous.md`, `request.md`) có reference đúng `.viepilot/ui-direction/` nhưng không có guard tường minh nói:
> "Do NOT read from `{root}/ui-direction/`. The ONLY valid ui-direction path is `{root}/.viepilot/ui-direction/`."

**3. LLM glob/search behavior**

Khi LLM thực hiện file search cho `ui-direction`, không có ưu tiên rõ ràng giữa:
- `ui-direction/` ← sai (reference)
- `.viepilot/ui-direction/` ← đúng (project SOT)

### Affected Files
- `~/.claude/viepilot/workflows/brainstorm.md` (line 416 — ambiguous path)
- `~/.claude/viepilot/workflows/crystallize.md` (thiếu disambiguation guard)
- `~/.claude/viepilot/workflows/autonomous.md` (thiếu disambiguation guard)
- `~/.claude/viepilot/workflows/request.md` (thiếu disambiguation guard)

## Steps to Reproduce
1. Project có thư mục `{root}/ui-direction/` (user tự tạo để chứa reference)
2. Project cũng có `.viepilot/ui-direction/{session}/` từ brainstorm
3. Chạy `/vp-auto` hoặc `/vp-crystallize` với task có UI scope
4. → LLM đọc hoặc tham chiếu `{root}/ui-direction/` thay vì `.viepilot/ui-direction/`

## Expected Behavior
- Tất cả workflows chỉ đọc `.viepilot/ui-direction/` như source of truth
- `{root}/ui-direction/` hoàn toàn bị ignored (không bao giờ được đọc bởi ViePilot workflows)
- Guard rõ ràng: nếu LLM tìm thấy `{root}/ui-direction/`, bỏ qua hoàn toàn

## Actual Behavior
LLM đọc `{root}/ui-direction/` hoặc đọc cả hai, lấy context sai → design decisions bị contaminated bởi reference materials.

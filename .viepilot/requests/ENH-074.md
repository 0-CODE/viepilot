# ENH: Centralize Shared Framework Files to `~/.viepilot/` — Single Install Source

## Meta
- **ID**: ENH-074

- **Status**: new
- **Priority**: high
- **Created**: 2026-04-24
- **Reporter**: User
- **Assignee**: AI

## Summary
Move shared framework files (`bin/`, `lib/`, `workflows/`, `agents/`, `templates/`) from
per-adapter directories (`~/.claude/viepilot/`, `~/.cursor/viepilot/`, etc.) to a single
canonical location `~/.viepilot/`. Adapter installs only write adapter-specific SKILL.md
wrapper files — no more full framework copies per adapter.

**Core problem:** Current install copies the entire framework to each adapter independently.
When two adapters are installed, they drift silently. `npm update viepilot` alone doesn't update
adapters — user must manually re-run `vp-tools install --target` for each one.

## Current Behavior

```
npm install -g viepilot           → npm global (source of truth)
vp-tools install --target claude-code  → COPY everything to ~/.claude/viepilot/
                                        bin/ lib/ workflows/ agents/ templates/ package.json
vp-tools install --target cursor       → COPY everything to ~/.cursor/viepilot/
                                        bin/ lib/ workflows/ agents/ templates/ package.json
```

**Problems:**
1. `~/.claude/viepilot/` and `~/.cursor/viepilot/` are independent copies — no link
2. `npm update viepilot` updates npm global; adapters silently run old workflows until manually reinstalled
3. Every adapter wastes disk: `workflows/brainstorm.md` (~50KB) × 5 adapters = 5 identical copies
4. All `node "$HOME/.claude/viepilot/bin/vp-tools.cjs"` refs in SKILL.md are adapter-specific hardcodes — Cursor users running Claude Code SKILL.md get wrong path

## Desired Behavior

```
npm install -g viepilot           → installs to ~/.viepilot/ (canonical shared location)
                                    ~/.viepilot/bin/vp-tools.cjs
                                    ~/.viepilot/lib/
                                    ~/.viepilot/workflows/
                                    ~/.viepilot/agents/
                                    ~/.viepilot/templates/
                                    ~/.viepilot/package.json

vp-tools install --target claude-code  → writes ONLY adapter wrappers:
                                          ~/.claude/skills/vp-*/SKILL.md (patched for claude-code)
                                          execution_context → $HOME/.viepilot/workflows/*.md
                                          vp-tools refs → $HOME/.viepilot/bin/vp-tools.cjs

vp-tools install --target cursor       → writes ONLY cursor wrappers:
                                          ~/.cursor/rules/vp-*.mdc (or equivalent)
                                          execution_context → $HOME/.viepilot/workflows/*.md
                                          vp-tools refs → $HOME/.viepilot/bin/vp-tools.cjs
```

**After npm update:** workflows/lib/bin updated in `~/.viepilot/` → ALL adapters immediately benefit.
No re-install needed.

    evolve.md
    ...
  agents/

~/.claude/skills/                     ← ADAPTER WRAPPERS ONLY
  vp-brainstorm/SKILL.md

### Path Changes in SKILL.md (all adapters unified)
```
# Before (Claude Code specific):
node "$HOME/.claude/viepilot/bin/vp-tools.cjs" check-update --silent
@$HOME/.claude/viepilot/workflows/brainstorm.md

# After (universal for all adapters):
node "$HOME/.viepilot/bin/vp-tools.cjs" check-update --silent
@$HOME/.viepilot/workflows/brainstorm.md
```
`{envToolDir}` placeholder replaced by `$HOME/.viepilot` for ALL adapters (no longer adapter-specific for framework files).

### `vp-tools install` — new behavior
1. Check if `~/.viepilot/bin/vp-tools.cjs` exists → if not, `npm install -g viepilot` first
2. Copy ONLY SKILL.md wrapper files to adapter's skill directory
3. Patch `{envToolDir}` → `.viepilot` in all SKILL.md execution_context blocks
4. Patch all `vp-tools.cjs` references → `$HOME/.viepilot/bin/vp-tools.cjs`
5. No longer copy `bin/`, `lib/`, `workflows/` to adapter directories

### `npm postinstall` hook
Add `postinstall` script to `package.json`:
```json
{
  "scripts": {
    "postinstall": "node bin/viepilot.cjs post-install"
  }
}
```
`post-install` command:
1. Creates/updates `~/.viepilot/` from npm global package location
2. Copies `bin/`, `lib/`, `workflows/`, `agents/`, `templates/` → `~/.viepilot/`
3. Writes `~/.viepilot/package.json` with version stamp
4. Silent on error (don't break npm install)

### Backward Compatibility
- `~/.claude/viepilot/bin/vp-tools.cjs` — keep as thin shim that delegates to `~/.viepilot/bin/vp-tools.cjs` (prevents breakage for users on old installs)
- Old `execution_context` paths still work via `resolveViepilotPackageRoot()` which can fall back
- `vp-tools update` command updates `~/.viepilot/` from npm latest (already somewhat does this)

### `resolveViepilotPackageRoot()` update
Current: walks up from adapter dir looking for `package.json`
New: check `~/.viepilot/package.json` first (single canonical location)

## Discussion
Reported 2026-04-24. Current `{envToolDir}` placeholder (ENH-035) already abstracts the
adapter path at install time — but still copies the full framework per adapter. This ENH
takes the next step: unify the framework install location, making adapters pure wrappers.

Key insight: SKILL.md files are already mostly identical across adapters except for the
`<cursor_skill_adapter>` / `<claude_code_skill_adapter>` block. Centralizing the workflow
references (`execution_context`) eliminates the only other adapter-specific part of SKILL.md
paths — making `{envToolDir}` = `.viepilot` universally.

# ViePilot Example: CLI Tool

A command-line tool built with ViePilot — dogfooding the framework to build a CLI.

**Setup time**: ~2 minutes  
**Stack**: Node.js (CommonJS), no external dependencies

## What This Demonstrates

- Using ViePilot to build a CLI (meta: framework building a CLI)
- `/vp-evolve` for adding commands to an existing CLI
- Keeping zero external dependencies (like ViePilot itself)
- Comprehensive test coverage with Jest

## The CLI We're Building

A `notes` CLI for managing markdown notes:

```bash
notes add "Buy groceries"
notes list
notes done 1
notes delete 1
notes search "groceries"
```

## Quick Start

```bash
# 1. Install ViePilot
# 2. Create project
mkdir my-notes-cli && cd my-notes-cli && git init

# 3. In Cursor:
# /vp-brainstorm Build a CLI tool for managing markdown notes
# /vp-crystallize
# /vp-auto
```

## Phases Generated by ViePilot

| Phase | Name | Tasks |
|-------|------|-------|
| 1 | CLI Scaffold | Entry point, help, arg parsing |
| 2 | Storage Layer | JSON file, CRUD operations |
| 3 | Commands | add, list, done, delete, search |
| 4 | Formatting | Tables, colors, progress bars |
| 5 | Tests | Unit tests, >80% coverage |

## Evolving the CLI

After v1 is done, add tag support:

```
/vp-evolve Add tag support: notes add "Buy groceries" --tag shopping
```

ViePilot creates a new milestone M2 with phases for the feature.

## Key Difference from Web Apps

CLI tools have a clear acceptance test:

```bash
# Phase 1 complete when:
./bin/notes.js --help   # shows all commands
./bin/notes.js add "test"   # no error
./bin/notes.js list     # shows the added note
```

ViePilot's quality gates check exactly this after each phase.

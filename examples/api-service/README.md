# ViePilot Example: API Service

A Node.js REST API with authentication, built with ViePilot.

**Setup time**: ~3 minutes  
**Stack**: Node.js, Express, PostgreSQL, JWT

## What This Demonstrates

- Multi-phase API development workflow
- `/vp-request --feature` for adding features mid-build
- `/vp-debug` for resolving test failures
- Checkpoint recovery with `/vp-rollback`

## Quick Start

```bash
# 1. Install ViePilot (if not already done)
# See examples/web-app/README.md for install steps

# 2. Create a new project
mkdir my-api && cd my-api && git init

# 3. In Cursor:
# /vp-brainstorm Build a REST API for user management with JWT auth
# /vp-crystallize
# /vp-auto
```

## Expected Project Structure

```
my-api/

├── src/

├── tests/
│   ├── auth.test.js

```

## Phases Generated by ViePilot

| Phase | Name | Tasks |
|-------|------|-------|
| 1 | Project Setup | Express, middleware, env |
| 2 | Database | PostgreSQL schema, migrations |
| 3 | Auth Routes | Register, login, JWT |
| 4 | User Routes | CRUD with authorization |
| 5 | Tests | Jest + Supertest |

## Adding Features Mid-Build

After Phase 2, add email verification:

```
/vp-request --feature Add email verification on registration
```

ViePilot inserts a new phase 2.5 between 2 and 3.

## Debugging Failed Tests

If Phase 5 tests fail:

```
/vp-debug investigate: JWT token validation failing in test environment
```

ViePilot tracks the investigation across multiple attempts.
